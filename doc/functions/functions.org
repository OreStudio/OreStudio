A [[id:66f2a9e8-0385-4d56-b4b1-0957208990dc][function]] is a named role — /what someone (or some LLM session)
does when they are wearing a given hat/. A function doc is the
priming context an LLM session loads when it takes on the role: the
purpose, what to pre-read, which [[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skills]] to use, what /not/ to do.
Functions are role-shaped (descriptive); skills are task-shaped
(operational). Functions compose skills.

There is one function per [[id:926b916e-a57e-498e-99ab-720b583c0362][cybernetic level]], plus one for the
independent audit channel.

* Per system

- [[id:b544713a-ba23-4d5a-9cee-084212d7c8ba][S1 Agent]] — does one [[id:31c868b9-306e-4282-ba8c-07e647021b34][task]] and lands one PR.
- [[id:eeca6981-3240-44d9-ada7-4b8c64a76b44][S2 Orchestrator]] — coordinates one [[id:699a8d45-b67e-4d3e-9206-24fa17c51ada][story]], decomposes it into
  tasks, oversees S1 agents.
- [[id:f1d99662-c6b1-4839-92e1-d061ccf6b83f][S3 Sprint Planner]] — opens, runs, and closes a [[id:0820b7fd-147c-4832-ac25-c043d38d5b61][sprint]]; refines
  the [[id:d4219199-7b17-4a89-b4bc-6019738642da][product backlog]].
- [[id:049b086e-f131-4d7a-8244-c1b2ff297ab6][S3* Auditor]] — observes the whole information architecture from
  outside; reports drift.
- [[id:66eb6824-52b8-4603-b302-f805da9ec443][S4 Version Planner]] — plans a [[id:ca4d4bde-2eab-4f0a-bc8b-7c0bcb17bc82][version]], sequences its sprints,
  judges its Definition of done.
- [[id:5cad21da-21fb-460b-9286-8998b4c6ef52][S5 Identity Steward]] — maintains the [[id:2f71292f-cdb0-4e2e-b50f-4f02e10597c4][product identity]], rules on
  what is in / out of scope.

Each function doc lists its pre-reads, the [[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skills]] it draws on, the
recipes it exercises, and explicit boundaries (what this role does
/not/ do, and which other function picks it up instead).
