** Summary

- *Root cause*: MSVC C1202 fires in any TU that reflects both =get_trades_response= (a =vector<trade>=) and another large rfl type, because reflect-cpp's =no_duplicate_field_names.hpp= runs an O(n²) compile-time field-uniqueness check — at 25 fields =trade= blows the internal template depth limit.
- *Fix*: Split =trading::domain::trade= into five sub-structs (=trade_identity=, =trade_parties=, =trade_classification=, =trade_lifecycle=, =trade_audit=) composed via =rfl::Flatten<T>=. The largest sub-struct has 7 fields. The flat JSON wire format is preserved. The SQL table and =trade_entity= (sqlgen) are unchanged — sqlgen never goes through rfl, so it was never affected.
- *Codegen*: Added a new =field-group= model type that generates a single =.hpp= per sub-struct (no primary key, no audit columns, no DB entity, no mapper). Qt mustache templates extended with optional =*_access= override fields so flattened sub-struct access paths (=identity.get().id=) work for trade while all other entities fall back to flat access.
