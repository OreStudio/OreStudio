** Summary

- *Label filter on login dialog*: when =ORES_CHECKOUT_LABEL= is set (e.g. =local1=), environments whose =subject_prefix= ends with =.local1= are automatically tagged on startup. The login dialog shows a label filter combo when ≥2 distinct tags exist across environments/connections; the combo pre-selects the checkout label so developers only see relevant connections by default. Users with a single label or no labels see no change.
- *=.env= version guard in =start-client.sh=*: sources =validate_env_version.sh= immediately after loading =.env=, giving a clear error when the file is stale.
- *Recent parties in party picker*: the last 5 selected parties are persisted in a new =recent_parties= SQLite table (connections.db) and shown as a "Recent" section at the top of =PartyPickerDialog=, above "All Parties". The text/centre filter applies across both sections; the Recent header hides automatically when no matching recents exist.
