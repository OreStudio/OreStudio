* Question

How do I validate component documentation before committing?

* Answer

From the project root:

#+begin_src sh :results verbatim
./projects/ores.codegen/validate_docs.sh
#+end_src

Two checks run in sequence:

1. /Component structure/ — every =projects/ores.*/= has a
   =modeling/component_overview.org= with the required v2 frontmatter
   (=:ID:=, =#+type: component=, =#+description:=), required sections, and
   at least one =.puml= diagram. Exceptions are listed in
   =projects/ores.codegen/docs_exceptions.txt=.
2. /Version tagging/ — every =.org= file in the repo carries
   =#+version: 1= or =#+version: 2= (see
   [[id:23BCAC49-325F-4083-B854-87EA747C2ABE][How do I audit and fix document version tags?]]).

Fix any reported errors, then re-run until the script exits =0=.

* Script

=projects/ores.codegen/validate_docs.sh= — wraps the component-structure
checker and =doc_version_audit.py --check= in sequence.

* Tested by

CI gate on every PR. The check runs as part of the documentation
validation workflow and the PR goes red if it exits non-zero.
