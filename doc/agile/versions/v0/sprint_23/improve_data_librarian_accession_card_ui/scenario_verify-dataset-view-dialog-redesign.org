** Log in and select Barclays Plc

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123=, select
=BARCLAYS PLC=. Confirm all menus enable with no errors.

** Open the Data Librarian and a dataset's accession card

Open Reference Data > Data Librarian (or System > Data, wherever it
lives in this build). Pick any seeded dataset (e.g. =ORE Asset
Class=) and open its detail view (=DatasetViewDialog=).

** Persistent header banner

Confirm a header banner sits above the tabs showing the dataset's
Name and, on a second line, Code and Version. Confirm a "Copy ID"
button is present; click it and paste somewhere to confirm the full
UUID was copied (the banner itself only shows a truncated ID with a
tooltip holding the full UUID).

** Overview tab: no more Name/Code/Version/ID rows, governance pills

Open the Overview tab. Confirm the General section no longer repeats
Name/Code/Version/ID (now only in the header banner). Confirm Origin,
Nature, and Treatment in the Data Governance section render as
coloured pill badges, not plain text.

** Provenance & Methodology tab is merged

Confirm there is a single "Provenance & Methodology" tab (not two
separate tabs), showing the Provenance property tree, the Methodology
property tree, and the implementation-details text browser stacked
vertically.

** Dependencies tab still shows the interactive lineage graph

Confirm the Dependencies tab is unchanged: the interactive node/graph
lineage view still renders correctly (this tab was intentionally left
untouched by this task).

** No error toast or log error during the whole flow

Check the client's log tail
(=build/output/<preset>/publish/log/ores.qt.<colour>.log=) for any
ERROR-level line during this flow.
