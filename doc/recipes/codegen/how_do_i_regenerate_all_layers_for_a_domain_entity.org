#+title: How do I regenerate all layers for a domain entity?
#+description: Run all codegen profiles (SQL, all C++) for a single domain entity in one pass and validate the output.
#+type: recipe
#+level: cross
#+filetags: :codegen:recipe:entity:regenerate:
#+created: 2026-06-26

This recipe covers all code generation profiles for a =domain_entity= model.
See [[id:FE693595-6693-459B-AD9B-EB7E3ABBBC88][ores.codegen architecture]] for the profile system and [[id:F556E446-EB91-4656-BEDC-15AE0F7B5ECD][Entity commissioning process reference]] for output location checks and drift classification.

* Question

How do I regenerate all the code for a domain entity?

* Answer

Use =compass codegen entity generate <entity>= — it targets exactly one entity across every applicable profile (SQL and all C++ layers, including Qt), so there is no component-wide diff to scope or discard:

#+begin_src sh
./compass.sh codegen entity generate rounding_type
#+end_src

Pass =--address <address>= to restrict to one layer (e.g. =ores.cpp.qt=, =ores.sql.schema=) when you only touched one facet's inputs:

#+begin_src sh
./compass.sh codegen entity generate book --address ores.cpp.qt
#+end_src

Companion commands for the same entity-scoped workflow:

#+begin_src sh
./compass.sh codegen entity list                # every entity, grouped by metatype
./compass.sh codegen entity show book            # generated files + staleness, this entity only
./compass.sh codegen entity diff book             # git diff of this entity's generated files only
#+end_src

*Do not* use =compass codegen regenerate --component <component> --profile <profile>= for single-entity work — that command regenerates *every* entity in the component, and on a component with template/generator drift accumulated against stale entities (common in this codebase — see [[id:3A49A0B0-9FCF-466C-B097-ECF9A60C0F3D][Adding a flagged combo field to a codegen entity]] for a worked example of the resulting diff size), the resulting diff can span dozens of unrelated files that then have to be manually discarded. =codegen regenerate= is for genuine component-wide work (e.g. rolling out a template change across every entity that uses it), not for adding one field to one entity.

*Before committing*, verify each generated file landed in the correct split project directory — not the stale monolith paths. See [[id:F556E446-EB91-4656-BEDC-15AE0F7B5ECD][Entity commissioning process reference]] § Codegen sync task acceptance criteria, output location check.

For per-layer details and drift notes, see the individual layer recipes in the See also section.

* Script

=./compass.sh codegen entity generate <entity>= is the entry point (=compass_codegen_entity.py=, lazily loaded when =entity= is codegen's first argument). With no =--address=, it runs every profile applicable to that entity's metatype. See [[id:FE693595-6693-459B-AD9B-EB7E3ABBBC88][ores.codegen architecture]] for the profile/address registry.

* Tested by

Manual: run the command for a target entity, inspect the diff (already scoped — no manual filtering needed), verify output paths against the location table in [[id:F556E446-EB91-4656-BEDC-15AE0F7B5ECD][Entity commissioning process reference]].
