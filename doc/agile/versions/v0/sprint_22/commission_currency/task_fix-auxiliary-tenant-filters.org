* Investigation

Symptom (screenshot 2026-06-04): every rounding type appears twice in the
Qt MDI window — once =modified_by ores_local1_ddl_user= (system-tenant
seed) and once =ores_local1_iam_service= (tenant copy).

Root cause chain:

1. Commit =d7f6454a6= (2026-02-24) added =rounding_types=,
   =currency_asset_classes= and =currency_market_tiers= to the refdata
   copy step of =ores_iam_provision_tenant_fn= — tables legitimately hold
   system rows *and* per-tenant rows. Monetary natures joined later; the
   provisioner now copies 15 datasets.
2. The repositories for the three auxiliaries never filter reads by
   tenant: =rounding_type_repository.cpp=, =monetary_nature_repository.cpp=
   and =currency_market_tier_repository.cpp= contain zero =tenant_id=
   references, so queries return both tenants' rows.
3. Commit =bcb0c2bd2= (2026-06-03) fixed the identical bug for DQ change
   reason categories/reasons by adding =where("tenant_id"_c == tid && ...)=
   to every read/remove — the refdata auxiliaries were not covered.

Not affected: =asset_class_repository= already filters via raw SQL
(=WHERE tenant_id = $1=); the other provisioner-copied refdata tables
(book statuses, purpose types, party types/statuses, contact types, party
ID schemes) all carry sqlgen tenant filters.

A structural wrinkle: unlike =change_reason_entity=, the three auxiliary
entities do not declare a =tenant_id= field — writes relied on the insert
trigger's =ores_iam_validate_tenant_fn(null)= falling back to the session
tenant. The fix adds the field, stamps it in the repository write path
from context, and filters all reads/removes.

Promoted from inbox capture =2CA98337-C2F0-482F-87AC-C343B02DA5B2=
("currency rounding types are duplicated", 2026-05-27).

* Plan

Per repository (rounding_type, monetary_nature, currency_market_tier):

1. Entity header: add =std::string tenant_id;= (repository-layer only —
   domain, protocol and Qt remain untouched).
2. =write()= (single + vector): stamp =tenant_id= from
   =ctx.tenant_id().to_string()= after mapping.
3. =read_latest(ctx)=, =read_latest(ctx, code)=, =read_all(ctx, code)=,
   =remove(ctx, code)=, =remove(ctx, codes)=: prepend
   ="tenant_id"_c == tid= to the where clause.
