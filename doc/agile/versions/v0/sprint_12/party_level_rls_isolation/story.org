* Decisions

- /Visible-party set computed at login/ :: avoids per-query recursive CTE; tradeoff is hierarchy changes need re-login.
- /System party sees all/ :: necessary for admin paths; matches the system-tenant-cross-tenant pattern from sprint 11.
- /party_counterparties as the first scoped junction/ :: proves the pattern end-to-end before applying broadly.

* Out of scope

- Party picker UI for multi-party users (deferred).
- Live re-computation of visible set on hierarchy change.
