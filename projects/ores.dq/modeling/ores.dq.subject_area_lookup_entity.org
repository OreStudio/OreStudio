Sub-classification within a data domain. Examples: currencies,
countries, images. Rows are authored directly (not mirrored from an
external source).

* Primary key
:PROPERTIES:
:column:  name
:type:    text
:is_text: true
:END:

Unique name identifying this subject area within its data domain.
Examples: "currencies", "countries", "images".

* Columns

** domain_name
:PROPERTIES:
:type:     text
:nullable: false
:END:

Name of the data domain this subject area belongs to. References
=ores_dq_data_domains_tbl= (soft FK). The real table's natural key is
compound (=name= + =domain_name=), but =lookup_entity= only supports
a single-column primary key today
([[id:52FE40FA-C696-4C77-8A3F-65FB00196EAD][52FE40FA]]); per that
descoping decision =domain_name= is modeled as a plain =Column=
rather than part of a true composite key -- the generated DDL's
primary key/exclusion constraint/uniqueness index cover
=(tenant_id, name, valid_from, valid_to)= only, weaker than the
hand-written table's =(tenant_id, name, domain_name, ...)=. Do not
regenerate/swap this table over the hand-written one until 52FE40FA
lands.
