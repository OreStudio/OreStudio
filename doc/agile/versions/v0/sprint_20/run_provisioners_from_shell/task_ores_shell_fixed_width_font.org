* Goal

ores.shell output is not rendered with a fixed-width font, which
misaligns all table output (currencies list, history, history-diff)
and makes the shell effectively unusable from the Qt UI at present —
a blocker for this story, since the provisioner shell commands and
scripts must be exercised through the shell. Pin a monospace font on
the Qt-embedded shell display; check other display contexts (Emacs
term/comint buffer, dashboard-spawned terminal) and pin a monospace
face there too where needed. If the shell's own column-padding
assumptions contribute (e.g. wide glyphs), note and fix those as
well.

* Result

Root cause was a font-mechanism conflict, not a missing font. The Qt
shell widgets called =setFont(FontUtils::monospace())=, but the global
=TradingStyle.qss= theme set =QWidget { font-family: <sans-serif> }=,
and a QSS font-family always overrides =setFont()= — so the terminal
rendered proportional. A secondary trap: =FontUtils::monospace()= asked
for "Fira Code", which on the reporter's machine fuzzy-matched the only
near-name installed — the symbols-only "Fira Code Symbol" (not fixed
pitch).

Fixed by giving font /family/ a single owner — the Qt font system, not
the stylesheet:

- Dropped =font-family= (and the global =font-size=) from the =QWidget=
  QSS rule; the theme keeps colours, borders and scoped per-widget
  sizes.
- Set the application default proportional font via
  =QApplication::setFont(FontUtils::applicationFont())= in =main=.
- Monospace surfaces (shell output/input, script editor) use
  =QWidget::setFont(FontUtils::monospace())=, now authoritative; the
  per-widget =setStyleSheet= workaround and =monospaceCssFragment= were
  removed.
- =FontUtils::monospace()= resolves to the first /installed/ and
  /genuinely fixed-pitch/ family from a fallback chain (Fira Code →
  DejaVu Sans Mono → Liberation Mono → Menlo → Consolas → Courier New),
  so the symbols-only "Fira Code" is skipped. The resolution is logged
  at =ores.qt.font_utils= ("Monospace font resolved to: '<family>'" at
  info, each candidate's installed/fixed_pitch at debug, a warn when
  none match) so the active face is a trivial grep — verified resolving
  to =DejaVu Sans Mono= on the reporter's machine.

Bundled into the same PR: =doc/recipes/shell/= is partitioned into the
library's category sub-folders (mirroring =scripts/library/=), with the
tangler now scanning recipes recursively. Links are unaffected (org-roam
ids).
