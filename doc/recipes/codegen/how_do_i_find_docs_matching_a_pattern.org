#+description: List documents by regex on title/description, by filetag, by #+type:, or by path prefix, using doc_list.sh.
#+type: recipe

#+level: cross

* Question

How do I find documents in the repo when I know a fragment of the
title, a tag, the doctype, or roughly which folder it lives in?

* Answer

Run =./projects/ores.codegen/doc_list.sh= with one or more filters.
Each filter narrows the result; combining filters AND's them.

1. /Regex on title or description/ (case-insensitive):

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --regex 'currency'
   #+end_src

2. /By =#+type:= /  — recipe, knowledge, task, story, sprint, version,
   component, capture, function, skill, meta, ...:

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --type recipe
   ./projects/ores.codegen/doc_list.sh --type knowledge
   #+end_src

3. /By filetag/ (repeatable; multiple tags are AND'd):

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --tag cli
   ./projects/ores.codegen/doc_list.sh --tag build_quality --tag task
   #+end_src

4. /Under a path prefix/ — restrict to one subtree (repeatable):

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --under doc/agile/versions/v0
   ./projects/ores.codegen/doc_list.sh --under doc/recipes/cli
   #+end_src

5. /Combine filters/ for a precise lookup:

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh \
       --type recipe --tag cli --regex 'list'
   #+end_src

6. /Just count/ how many docs match — useful for sanity checks:

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --type recipe --count
   #+end_src

7. /Pipe paths into grep/xargs/ when you want to read or edit the
   matched files:

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --tag cli --paths | \
       xargs grep -l 'compute_host'
   #+end_src

8. /Sort by recency/ — most recently touched first:

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --sort updated | tail -20
   #+end_src

* Output format

Each match is one line:

#+begin_example
<uuid> | <type> | <title>  —  <description>   (rel/path)
#+end_example

The =--paths= mode emits one path per line for use in pipes; =--count=
emits a single integer.

* Script

The wrapper is =projects/ores.codegen/doc_list.sh=. It delegates to
=scripts/doc_list.py=, which uses the shared =doc_index.py= library
(see [[id:e8231c22-cb31-4d85-a9c0-7602953764ff][How do I show a doc by UUID?]] for the model).

* Tested by

Manual usage — these scripts are read-only over the doc tree, so the
=#+type:=, =#+filetags:=, and =:ID:= contract enforced by
[[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document types]] and audited by [[id:8810e0f9-c5b9-4cfa-a710-774ad6c771dd][lifecycle]] is what guarantees
correctness.
