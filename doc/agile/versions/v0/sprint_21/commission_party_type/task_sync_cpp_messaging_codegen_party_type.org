* Result

Ran all three messaging codegen profiles for =party_type= and classified drift. Template bug found and fixed.

** Drift table

| Profile      | Artefact                        | Diff? | Category                       | Decision                                         |
|--------------+---------------------------------+-------+--------------------------------+--------------------------------------------------|
| protocol     | party_type_protocol.hpp         | Yes   | Template gap                   | Accept — pre-template hand-written code; updated to canonical API shape |
| nats-eventing | party_type_changed_event.hpp   | No    | —                              | Zero diff; no action                             |
| nats-handler | party_type_handler.hpp          | Yes   | Template gap + Template bug    | Accept — handler also predated template; fixed template bug (=entity_plural= → =entity_plural_short= in list response) |

** Changes accepted in protocol

- NATS subject separators: =party-types= → =party_types= (underscore convention).
- offset/limit field types: =int= → =std::uint32_t= (requires =<cstdint>=).
- List response field name: =party_types= → =types= (short form per =entity_plural_short=).
- Delete request: =std::string type= → =std::vector<std::string> codes= (batch delete; field name from model knob =delete_request_id_field=).
- History request: =std::string type= → =std::string code= (field name from model knob =history_request_id_field=).
- =success= / =message= fields added to all response types.
- =from()= factory method added to =save_party_type_request=.
- History response field ordering: =history= first, then =success/message=.

** Changes accepted in handler

- =log_handler_entry= / =[[maybe_unused]]= pattern removed (template normalisation).
- Context naming: =ctx_expected=/=ctx= → =req_ctx_expected=/=req_ctx=.
- Constructor: stacked-initialiser → single-line style.
- List: =resp.party_types= → =resp.types=; decode-inside-if pattern; =static_cast= removed (request now =uint32_t=).
- History: =req->type= → =req->code=; method moved before =remove=.
- Remove: =svc.delete_type(req->type)= → =svc.delete_types(req->codes)= (batch).
- Trailing newline before =#endif= added.

** Template bug fixed

=cpp_nats_handler.hpp.mustache= (and its org source =cpp_nats_handler_hpp.org=) used =resp.{{entity_plural}}= in the list method (both with and without offset/limit filter). The protocol template uses =entity_plural_short= for the response field name. Fixed 4 occurrences: 3 × =resp.{{entity_plural}}= and 1 × =resp.{{entity_plural}}.size()=.

** Model knobs corrected

=ores.refdata.party_type.org= had stale overrides in the =** Repository= PROPERTIES block:

| Knob                     | Was    | Now    |
|--------------------------+--------+--------|
| =delete_request_id_field=  | =type=   | =codes=  |
| =single_delete=            | =true=   | =false=  |
| =history_request_id_field= | =type=   | =code=   |

These now match the primary key column name (=code=) and the batch-delete convention.
