* Goal

Return the change reason constants to their semantic home. They are
DQ-domain — the =change_reason= entity lives in =ores.dq.api/domain= —
but the constants sit in =ores.database=, where they were parked to
break a historical dq↔iam circular dependency. =ores.database= has
zero internal uses of them; it is purely a host.

Now that =ores.dq.api= is light (eventing/platform/utility only,
after the [[id:681BBF6B-4DD8-411A-9B19-76AA5E14C1FC][inventory task]] cleanup), the move back is cycle-safe:
=iam.core → dq.api= cannot loop because =dq.api= depends on nothing
iam-ish. Per project direction, no compatibility aliases: move the
header, fix every consumer, delete the =ores.database= copy.

Deliberately deferred from the inventory task as a post-cleanup
change — the precondition (light dq.api) had to land and be verified
first.
