* When to use this skill

When implementation of a feature or task is otherwise complete and it's
time to verify it before raising a PR: a confirmed build, a fresh
Barclays-provisioned database, the full test suite, a written test
scenario, and a client handed to the user to execute it.

* How to use this skill

Run [[id:BE325E3E-1BAB-4D9D-964E-78333CDB97D7][How do I test a feature after implementation?]] end to end:

1. Its *Prerequisites* section — build/test if needed, validate
   schemas, then [[id:0A22B752-D5CF-4B36-8429-2950130AAABD][ready the environment]] (fresh database, Barclays Plc
   provisioned by you, not the user). Non-negotiable, in order.
2. Its *Writing and running the scenario* section — a =test_scenario=
   doc covering every aspect of the feature (CRUD + eventing + history
   for an entity), then a client started with the scenario preloaded
   and handed to the user to execute.

*Leave =Clients= in =* Scenario Info= genuinely empty* for a
single-client scenario (the common case) — any non-empty value there,
even a placeholder like "single client", makes the Runner load *zero*
steps, silently.
