This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *inbox* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

There is no =repository_market_observation_repository_tests.cpp= (or
equivalent) in =projects/ores.marketdata/core/tests/= — the sibling
codegen entity =market_observations= (plural) has one, but the
hand-written =market_observation= (singular) repository, including its
=read_latest_by_series_id=/=get_total_market_observation_count_by_series_id=
=list_by= methods and the plain =read_latest=/=save=/=remove= surface,
has zero direct repository-level test coverage. Add a test file
covering at minimum: series_id filtering correctness,
=observation_datetime desc= ordering (a real ordering regression was
caught only by manual PR review, not a test — see References), and
pagination (offset/limit).

* Why

A repository-level test asserting result ordering/content on the
=list_by_series_id= path would have caught a real ordering regression
(list_by defaulting to primary-key/UUID order instead of
=observation_datetime desc=) directly, rather than relying on manual
PR review to notice it. This gap pre-dates that PR — the
=market_observation_repository= file itself has never had tests — but
it's now the only path every real caller uses, making the gap higher
value to close.
