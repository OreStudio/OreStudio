* When to use this skill

When creating any new v2 information-architecture document — task,
story, sprint, version, component, recipe, knowledge, or skill — and
you want the frontmatter, UUID, and skeleton sections generated rather
than hand-written.

* How to use this skill

1. Identify the document type from the contract in [[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document_types]].
2. Pick a slug. Conventions:
   - tasks / stories / sprints / versions / knowledge: =snake_case=.
   - skills: =kebab-case= (the slug also becomes the skill's =name:=).
   - recipes: =how_do_i_<thing>= filename stem.
   - components: matches the component folder (e.g. =ores.codegen=).
3. Run =projects/ores.codegen/generate_v2_doc.sh= with =--type=,
   =--slug=, =--parent-dir=, =--title=, =--description=, and
   =--tags=. Parent =:ID:= and =#+title:= are auto-detected from the
   parent document for task/story/sprint, so no UUID typing is
   needed. The recipe at [[id:3153aa4b-23d4-4111-9710-5a4971102edc][How do I create a new v2 doc?]] has copy-
   pasteable invocations per type.
4. Fill in the skeleton sections in the generated file.
5. For stories that continue from a previous sprint, update the
   predecessor story to add =#+successor:= and a =Continued in:=
   note in =* Decisions= (this step is manual).
