* Goal

Two related codegen cleanup items surfaced while closing out task
257F85B2 ("Update ores.cpp.qt codegen templates for the generic
HistoryDialog"):

1. *Delete the per-entity history-dialog archetypes outright.*
   257F85B2 marked =ores.cpp.qt.history_dialog_impl.org=/
   =history_dialog_header.org=/=history_dialog_ui.org= =#+default:
   disabled= — "retained for reference" — but the per-entity
   =*HistoryDialog= class they generate is deprecated and being
   actively removed, not a pattern anyone should reference or
   accidentally re-enable. Delete the three =.org= archetypes and
   their tangled =cpp_qt_history_dialog.{hpp,cpp}.mustache=/
   =qt_history_dialog_ui.mustache= outputs entirely, and any
   =facet_catalogue.org=/=template_variables.org= references to them,
   so the pattern cannot be generated by anyone, full stop.
2. *Reconcile the dynamic_combo detail-dialog codegen drift* captured
   in [[id:BE3DEB46-1022-4A68-A1CD-7575CF6CDC33][Reconcile dynamic_combo detail-dialog codegen drift]]: several
   entities' detail dialogs (e.g. currency) still use the old
   =setCurrentText()=-based combo pattern instead of the current
   template's =findData()=/=setCurrentIndex()= + loading/error
   placeholder text — found while regenerating currency for 257F85B2,
   deliberately left out of that diff to keep it scoped.

* Acceptance

- =ores.cpp.qt.history_dialog_impl.org=/=history_dialog_header.org=/
  =history_dialog_ui.org= and their tangled =.mustache= outputs no
  longer exist in =projects/ores.codegen/library/templates/=.
- No remaining reference to them in =facet_catalogue.org=,
  =template_variables.org=, or =cpp_qt.org=.
- =compass codegen entity generate <entity> --address ores.cpp.qt=
  for any entity produces no per-entity =*HistoryDialog=
  class/header/.ui output — confirmed the generator itself no longer
  knows how to emit one (not just that nobody currently calls it).
- Every =dynamic_combo= entity's detail dialog is reviewed against
  the current template; diffs that are genuinely additive/correct
  (loading/error placeholder text, =findData()=-based value matching)
  are applied and committed per entity, with each diff inspected
  before applying — not a blind bulk regenerate, since this changes
  combo value-matching semantics and a wrong diff could silently break
  a working dialog.
- [[id:BE3DEB46-1022-4A68-A1CD-7575CF6CDC33][Reconcile dynamic_combo detail-dialog codegen drift]] closed
  (discarded/resolved) once addressed.
