#+title: How do I generate PlantUML diagrams?
#+description: Build all PlantUML diagrams with the `mad` target, or a single component's diagram with `generate_<comp>_diagram`.

* Question

How do I (re)generate the PlantUML diagrams the project uses in its
docs and site?

* Answer

Via =compass build= (not raw =cmake --build=, so it takes the
host-wide build lock — see [[id:F176FCA1-68D5-420D-B076-75A65FEE5EBB][How do I build the system?]]).

** All diagrams

The =mad= target ("make all diagrams") builds every PlantUML diagram
declared in the build:

#+begin_src sh :results verbatim
./compass.sh build mad
#+end_src

** A single component's diagram

Component diagrams follow =generate_<component>_diagram= where
=<component>= uses dots, just like the component name:

#+begin_src sh :results verbatim
./compass.sh build generate_ores.refdata_diagram
#+end_src

#+begin_src sh :results verbatim
./compass.sh build generate_ores.comms_diagram
#+end_src

** The system-level diagram

For the whole-project view used on the home page:

#+begin_src sh :results verbatim
./compass.sh build generate_ores_diagram
#+end_src

** Prerequisites

- PlantUML on PATH (typically via =apt install plantuml=).
- The project must be configured (=cmake --preset ...=).

* Script

The targets above are wired in =CMakeLists.txt= via the project's
PlantUML helpers; the source =.puml= files live alongside the
component models under =projects/<comp>/modeling/= and at
[[proj:projects/modeling][=projects/modeling/=]].

* Tested by

Manual — the diagrams are regenerated when their =.puml= sources
change.
