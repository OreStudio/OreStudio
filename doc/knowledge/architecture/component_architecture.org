#+title: Component architecture
#+description: api/core/service split, facet placement, CMake dependency chain, and scaffold profiles for ORE Studio components.
#+type: knowledge
#+level: cross
#+filetags: :knowledge:architecture:component:cmake:codegen:
#+created: 2026-05-21
#+updated: 2026-05-21

Most domain components in ORE Studio are split into three sibling
projects: =.api= (public types, no heavy deps), =.core= (repositories,
services, DB access), and =.service= (runnable host with message
handlers).  Not every component needs all three — a pure reference
component may stop at =.api=; one without a service boundary stops at
=.core=.  Scaffold each part with codegen (=--profile component-api=,
=--profile component-core=, =--profile component-service=) rather than
hand-writing boilerplate. Return to [[id:7474935C-95D1-3034-B833-83DCD9FB3A09][Knowledge]].

* api / core / service split

| Part | Role | Key deps |
|---|---|---|
| =ores.COMPONENT.api= | Public domain types, generators, messaging protocols, eventing | =ores.eventing=, =ores.platform= |
| =ores.COMPONENT.core= | Repositories, services, DB access — depends on =.api= | =ores.COMPONENT.api=, =ores.database=, =sqlgen= |
| =ores.COMPONENT.service= | Runnable host: message handlers, startup — depends on =.core= | =ores.COMPONENT.core=, =ores.service=, =ores.nats= |

* Facet placement

Code is organised into facets within =include/= and =src/=. Each
facet belongs in exactly one part:

| Facet | =.api= | =.core= | =.service= | Description |
|---|---|---|---|---|
| =domain= | ✓ | | | Domain types (POCOs), JSON I/O, table I/O |
| =generators= | ✓ | | | Test-data generators |
| =messaging= | ✓ | | | NATS request/response types and protocol |
| =eventing= | ✓ | | | Event types |
| =repository= | | ✓ | | sqlgen entities, mappers, repositories |
| =service= | | ✓ | | Business-logic service layer |
| =app= | | | ✓ | Executable hosting and startup |
| =config= | | | ✓ | CLI parser and options struct |

* CMake dependency chain

Typical =target_link_libraries= per part (generated by the scaffold
profiles; adjust to actual deps after generation):

| Part | PUBLIC | PRIVATE |
|---|---|---|
| =.api= | =ores.eventing.lib=, =ores.platform.lib= | =ores.utility.lib=, =faker-cxx=, =libfort= |
| =.core= | =ores.COMPONENT.api.lib=, =ores.nats.lib= | =ores.service.lib=, =ores.platform.lib=, =ores.utility.lib=, =ores.security.lib=, =faker-cxx=, =libfort= |
| =.service= | =ores.service.lib=, =ores.security.lib=, =ores.COMPONENT.core.lib=, =ores.eventing.lib=, =ores.nats.lib=, =ores.database.lib=, =ores.utility.lib=, =ores.telemetry.lib= | — |

*CMake visibility rules:*
- =PUBLIC=: headers exposed in the component's own public API; consumers
  also see (and link) the dependency.
- =PRIVATE=: used only in =.cpp= files; consumers do not inherit it.
- =INTERFACE=: expose to consumers without linking directly (avoids
  duplicate-library warnings on macOS).

*Common transitive deps — do not add explicitly:*
- =reflectcpp= arrives transitively via =ores.platform.lib= (PUBLIC)
- =Boost::program_options= arrives via =ores.telemetry.lib= (PUBLIC)

* Scaffold profiles

Codegen (=--profile <name>=) generates all boilerplate for each part.
The =.api= and =.core= profiles produce 9 files each; =component-service=
produces 18.  See [[id:08FBD248-257A-A474-DD43-DB08949978F1][ORE Studio Codegen]]§"Profile catalogue" for the
template list.

| Profile | Part | Files generated |
|---|---|---|
| =--profile component-api= | =.api= | CMakeLists (root, src, tests, modeling), component header, stub header+impl, test main+stub |
| =--profile component-core= | =.core= | same set with core-flavoured CMake deps |
| =--profile component-service= | =.service= | same core set + app/config headers+impls + main.cpp |

After generation, register each part in =projects/CMakeLists.txt= in
dependency order (=.api= → =.core= → =.service=).  Each part also needs
a =.puml= diagram stub in =modeling/= — not yet generated by codegen,
so add it manually (one per part).

* Directory structure

Each part follows the same layout:

| Directory | Purpose |
|---|---|
| =include/ores.COMPONENT.PART/= | Public headers, organised by facet |
| =src/= | Implementation files, organised by facet |
| =tests/= | Catch2 tests |
| =modeling/= | =component_overview.org= + =*.puml= + =*.png= |
