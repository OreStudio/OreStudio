** Summary

Follow-up to #714. That PR moved package_uri to the junction and introduced vcpkg triplets; this one makes the Qt dialog usable end-to-end, aligns the sample tarball with the new convention, and covers the junction sync path with tests.

- *Packages tab (=4e190b3a2=)* — new tab with a =QTableWidget= showing one row per assigned platform (Platform / File / Status / Browse…). Each row tracks a lifecycle: =NoFile → Selected → Uploading → Uploaded/Failed=. Edit mode pre-seeds rows as =Uploaded= from =list_app_version_platforms= so existing versions don't have to re-upload. Rows sync with the Platforms tab's assigned list; add/remove there adds/removes here.
- *Save+Upload collapsed* — the old two-button dance (Upload Package vs Save) made ordering implicit and easy to get wrong with per-triplet blobs. Save now uploads every pending file in parallel, then fires =save_app_version_request= once all PUTs succeed. Validation requires a file or remote URI on every assigned row.
- *Package rename (=405b6c225=)* — =external/ore/packages/ore_1.8.15.0.tar.gz= → =ore-1.8.15.0-x64-linux.tar.gz=; binary inside renamed to match (=ore-1.8.15.0-x64-linux=); =manifest.json= updated. SQL seed already expected this name.
- *Integration tests (=ca491a015=, =49599a15b=)* — new =ores.compute.core.tests= cover =app_version_platform_repository=: initial insert, swap (partial overlap), empty-set drop, idempotent re-save (exercises the gist exclusion constraint on =tstzrange(valid_from, valid_to)=), unknown-id list, and filtering by =app_version_id=. All 25 assertions green locally.

** Follow-ups (deferred)
