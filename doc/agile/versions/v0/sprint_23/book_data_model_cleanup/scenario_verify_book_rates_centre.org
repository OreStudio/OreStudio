Round 1 (commit =85042d435=) FAILED: the field was nullable and no
seed data set it, so every book showed a blank Rates Centre — and
manually testing this also surfaced a real bug (fixed in a follow-up
commit): the codegen Insert Trigger Validations mechanism never
guarded validation on nullability, so *every* nullable-column
validation was silently unconditional. Both issues led to a design
change: =rates_centre_code= is now required (not nullable), defaulting
to =WRLD= like =book_status=/=regulatory_book_type=, and testdata now
assigns each seeded book a real rates centre by region. Re-run this
scenario against the new commit.
