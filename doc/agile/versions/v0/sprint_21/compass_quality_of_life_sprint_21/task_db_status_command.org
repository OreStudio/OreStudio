* Goal

Add =compass db status= as a standalone command that gives a complete
database health overview in one shot, without having to run =compass
bearings= and parse the environment block.

The command should surface:

- *Restore info*: when the database was last restored and from which
  schema version / git commit (already in =compass_db.database_info=).
- *Schema drift*: how far the DB schema lags behind HEAD, with the same
  ok/warn/stale colour coding used in bearings.
- *Bootstrap state*: whether =system.bootstrap_mode= is still =true=
  (i.e. the system provisioning wizard has not been run); if so, suggest
  =compass db setup= or the relevant provisioning step.
- *Tenants*: count and list of non-system tenants (tenant code, status,
  created date) so it is clear whether a real tenant has been provisioned.
- *Parties*: count of party records (a non-zero count indicates real data
  has been loaded beyond the system seed).
- *Accounts*: count of non-system accounts as a quick data-sanity signal.
- Anything else easily queryable that helps diagnose a cold-start or
  post-restore environment (e.g. currency count, market data series count).

Reuse =_psql= / =compass_db.database_info= from =compass_db.py=.  The
command should degrade gracefully when the DB is unreachable (print a
clear error and suggest =compass db recreate -y -k=).

* Result

Delivered on =feature/db-status-command=.

Added =cmd_db_status()= in =compass_db.py= and wired it as =compass db status=.

Also extracted =schema_drift(project_root, info)= as a shared helper so both
=cmd_db_status= and =cmd_bearings= in =compass.py= share the same ok/warn/stale
thresholds — eliminating the duplicate drift logic that previously lived in
both places.

The command shows:

- *Schema*: version, restore timestamp, git commit/date, drift with
  green/yellow/red colour coding matching the bearings UX.
- *Bootstrap mode*: reads =system.bootstrap_mode= from
  =ores_variability_system_settings_tbl=; shows green "false" when
  provisioned, yellow "true" with an actionable hint when not.
- *Tenants*: non-system tenants with status and created date.
- *Data counts*: parties, user accounts (=account_type = 'user'=),
  currencies — zero counts render yellow as a data-sanity signal.

Degrades gracefully when DB is unreachable: prints a red error and a
=compass services stop && compass db recreate -y -k= hint.
