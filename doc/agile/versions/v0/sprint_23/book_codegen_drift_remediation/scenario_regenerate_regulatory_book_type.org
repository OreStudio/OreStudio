** Log in and select Barclays Plc

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123=, select
=BARCLAYS PLC=. Confirm all menus enable with no errors.

** Regulatory Book Types list opens from Reference Data > Book Codes

Open Reference Data > Book Codes > Regulatory Book Types. Confirm
exactly two rows appear — Trading, Banking — each with Description and
Display Order populated, no error banner.

** Detail dialog opens and displays fields correctly

Double-click the "Trading" row. Confirm Code (read-only), Name,
Description, Display Order, Version, and Modified By all populate
correctly.

** Edit and save a regulatory book type

Change the "Trading" row's Description and Save. Confirm the change-
reason prompt appears (new: has_change_reason_cache was just added),
the save succeeds, the grid refreshes, and Version increments by 1.

** History dialog opens and shows the new version

Open History on "Trading". Confirm both versions appear with distinct
valid_from/valid_to timestamps, and that closing the history dialog
only closes the sub-window, not the whole app.

** No error toast or log error during the whole flow

Check the client's log tail
(=build/output/<preset>/publish/log/ores.qt.<colour>.log=) for any
ERROR-level line — in particular no "Server reported failure" message
(the new success-check added to ClientRegulatoryBookTypeModel.cpp).
