* Goal

Eliminate the two-path timestamp problem in the repository layer. Every entity
timestamp field (=valid_from=, =valid_to=, =created_at=, =started_at=, etc.) currently
uses =optional<db_timestamp>= or =db_timestamp=, which routes reads through
=strptime= and silently produces epoch on parse failure.

Replace all entity timestamp fields with =std::string=. sqlgen then reads the raw
PostgreSQL string (e.g. ="2026-06-03 10:57:32.015657+00"=). The mapper calls the
single =timestamp_to_timepoint(std::string_view)= overload which goes through
=from_iso8601_utc= and throws on any failure. The =db_timestamp= overload of
=timestamp_to_timepoint= and the =timepoint_to_timestamp= helper are removed — one
path, one function, throws on error.

Scope: ~60 entity files and ~160 mapper call sites across all services.

* Acceptance

- No entity file declares =db_timestamp= or =optional<db_timestamp>= for any timestamp field.
- =timestamp_to_timepoint= has exactly one overload (=std::string_view=).
- =timepoint_to_timestamp= is deleted from =mapper_helpers.hpp=.
- All mapper read sites call =timestamp_to_timepoint(v.field)= where =v.field= is =std::string=.
- All mapper write sites use =datetime::to_db_string(tp)= to assign timestamp fields.
- Full build green; no epoch timestamps in any history dialog.

* Decisions

- =std::string= is the correct entity field type because sqlgen reads the raw PostgreSQL
  string directly, bypassing =strptime=. The only parse function is =from_iso8601_utc=
  which handles =+00=, =+00:00=, and =Z= designators and throws on failure.
- =datetime::to_db_string(tp)= (format: ="YYYY-MM-DD HH:MM:SS"= without designator) is
  the correct write-side function; PostgreSQL accepts it for =TIMESTAMPTZ= columns when
  the session timezone is UTC.
- =timepoint_to_timestamp= is removed: it returned an empty =db_timestamp= on failure
  (silent epoch) instead of throwing, and is no longer needed once entities use =std::string=.

* Out of scope

- Changing the temporal SQL schema or trigger behaviour.
- Non-timestamp fields in entity structs.
- =db_timestamp= usage outside the repository/mapper layer.
