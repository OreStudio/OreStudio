* Steps

** Connect and log in

Start =ores.shell= (or =compass shell=), connect to the running NATS
instance, and log in as =tenant_admin@barclays_plc= /
=Secure-Password-123= (from =barclays_system_provision.ores=).

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Run the CRM rates recipe script

Run the generated =crm_rates.ores= script (=compass shell -u
tenant_admin@barclays_plc -p Secure-Password-123 -f
projects/ores.shell/scripts/library/marketdata/crm_rates.ores=), or
type the commands directly: =crm rates majors --party "BARCLAYS
PLC"=, then =crm rates exotics --party "BARCLAYS PLC"=, then =crm
rates scandies --party "BARCLAYS PLC"=.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |
| Notes  | Each named CRM printed its own table (CRM, Pair, Rate, Change, Status, As Of columns), scoped to just that CRM's pairs. |

** Run with no CRM name (all enabled CRMs)

=crm rates --party "BARCLAYS PLC"= (no positional).

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |
| Notes  | Returned the union of majors/exotics/scandies rows in one table, each correctly tagged in the CRM column with which CRM it came from. |

** Resolve --party by name instead of UUID

Confirm =--party "BARCLAYS PLC"= resolves correctly without needing
the party's UUID (same convention as =provision party <party>=).
