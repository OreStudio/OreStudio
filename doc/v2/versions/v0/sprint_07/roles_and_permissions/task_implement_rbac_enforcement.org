* Goal

Enforce role-based access at the message-handler layer: reject requests the calling session lacks permission for.

* Acceptance

- Each handler declares the permission(s) it requires.
- Dispatch checks the session's roles against the required permissions.
- Rejection returns a typed error message; no information leak.
- Tests cover both allow and deny paths.

* Plan

Captured during execution; cleared into the parent story on close.

* Notes

Permission declaration is per-handler so we can read the access surface by grepping handlers.

* Result

RBAC enforced everywhere a session-bound message lands.
