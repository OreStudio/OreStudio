* Goal

Add the two pieces of shell infrastructure the provisioning design rests on: a small parser for optional --flag arguments usable by any command (the cli library only hands us strings today), and stop-on-error semantics for the load command — abort at the first failed command reporting line number, command and error, with --continue-on-error restoring the old behaviour. Commands need a uniform way to signal failure to load.

- Commands can declare optional flags with defaults; unknown flags are an error.
- load aborts on first failure with line/command/error; --continue-on-error restores today's behaviour; a failed run never prints 'Script complete'.
- Unit tests cover flag parsing and both load modes.
