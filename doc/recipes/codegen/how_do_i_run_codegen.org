For background on what the generator is and what it generates, see
[[id:08FBD248-257A-A474-DD43-DB08949978F1][ores.codegen]] (the component model).

* Question

How do I run the ORE Studio C++ code generator?

* Answer

There are three entry points depending on scope:

** By entity name (preferred for single-entity work)

Resolve by name — no need to know the model file path or component:

#+begin_src sh
./compass.sh codegen entity generate <entity> --address <address>
#+end_src

Examples:

#+begin_src sh
./compass.sh codegen entity generate country --address ores.cpp.qt
./compass.sh codegen entity generate country --address ores.sql.schema
./compass.sh codegen entity generate country --address ores
#+end_src

Omit =--address= to generate the entity's full supported set — =ores=
is the graph root, so =--address ores= is equivalent.

Preview what would change without writing:

#+begin_src sh
./compass.sh codegen entity generate country --address ores.cpp.qt --diff
./compass.sh codegen entity generate country --address ores.cpp.qt --dry-run
#+end_src

See [[id:8010834F-9179-4ECD-999D-5D017D393D36][How do I generate a codegen entity?]] for the full workflow.

** By model file

Generate from a specific org model file:

#+begin_src sh
./compass.sh codegen generate --model projects/path/to/model.org --address <address>
#+end_src

** Across a whole component

Regenerate every entity in a component:

#+begin_src sh
./compass.sh codegen regenerate --component refdata-cpp --address ores.cpp
./compass.sh codegen regenerate --component refdata --address ores.sql.schema
#+end_src

* Script

All three entry points are implemented in
[[proj:projects/ores.compass/src/compass_codegen_entity.py][=projects/ores.compass/src/compass_codegen_entity.py=]] (entity-name
workflow) and =cmd_codegen= in
[[proj:projects/ores.compass/src/compass.py][=compass.py=]] (model-file and component-wide workflows).

* Tested by

Run =./compass.sh codegen entity generate country --address ores.cpp.qt --dry-run=
and verify twelve Qt file paths are printed without errors.
