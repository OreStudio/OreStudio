* Plan

The plan followed for this task:

1. Scaffold chapter folders =chapter_00= and =chapter_01= under
   =doc/manual/user_guide/chapters/=.
2. Write =introduction.org= with an alpha-warning quote block, prose
   covering: what ORE Studio is, who it is for (5 audience types), 5
   functional areas, what it is not, and how the manual is organised.
3. Rewrite =user_manual.org= as a root composition file: =#+include:=
   each chapter at =:minlevel 2=; chapter headings are level-1 (=*=).
4. Move the three existing screenshots from =chapter_01/= to
   =assets/images/=; update the reference in =user_manual.org= to use
   the canonical path =../../../assets/images/=.
5. Add a =#+caption:= for the main-window screenshot.
6. Update =doc/compass.org= =manual/= entry with an id-link to the
   manual root file.
7. Run =deploy_site= to verify no regressions; fix heading-level and
   description frontmatter issues surfaced by review.
8. Rebase on main (resolved conflict in =sprint.org= stories table),
   force-push, and merge.

* Notes

- Image path resolution was the main friction point: org-publish could
  not resolve cross-directory image links from included files. The fix
  was to always reference images from the root =user_manual.org= using
  =assets/images/= paths, consistent with =release_notes.org= precedent.
- Review identified two issues: incorrect =#+description:= (copy-pasted
  from agile template) and broken heading hierarchy (=** Introduction=
  with =:minlevel 1= created double-nesting). Both fixed before merge.
