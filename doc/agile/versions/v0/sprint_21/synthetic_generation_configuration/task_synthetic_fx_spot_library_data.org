* Goal

Deliver a "library data" bundle of synthetic FX spot generation configs covering
the 30 benchmark pairs listed below. A user should be able to open the Librarian,
select the bundle, press *Apply*, and immediately have all 30 pairs generating
synthetic ticks — without manually authoring configs or typing starting spots.

The 30 benchmark pairs are the minimum set from which the CRM can derive any
cross rate:

*G10 vs USD (10)*
EUR/USD, GBP/USD, USD/JPY, USD/CHF, AUD/USD, NZD/USD, USD/CAD, USD/SEK,
USD/NOK, USD/DKK

*Key EM vs USD (~20)*
- Asia: USD/CNH, USD/HKD, USD/SGD, USD/KRW, USD/TWD, USD/THB, USD/MYR,
  USD/IDR, USD/PHP, USD/INR
- LatAm: USD/BRL, USD/MXN, USD/CLP, USD/COP
- Africa/MENA: USD/ZAR, USD/NGN, USD/EGP, USD/TRY, USD/SAR, USD/AED
- CEEMEA: USD/PLN, USD/CZK, USD/HUF, USD/RON (optional stretch)

Each config must include a plausible GMM starting spot (mid price) consistent
with a single declared reference time-point =t= (e.g. 2024-01-02 London open).
All spots should be mutually consistent — if EUR/USD = 1.0950 and USD/JPY =
157.50, then EUR/JPY implied = 172.44, and that ratio should hold across the
full set.

** Open question: how to seed reference spots

Analyse the ORE sample data (shipped with the ORE distribution) to identify a
reference date =t= at which spot rates are already defined for the benchmark
pairs. Use those rates to anchor the starting spots for each config, keeping the
full set internally consistent. This avoids any dependency on external data
sources or HTTP client support in the Librarian.
