* Notes

The implementation went through several iterations to arrive at the final design:

- Initially registered in =org-roam-capture-templates= (wrong variable); corrected to =org-capture-templates=.
- The =(file FUNCTION)= target requires passing the function *symbol*, not a call form.
- Cancel was creating files in =inbox/= because =org-roam-db-autosync-mode= or Emacs auto-save was writing the buffer before abort. Tried a temp-file-in-=/tmp/= workaround (move to inbox on save, delete on abort), but this broke =ores/checkout-root= in the =:hook= since the temp buffer has no project context.
- Final design: build the full snippet body (with title substituted) inside =ores/capture--inbox-file= while still in project context; cache it in =ores/capture--snippet-body=; the =:hook= calls =yas-expand-snippet= with the pre-built body — no project-root lookup needed at hook time. Cancel cleanup uses =org-capture-prepare-finalize-hook= (marks buffer unmodified) and =org-capture-after-finalize-hook= (deletes file if auto-save wrote it).
