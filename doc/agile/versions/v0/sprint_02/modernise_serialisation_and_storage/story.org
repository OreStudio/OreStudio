* Goal

Replace the hand-written serialisation and storage layers with
generated ones, so adding new data types is cheap. Reflect-cpp covers
JSON and XML; sqlgen covers Postgres.

* Acceptance

- Reflect-cpp generates JSON serialisers for the existing types.
- XML parsing now goes through reflect-cpp.
- Postgres CRUD goes through sqlgen-generated code.
- Round-trip tests pass for all migrated types.

* Decisions

- /Reflect-cpp over Boost.PFR + manual/ :: reflect-cpp gives us
  bidirectional codecs for both JSON and XML in one library; the
  Boost.PFR approach would have needed a second library for XML.
- /Sqlgen over hand-rolled prepared statements/ :: keeps the schema
  shape and the C++ types in sync without manual marshalling.

* Out of scope

- Migration of the entire codebase to sqlgen — only the active types
  in v0 are converted.
- Performance benchmarking of the generated code.
