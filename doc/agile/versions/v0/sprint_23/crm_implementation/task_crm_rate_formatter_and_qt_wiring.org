* Goal

The Qt CRM matrix panel hand-formats rates itself
(=QString::number(rate, 'f', 5)=, fixed precision, no base/quote
canonical ordering) -- the same "logic in the UI" problem the
sibling task [[id:60B9E7A1-1CFC-4A33-A3AC-C36078D53FDB][Move CRM
matrix's numeric logic (inversion, deltas) out of the Qt client]]
already fixed for inversion and %-change. Formatting is the
remaining piece.

Add a stateless, plain C++ presentation overlay -- a new
=presentation= namespace/subfolder inside =ores.marketdata.client=
(thin, DB-free, matching its existing role) -- that maps a batch of
=crm_rate_item= (or =ores.analytics.quant::domain::crm_rate_view=)
plus a resolved =currency_pair_convention= into ready-to-bind
display strings (rate text, %-change text, tooltip text). The
formatter itself takes the convention as a plain parameter -- it has
no NATS/cache dependency of its own; the caller (the Qt window)
resolves the convention via
[[id:17CAD54D-0C46-427F-9117-E2B82346BAE5][ores.refdata.client's
currency_pair_convention_cache]] first. Qt's own cell widget already
accepts =QString=, not a raw double, at its display boundary -- see
=CrmCrossRatesMatrixMdiWindow::reload()= -- so this is a drop-in
replacement for that formatting call, not a new UI contract.

* Acceptance

- New =presentation= namespace/subfolder inside
  =ores.marketdata.client= (e.g.
  =ores.marketdata.client/presentation/crm_rate_formatter.hpp=)
  provides a stateless mapper from raw CRM rates + a resolved
  =currency_pair_convention= to display-ready strings; no NATS or
  Qt/Wt dependency in the mapper itself.
- =CrmCrossRatesMatrixMdiWindow= calls
  =currency_pair_convention_cache= then the formatter instead of
  formatting rates itself; the fixed ='f', 5= precision and lack of
  base/quote canonical ordering are replaced by convention-driven
  formatting (=pip_factor=/=tick_size=/=decimal_places=,
  =currency.base_precedence=).
- No new convention entity/type is introduced -- uses the existing
  =currency_pair_convention=/=currency= fields.
- Real Catch2 coverage of the formatter itself: precision rounding,
  base/quote reordering, inverted-pair display, missing/unavailable
  status, independent of any NATS/Qt harness.
