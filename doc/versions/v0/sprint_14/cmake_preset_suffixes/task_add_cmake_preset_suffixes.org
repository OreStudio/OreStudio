* Goal

Make every CMake preset name its build tool so multi-tool environments work cleanly.

* Acceptance

- Pattern: ={platform}-{compiler}-{buildtype}-{tool}=.
- Hidden =ninja= + =make= presets centralise generator settings (incl. disabling coloured Make output).
- Linux + macOS support both Ninja + Make; Windows on Ninja only.
- Automatic parallel-build level detection removed from =CMakeLists.txt= — deferred to user env vars or build-tool defaults.
- Documentation + skill files updated to the new naming.

* Plan

Captured during execution; cleared into the parent story on close.

* Notes

Make support became a recurring ask; explicit suffix keeps existing Ninja workflows but adds Make as a peer.

* Result

Preset naming explicit; both build tools supported.
