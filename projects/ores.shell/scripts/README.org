#+title: ores-shell script library
#+description: Runnable .ores scripts generated from the shell recipes.

This directory holds the ores-shell script library that the shell's
=load= command (and the Qt script panel) consume. The scripts are
*generated*, not hand-written: every =.ores= under [[file:library/][=library/=]] is tangled
from a shell recipe under =doc/recipes/shell/=, which is the single
source of truth.

* Layout

- =library/= — the generated =.ores= artefacts. Each carries a
  =# GENERATED from <recipe> — do not edit by hand= header and is
  committed so the shell and the Qt panel can load it directly. This is
  also what gets packaged (installed to =share/ores/shell-scripts=).
- There are no =.org= sources here any more: a recipe at
  =doc/recipes/shell/<name>.org= tangles to =library/<name>.ores=.

* Authoring

To add or change a script, edit (or create) its *recipe*, never the
=.ores=. See [[id:D4E65EF1-54F4-47E1-9388-58BE9E36FBE3][How do I create a recipe?]]. The recipe's =ores-shell= src
block is the script body; =load='s stop-on-error semantics mean an
error-free completion is meaningful, so order the commands so failures
abort early.

* Generating

The =tangle_shell_scripts= CMake target regenerates the whole library:

#+begin_src sh
cmake --build <build-dir> --target tangle_shell_scripts
#+end_src

It runs =projects/ores.lisp/src/ores-build-recipe-scripts.el=, which
tangles each recipe's =ores-shell= block into =library/<name>.ores=
(via =org-babel-tangle-file='s TARGET-FILE, with LANG-RE keeping the
=sh= runner blocks out) and prepends the generated header.

* Running

#+begin_src sh
ores.shell
ores-shell> load projects/ores.shell/scripts/library/barclays_system_provision.ores
#+end_src

or, with the connection inherited from the environment:

#+begin_src sh
compass shell -f barclays_system_provision.ores
#+end_src
