into docs/code/ci classes, conditional jobs per class (docs → site

check. Fold the roundtrip check (misnamed =ore_coverage.yml=) into
the code class; retire the standalone PR triggers the classes
replace.

* Result

pr.yml added as the single PR entry point: a classifier job buckets
the changed paths via the pulls/files API (ci: .github and codegen
templates; docs: doc, assets, .claude, .gemini, markdown, and
codegen tooling; everything else: code), conditional jobs per class
(site build for docs, canary plus roundtrip for code, everything for
ci), and an always-reporting pr-gate that fails unless every class
job succeeded or was skipped. canary-linux.yml converted to
workflow_call; build-site.yml and ore_coverage.yml lost their PR
triggers (push-to-main and dispatch remain); whole-workflow
concurrency coalesces pushes per PR. Gemini's config file deleted —
the app is uninstalled. All workflows yamllint-clean.

Two post-merge fixes landed on the same PR: =*.org= added to the
docs pattern (org-mode files outside =doc/= were falling through to
code=true and triggering the canary unnecessarily — acceptance
criterion was wrong, modeling files are docs not code); claude-review
gained =display_report: true= and =track_progress: true= so automatic
reviews are visible even when they produce no inline findings.
