* Plan

Implemented =ores-dashboard.el= as a =special-mode=-derived major mode
rather than using the =emacs-dashboard= framework, as the card/box layout
required full control over rendering.

Key design choices:

- *Rendering*: Custom box-drawing characters (┌─┐ ├─┤ └─┘ │) produce a
  3-column card grid. =ores/dashboard--group-cells= builds equal-height
  cell columns; =ores/dashboard--render-row= inserts them side by side.
  Text properties carry =ores/action= lambdas; =RET= and mouse-1 dispatch
  via =ores/dashboard--activate=.

- *Keyboard shortcuts*: =ores/dashboard--alloc-key= assigns =a=–=z= then
  =A=–=Z= globally across all items during render. Each action is wrapped
  in an interactive lambda and registered in a per-render sparse keymap
  =ores/dashboard--item-keys=. After render, =use-local-map= activates a
  =make-composed-keymap= of item keys over =ores/dashboard-mode-map= so
  =g= and =q= still work.

- *Env binding*: =default-directory= is set to checkout root on the buffer
  at open time. All action lambdas close over =root= and =env= captured
  during =ores/dashboard--render=.

- *Output display*: =ores/dashboard--display= calls =switch-to-buffer= +
  =delete-other-windows= so output always fills the full frame, matching
  the dashboard layout.

- *DB connections*: =ores-db/setup-connections= populates per-service
  connections; a postgres superuser entry is added pointing to
  =ores_dev_<label>=. =cl-letf= overrides =read-passwd= inside
  =sql-connect= so the alist password is used silently without prompting.
  =PGPASSWORD= is also injected into =process-environment= for the psql
  subprocess.

- *Project structure*: All =.el= sources moved to =projects/ores.lisp/src/=;
  =modeling/component_overview.org= written. =.build-site.el= path updated.

* Result

=projects/ores.lisp/src/ores-dashboard.el= implemented as a
=special-mode=-derived major mode with:

- 11 card groups: Environment, Database, Services, Compass, Build, Site,
  Skills, Bookmarks, Links, Shell, NATS.
- Global keyboard shortcuts =a=–=z=/=A=–=Z= assigned to every item;
  each label shows its key as a dimmed =(x)= suffix.
- Env-aware: binds to the checkout =.env= via =ores-env.el=; default-directory
  set to checkout root so all action lambdas resolve project-current correctly.
- Database card: full SQL connection picker with per-service PGPASSWORD
  injection; postgres superuser connects to =ores_dev_<label>=; Kill
  connections, Teardown DB, Recreate DB, Recreate environment items.
- NATS card: Init NATS and Init NATS + provision.
- Output always shown full-frame via =switch-to-buffer= + =delete-other-windows=.
- =ores.lisp= sources moved to =src/=; =modeling/component_overview.org= added.
- =.build-site.el= path updated to follow the =src/= move.
