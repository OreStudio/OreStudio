* Goal

The pair-code inline flag on =CurrencyPairDetailDialog='s Pair Code
field (and any other =has_key_flag_icon= entity's key field) stays
visibly tiny regardless of the =QSize= passed to =keyFlagIconSize()=,
unlike the same flags rendered via =QComboBox= (Base/Quote/Settlement
Currency) or =QTableView= (list grids), which correctly respect
=single_flag_icon_size()=/=currency_pair_icon_size()=.

Root mechanism: =set_line_edit_flag_icon()=
(=projects/ores.qt/api/include/ores.qt/FlagIconHelper.hpp=) adds the
icon via =QLineEdit::addAction()=, which Qt renders through an
internal =QToolButton= — reaching into it via
=findChild<QToolButton*>()->setIconSize(...)= is the documented
approach but had no visible effect in testing. A follow-up attempt
that also forced =setFixedSize()= and =updateGeometry()= on both the
button and the line edit *also* made no difference and was reverted
(too much complexity for an unconfirmed fix) — see
=set_line_edit_flag_icon()='s current comment.

Needs proper investigation: attach a debugger or add temporary
logging to confirm whether =findChild<QToolButton*>()= is even
finding the right widget (timing — is it created synchronously by
=addAction()=, or lazily on first paint?), and whether the style in
use has its own icon-size override that's winning over the widget
property.
