* Goal

Add =compass db tenants= so that developers can discover tenant UUIDs
with a single compass command, without knowing the DB schema or having
to write SQL.

The command should:
1. Source =.env= (for =ORES_CLI_DB_*= and =ORES_CLI_*= credentials).
2. Call =ores.cli iam tenants list --format json=.
3. Format and print a table of =code= / =name= / =tenant_id=.
4. Alongside each row print the copy-pasteable export line:
   =export ORES_TENANT=<tenant_id>=

Expected output:
#+begin_example
Provisioned tenants:

  code      name                    tenant_id
  --------  ----------------------  ------------------------------------
  barclays  Barclays Bank PLC       36d4c3c9-195c-46b5-a4d8-b36cb48c8f46
  acme      Acme Corporation        ...

To target a tenant, export its ID:
  export ORES_TENANT=36d4c3c9-195c-46b5-a4d8-b36cb48c8f46
#+end_example

* Acceptance

- =compass db tenants= lists all active tenants with =code=, =name=,
  and =tenant_id= (UUID).
- Each row is accompanied by a =export ORES_TENANT=<uuid>= hint.
- The command sources =.env= automatically (no manual =source .env=
  required).
- A recipe =How do I list tenants?= (or =How do I set ORES_TENANT?=)
  documents the compass variant.
- The command is wired into =compass db --help= output.

* Plan

1. Add =tenants= subcommand to the =cmd_db()= handler in =compass.py=.
2. Source =.env=; call =ores.cli iam tenants list --format json=.
3. Parse JSON output and format as the table shown in Goal above.
4. Print the =export ORES_TENANT=<uuid>= hint for each row.
5. Add the recipe and update =compass db --help= epilog.
