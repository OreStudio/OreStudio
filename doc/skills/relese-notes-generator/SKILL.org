* When to use this skill

When closing a sprint — i.e. assembling release notes for the
sprint's PRs, tagging =main=, and opening a draft GitHub release.

* How to use this skill

1. /Confirm the sprint to close/. The most recently-completed
   sprint folder under =doc/agile/versions/v0/= is the target; ask
   the user if there is any ambiguity.

2. /Run/ [[id:9f274ee4-3cab-47fd-bb6e-449d8b789d8c][How do I generate release notes?]] end-to-end. The recipe
   covers branch prep, PR data collection, reading the sprint,
   drafting the notes against the template, committing, tagging,
   and the =gh release create --draft= step.

3. /Hand off to a human/. The release is always created in /draft/
   so a maintainer can review, attach binaries/screenshots, and
   publish.

The release is the upward signal that work landed (see [[id:fabb50e5-518c-42d6-901e-848644e56ca2][Release]] in
the glossary). Cutting it is a [[id:7d72a9da-984f-4130-bff6-97b66b233907][Sprint closure]] task — System 3
owns it at sprint level.
