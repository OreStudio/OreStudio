* Goal

The CLI grew up alongside the project's initial layout and no longer
fits it. Restructure the CLI to follow the new project shape and add
tests to the parser so future changes are caught.

* Acceptance

- CLI sources live under the new layout (=ores.cli=).
- CLI commands resolve through the new module structure.
- CLI parser has a Catch2 test suite covering the main flag and
  command paths.

* Decisions

- /Test the parser, not the commands/ :: command execution touches
  the full stack; the parser is the part with the most surface area
  for regression and the cheapest to test.

* Out of scope
