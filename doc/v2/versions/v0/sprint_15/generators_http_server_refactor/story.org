* Goal

Close the api/core migration by relocating two cross-cutting
concerns into the right modules: synthetic generators (api) and
domain-specific HTTP route handlers (core).

* Acceptance

- Generators moved to api layers across trading + reporting + iam + refdata + compute.
- auth_session_service relocated from iam.core to iam.api.
- HTTP route handlers moved into domain core modules.
- ores.http.server retains generic server infrastructure only.
- Generator directory standardised to generators/ (plural).
- faker-cxx added as private dep on affected api libs.
