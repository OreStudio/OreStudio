* Goal

There is no =task move=: relocating a task between stories means
reverting the old story's wiring, re-running =compass add= with
=--id= into the new parent, deleting the old file, and hand-editing
tables. A =compass task move <slug> --story <target>= should rehome
the file, rewire both stories' =* Tasks= tables, and update the
task's parent links — preserving the task's UUID throughout.
