** Summary

Full implementation of the workspace entity as a first-class domain object with bitemporal history, a complete NATS service layer, IAM permissions, Qt UI, and archive/delete lifecycle operations.

*** Schema (Phase 1)
- Workspace table rewritten as bitemporal (UUID PK, =valid_from=/=valid_to=, full audit columns)
- =workspace_id uuid= added to all 50+ data tables across refdata, trading, marketdata, and reporting schemas as a soft FK (no hard FK constraint — partial unique indexes are not FK-targetable in PostgreSQL)
- Live workspace sentinel UUID (=aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa=) added as utility constant
- =ores_workspace_resolution_order_fn= recursive CTE function for ancestor chain resolution
- =ores_workspace_trade_scope_tbl= for per-workspace trade whitelists
- Badge system: generic =archived= badge (replaces FSM-specific =fsm_archived=)
- SQL codegen template updated: =workspace_id= generated as soft FK with correct default

*** Service layer (Phase 2)
- =ores.workspace.api=: protocol structs for list, create, archive, remove, resolve, set/clear trade scope
- =ores.workspace.core=: entity/mapper pattern (isolates =row[N]= indexing), repository, service, NATS handler, registrar
- IAM permissions: =workspaces:read=, =write=, =archive=, =archive_any=, =live_workspace:archive=, =delete=, =delete_any=
- Three-tier ownership-aware permission checks on archive and delete handlers
- Remove (soft-delete) endpoint: closes bitemporal record + clears trade scope

*** UI (Phases 3 & 4)
- =WorkspaceContext= propagated via MDI area property to all windows
- =WorkspaceMdiWindow=: paginated table, reload/add/edit/archive/delete toolbar actions
- =WorkspaceDetailDialog=: create/edit/archive form with provenance widget
- =ClientWorkspaceModel=: Qt item model backed by NATS list endpoint

*** Icons
- =Icon::Archive= added to icon system with Fluent UI (20px regular/filled) and Solar (Linear/Bold) variants
- Archive toolbar action uses =Icon::Archive=; delete uses existing =Icon::Delete=
