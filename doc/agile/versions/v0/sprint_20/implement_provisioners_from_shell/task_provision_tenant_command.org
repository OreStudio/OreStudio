* Goal

Add 'provision tenant': logged in as the tenant admin of a bootstrap-mode tenant, execute the wizard's four phases — publish the selected bundle (--bundle, default first available; --root-lei adds lei_parties params) and wait on its workflow; for --source synthetic run synthetic.v1.organisation.generate with the knob flags; associate the admin with all Operational parties (iam.v1.account-parties.save, non-fatal like the wizard); clear system.bootstrap_mode and complete provisioning. GLEIF is the default source with --dataset-size large|small.

* Acceptance

- Defaults reproduce the wizard's GLEIF path; --source synthetic --seed N reproduces the synthetic path deterministically.
- Workflow progress is streamed; phase failures abort with the plumbing command to resume; association failure is non-fatal, finalize failure is fatal.
- Cancelling/aborting never clears the bootstrap flag (divergence from the wizard, by design).

* Plan

1. Extend =nats_client::login_info= with =account_id= (set at login
   and auto-login) — phase 3 needs the logged-in account's id, which
   the session previously discarded.
2. Refactor =synthetic_commands= to expose =generate_flag_specs()=,
   =build_generate_request()= and =generate()= so the porcelain
   embeds generation without duplicating the 15 knobs.
3. =provision tenant= in =provision_commands=: validate flags up
   front (source, root-lei applicability, timeout, knobs); resolve
   the bundle (=--bundle= or first available, the wizard's default);
   publish + =wait_for_instance=; synthetic generation when selected;
   associate the admin with Operational parties (non-fatal, feedback
   reset so scripts do not abort); clear =system.bootstrap_mode=
   (warn-only) and complete provisioning (fatal).
