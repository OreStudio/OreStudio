* Notes

** Pg_notify channel rename (2026-06-07)

The session began by establishing the =ores_{component}_{entity_plural}= naming
convention for PostgreSQL LISTEN/NOTIFY channels (documented in
[[id:81AB065B-E6EB-4CC2-BB18-F3FD2652EC47][postgresql_architecture.org]]). All 72 trigger files and the C++ =application.cpp=
registrations were updated accordingly (commit =4d082ca9a=).

** Template bug: validate function missing from drop script (2026-06-07)

=sql_schema_domain_entity_drop.mustache= did not drop the
=ores_{component}_validate_{entity_singular}_fn= functions created by the companion
=sql_schema_create.mustache= (table model). Fixed by introducing the
=<<paste:5E47F108-1350-4540-B3C2-E83DD5379B2D>>= block kind ("SQL: extra drop
statements") and wiring seven entity models (rounding_type, monetary_nature,
currency_market_tier, contact_type, party_id_scheme, party_status, party_type)
to declare their validate function drops. =_substitute_paste_markers= in
=generator.py= was fixed to collapse the marker line when no entity implements
the kind (commit =56855dff9=).

** Rounding_type notify trigger (2026-06-07)

=rounding_type= did not previously have a PostgreSQL notify trigger. Since it is
now a full domain entity model, codegen generates
=refdata_rounding_types_notify_trigger_create.sql= (channel:
=ores_refdata_rounding_types=). Accepted as correct. C++ subscription not wired
yet — tracked in the parent story.

** Zero-drift result

After the fixes, =codegen regenerate --component refdata --profile sql= produces
zero diff for: =refdata_currencies_create.sql=, =refdata_currencies_drop.sql=,
=refdata_currencies_notify_trigger_create.sql=,

=refdata_monetary_natures_create.sql=, =refdata_monetary_natures_drop.sql=,
=refdata_rounding_types_create.sql=.

Non-currency files with remaining codegen drift (books FK validation removal,
parties sequence drop, etc.) are out of scope for this task and deferred.
