## Summary

Stands up the v2 information architecture under =doc/v2/= and migrates the first
half of v0 (sprints 01-07) into it. Roughly the midway point of the v0 sprint
backfill — sprints 08-16 will follow in a separate PR.

### v2 architecture

- New tree organised by Beer's Viable System Model:
  =doc/v2/{meta,versions,components,recipes,knowledge,functions,skills,product_identity}=.
- Meta-documents define the contract every v2 doc follows
  (=meta/document_types.org=, =compass.org=, =glossary.org=,
  =cybernetic_levels.org=, =lifecycle.org=, =execution_model.org=).
- Composition tree: version → sprint → story → task (one PR per task).
- State carried as org-mode TODO keyword on the document's =* Status=
  headline; folders encode composition, not state.
- All cross-document references are org-roam =[[id:UUID][text]]= links —
  no file-path links anywhere in v2.
- Cross-sprint stories use =#+predecessor= / =#+successor= keywords plus
  prose "Continued from/in:" lines for bidirectional audit.

### Codegen

- New =projects/ores.codegen/src/v2_doc_generate.py= with shell wrapper
  =generate_v2_doc.sh= scaffolds every v2 doc type from Mustache
  templates.
- Auto-detects parent =:ID:= and =#+title:= from the parent doc;
  derives ancestor tags from the path; supports interactive prompts on
  TTY, CLI flags otherwise.
- Doc types covered: =task=, =story=, =sprint=, =version=,
  =component=, =recipe=, =knowledge=, =skill=.

backlog (goal, acceptance, notes, result); stories carry cross-doc
links to components, sibling stories, and (where relevant) predecessor
sprints.
