* Result

Abandoned before implementation got far. The idea assumed a "fleet"
worth cross-checking for pending scenarios per worktree/task, but this
project doesn't have proper fleet coordination for that yet — no
shared source of truth for which worktree is doing what beyond
=compass fleet='s own best-effort snapshot, and no mechanism to keep a
per-worktree scenario list meaningfully in sync across 10 independent
checkouts. Building a UI on top of that would create the appearance of
fleet coordination without the substance, likely misleading the tester
rather than helping them. Revisit if/when the project actually builds
real fleet coordination — this task's goal would then be a thin
consumer of that, not a first attempt at inventing it.
