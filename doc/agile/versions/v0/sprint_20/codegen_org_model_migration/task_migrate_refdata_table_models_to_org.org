* Goal

Convert the 10 =*_table.json= bi-temporal reference-data table
models (book_status, contact_type, currency_market_tier,
currency, monetary_nature, party_id_scheme, party_status,
party_type, purpose_type, rounding_type) into literate
=ores.refdata.<entity>_table.org= files under
[[proj:projects/ores.refdata/modeling][=projects/ores.refdata/modeling/=]]. Extend the codegen pipeline
with a new =ores.codegen.table= kind: dedicated
=load_org_table_model= producing the =sql_schema_create.mustache=
shape, filename-suffix dispatch on =_table.org=, and exclusion
from the generic entity match so routing is unambiguous.

* Acceptance

- =migrate_table_to_org.py= one-shot converter committed.
- 10 =ores.refdata.<entity>_table.org= files under
  [[proj:projects/ores.refdata/modeling][=projects/ores.refdata/modeling/=]]; 10 JSON files removed.
- =org_loader.load_org_table_model= produces a model dict
  exactly matching the JSON shape consumed by
  =sql_schema_create.mustache= (verified end-to-end:
  =json.load(... .json) == load_org_table_model(... .org)= for
  all 10).
- =is_table_model= accepts the new =_table.org= suffix;
  =is_domain_entity_model= excludes the same suffix.
- =_CODEGEN_ORG_TYPES= accepts =ores.codegen.table= so the
  per-component =modeling/= discovery picks it up.
- Both =generator.load_model= and =codegen.generate._generate_single=
  dispatch on the new suffix to the table loader.
- =codegen.sh generate --model X_table.org --profile sql=
  produces byte-identical output to the JSON path for all 10
  tables.

* Plan

Org file shape:

- Frontmatter scalars: =product=, =schema=, =component=,
  =entity_singular=, =entity_plural=, =has_tenant_id=,
  =coding_scheme=, =image_id=.
- Body: free-form description.
- =* Primary key= drawer (=column=, =type=, =is_text=).
- =* Columns= sub-headings (one per column) with drawer
  (=type=, =nullable=, optional =default=).
- =* Validation function= drawer (=tenant_scope=, optional
  =default=, =order_by=).
- =* Insert trigger= → =** Validations= holds an org table with
  =column= / =validation_function= rows.
- =* Check constraints= sub-headings with =:expression:= drawer
  (only present when JSON has the key — three tables out of ten
  have none, one has a single constraint).
- =* Indexes= sub-headings when present (none currently).
