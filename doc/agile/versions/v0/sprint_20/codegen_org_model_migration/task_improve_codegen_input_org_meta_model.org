* Goal

Define specific, well-typed org-mode document types for each codegen input file. Review MASD stereotypes and apply them where they clarify structure. Document the schema for each type so it can be validated, scaffolded, and understood by the meta-model layer.

* Acceptance

- Each codegen input org file (facet_catalogue.org, component_catalogue.org, modeline.org) has a specific #+type that maps to a documented schema — not the generic codegen_config bucket.
- The schema for each type is documented: required sections, heading stereotypes, property drawer keys, and table columns.
- MASD stereotypes used in org files are reviewed against the MASD concept library; any that clarify structure without adding noise are applied consistently.
- The three type names are chosen in line with the meta-model naming conventions used elsewhere in the codebase (ores.codegen.* or appropriate namespace).
