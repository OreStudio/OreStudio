* Summary

The assets facet groups the non-code artefacts =ores.codegen= projects
for a component: Qt Designer =.ui= forms, the PlantUML ER diagram, and
shell environment-variable snippets. It is generated under =--address
ores.cpp.qt= from the model, so these supporting files track the logical
model alongside the C++, SQL and build facets rather than drifting.

* Detail

The assets facet is the catch-all for technical-space artefacts that
are neither source code nor build scripts but are still derivable from
the model:

- /Qt Designer UI forms/ — =.ui= XML for the generated Qt windows and
  dialogs, consumed by =uic=.
- /PlantUML ER diagram/ — a parse-then-render asset: a stage reads the
  generated SQL DDL and emits a =.puml= entity-relationship diagram.
- /Shell environment variables/ — the variable snippets the shell and
  service tooling source.

As with every facet, read the template for the authoritative output
rather than reproducing it here.
