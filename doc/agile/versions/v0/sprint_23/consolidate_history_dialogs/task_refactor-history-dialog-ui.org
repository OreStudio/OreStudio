This page documents a [[id:31C868B9-306E-4282-BA8C-07E647021B34][task]] in the [[id:037B474D-84ED-4888-9054-D58AB1FB10D2][Consolidate history dialogs onto HistoryDialogBase]] story. It captures the goal, current status, acceptance, and any notes or results.

* Goal

Redesign =HistoryDialogBase= from a flat database-log view into an
interactive, "Git-style" version comparison tool, per
[[id:34D92C7A-4714-47CF-852C-0B236F6E1161][Gemini Review: History Dialog UI/UX]]:

- A master-detail split (~30/70) replacing the dead right-hand space:
  a compressed version timeline on the left, a live side-by-side diff
  panel on the right.
- Semantic diff colouring and inline word-level diffing, so a changed
  field's delta is visible without re-reading the whole value.
- Multi-version selection ("Compare Selected"), so users can diff any
  two versions directly, not just adjacent ones.
- Visual timeline polish: version "pill" badges and a connecting line
  between entries.

* Acceptance

- =HistoryDialogBase= shows a master-detail layout: version timeline
  on the left, live diff panel on the right — no dead space, no
  separate =Changes= / =Full Details= tab row.
- Selecting a version renders its diff against the previous version
  immediately in the right-hand panel, with changed fields
  colour-highlighted and only the changed span of each value
  called out.
- Selecting two versions (via multi-select) computes and displays
  the diff between exactly those two, not just adjacent versions.
- Existing history dialog behaviour (reload, open selected, revert
  to version) is preserved.

* Plan

(Implementation strategy. Written when work starts; key decisions
are distilled into the parent story's =* Decisions= at close, but the
plan itself stays — it is the historical record of what we did.)

Redesigned currency's =HistoryDialog= (the only dialog wired for
compare mode) against the Gemini mockup, iterating through several
rounds of manual QA feedback:

- Master-detail layout: =QSplitter= with a left ~25-35% "History
  Timeline" panel (=QListWidget= with card-per-version item widgets,
  no grid) and a right diff pane. Layout stretch factors on the main
  =QVBoxLayout= eliminate dead space above the splitter.
- Live diff pane rendered as one =QTextBrowser= HTML document (2
  columns: Field, Value Diff — old value stacked above new value in
  one cell) instead of a =QTableWidget=; this was the root-cause fix
  for a persistent multiline-value truncation bug that two prior
  =QTimer::singleShot= row-resize patches failed to fully solve.
- Two explicit =compareFromCombo=/=compareToCombo= selectors replace
  an earlier ctrl-click multi-select design (user feedback: fragile,
  and a combo box makes "any two versions" trivial); validated so
  "From" must be strictly older than "To", with equal-version compare
  handled directly (show all fields / nothing per the toggle) rather
  than as an error.
- Segmented =QPushButton=/=QButtonGroup= "All Fields | Only Changes"
  toggle — the same pill idiom as the synthetic market data
  generator's Simple/Advanced toggle — replacing an initial
  =QCheckBox= that user feedback flagged as non-idiomatic.
- Provenance fields (Modified By, Performed By, Change Reason Code,
  Change Commentary) moved out of the diff table and into the
  timeline cards as labelled, colour-coded lines (git-log-style:
  commit metadata beside the timeline, diff pane shows only content
  changes) — field names centralized as
  =ores::history::domain::provenance_fields= constants in
  =ores.history.api= so the mapper and the dialog share one
  definition.
- Timeline card polish: "History Timeline" header, hover tooltip
  showing the full ISO =Recorded At= timestamp, distinct label/value
  colours per field.
