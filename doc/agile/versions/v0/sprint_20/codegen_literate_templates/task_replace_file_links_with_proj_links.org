* Goal

No org document links a repo file via file: when the target is not published as HTML. All such links — including images where inline rendering is not required — use proj: links so they resolve to GitHub URLs on the published site. Where an image must render inline on the site, verify how the site build publishes it and keep the working form; document the rule (file: for published org/image targets, proj: for everything else) in the documentation conventions.
