* Goal

=compass env create <name> [--type light|full]= provisions a new named worktree from the genesis clone, assigns isolated ports, writes a skeleton =.env=, and optionally runs =env configure=. =env_init.py= is updated so that re-running =env configure= on any environment is idempotent and safe for adjective-noun names (hyphen-safe DB names, ports preserved from the skeleton).

* Plan

Two-file change:

1. *New =env_create.py=*: implements =compass env create=. Contains an embedded adjective-noun wordlist, port scanner (scans sibling =OreStudio.*/.env= files for used =ORES_BASE_PORT= / =ORES_NATS_PORT= values), and worktree creation via =git worktree add=. Writes a skeleton =.env= then optionally calls =env_init.run()= with the new worktree as =project_root=.

2. *Updated =env_init.py=*: moves =_read_env()= call to before port/db derivation so that =ORES_ENV_NAME=, =ORES_BASE_PORT=, =ORES_NATS_PORT= written by =env create= are honoured on the subsequent =env configure= run. Computes =label_lower= from =env_name= (not raw =label=) so adjective-noun names produce valid PostgreSQL identifiers. Writes =ORES_ENV_NAME= and =ORES_BASE_PORT= to the =.env= template.

3. *Wired in =compass.py=*: =env create= subcommand added to =cmd_env=.

* Result

Shipped in PR [[https://github.com/OreStudio/OreStudio/pull/1283][#1283]] (merged 2026-06-16).

- New =compass env provision <name> [--type light|full]= command: creates a
  named sibling worktree off =origin/main= using the GCP-style adjective-noun
  convention (auto-generated from an embedded wordlist when no name is given),
  scans sibling =.env= files for used ports, assigns isolated
  =ORES_BASE_PORT= / =ORES_NATS_PORT=, writes a skeleton =.env=, and optionally
  runs =env configure=. Companion =env deprovision= removes a worktree safely.
- =env_init.py= made idempotent: reads =ORES_ENV_NAME= / =ORES_BASE_PORT= /
  =ORES_NATS_PORT= / =ORES_ENV_TYPE= from an existing =.env= before deriving
  values, so re-running =env configure= preserves them. DB identifiers now use
  =label_lower= (underscored), fixing the hyphen-in-identifier bug for
  adjective-noun names.
- All six review findings addressed (five accepted and fixed in 08516a3 /
  029e15e; one declined as unreachable).

All acceptance criteria met. The command was named =provision= rather than the
provisional =create= used in early acceptance notes.
