#+title: How do I ask an LLM to spec a new feature?
#+description: Reusable prompt that drives an interview-style feature specification into the agile backlog.
#+type: recipe
#+version: 2
#+level: cross
#+filetags: :recipe:llm:prompt:agile:spec:
#+created: 2026-05-21
#+updated: 2026-05-21

This prompt requires *Claude Code* — it uses the =AskUserQuestion=
tool which is only available inside Claude Code sessions. It is not
suitable for generic LLM chat interfaces.

* Question

I have a rough idea for a new feature. How do I use Claude Code to
turn it into a detailed story (or set of stories) in the agile backlog?

* Answer

Replace =<story summary here>= with a one-paragraph description of
the idea, then send the prompt to Claude Code in the repo root.
The result lands in the [[id:d4219199-7b17-4a89-b4bc-6019738642da][product backlog]] as one or more
[[id:671f18e4-e09c-4b3b-bd24-d33df8ae38a6][captures]] (if not yet sprint-ready) or directly as
[[id:699a8d45-b67e-4d3e-9206-24fa17c51ada][stories]] in the current sprint.

#+begin_example
Consider this idea for a new feature:

<story summary here>

Interview me in detail using the AskUserQuestion tool about literally
anything: technical implementation, UI & UX, concerns, tradeoffs,
etc. Make sure the questions are not obvious.

Be very in-depth and continue interviewing me continually until it's
complete, then write a detailed user story with a specification to the
current agile backlog as per the [[id:2df8adaf-20c9-4757-9399-798eec6df828][Agile Product Owner]] skill. If you think the work is
too large for a single story, break it into several stories, each of
which will have its own PR.
#+end_example

* Script

Use with Claude Code in the repo root. The [[id:2df8adaf-20c9-4757-9399-798eec6df828][agile-product-owner]] skill
handles the backlog write.

* Tested by

Manual usage with Claude Code.
