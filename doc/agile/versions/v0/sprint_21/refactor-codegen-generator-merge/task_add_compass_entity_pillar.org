* Goal

Extend =compass codegen= with an =entity= sub-namespace that treats entities
as first-class objects. Today the only handles are model file paths and
component names; these sub-commands surface entity-level operations so a
developer never needs to know where files live or which profiles apply.

** Model type taxonomy

Codegen recognises nine model types, split into two tiers:

| Tier | Types | Description |
|------+-------+-------------|
| Entity | =domain_entity=, =junction= | Full domain objects with SQL + C++ + Qt artefacts |
| Meta-entity | =enum=, =field_group=, =table=, =schema=, =data=, =component=, =service_registry= | Structural/support models |

=list= defaults to the Entity tier. Meta-entity operations use the same
sub-commands but are filtered or grouped separately so the two tiers are
never mixed in default output.

** Sub-commands

1. *entity list* — enumerate entities grouped by model type; default output
   shows Entity tier only; =--all= adds Meta-entity tier in a separate section;
   =--type <type>= filters to one specific model type. Columns: name, org-roam
   ID, component, model path.
2. *entity add <name>* — scaffold a new entity model file via the existing
   doc-generate machinery and wire it into the component.
3. *entity generate <name-or-id>* — resolve the entity from the index and
   regenerate all applicable codegen profiles. =--profile= restricts to one
   layer; =--dry-run= prints paths without writing.
4. *entity show <name-or-id>* — list every file codegen produces for the
   entity with a staleness indicator (file mtime vs template mtime).
5. *entity diff <name-or-id>* — unified git diff of all generated files for
   the entity; designed to be run immediately after =entity generate=.

All sub-commands resolve the entity by name or org-roam UUID prefix.
All call Python functions directly; no subprocess to =codegen.sh= or bash
wrappers except =git diff= as the canonical diff backend.

* Result

Implemented =compass_codegen_entity.py= (~345 lines), dispatched from
=cmd_codegen()= in =compass.py= when the first argument is =entity=.

Four sub-commands shipped:

- *list* — discovers all model files across all components via
  =manifest.all_components()= + =manifest.discover_models()=; groups by
  metatype; defaults to the Entity tier (=domain_entity=, =junction=); =--all=
  adds a Meta-entity section; =--type <metatype>= filters to one type.
- *generate* (alias =gen=) — resolves the entity by name or org-roam UUID
  prefix; runs =_generate_single()= from =codegen.generate= for every
  applicable leaf facet (profiles with =templates= key, not =includes=);
  accepts =--profile= (single facet) and =--dry-run=.
- *show* — lists all output files for the entity with a staleness indicator
  (output mtime vs template mtime: =ok= / =STALE= / =MISSING=).
- *diff* — runs =git diff= over all generated files for the entity;
  designed to run immediately after =entity generate=.

Resolution deduplicates by model path (same file discovered under multiple
components counts once) and prefers primary metatypes when a name matches
both =domain_entity= and a meta-entity type. Output path deduplication via a
=seen= set prevents the same file appearing twice when multiple profiles
produce the same output (e.g. =domain= and =non-temporal-domain=).

The =add= sub-command was deferred — it requires doc-generate scaffolding
not yet available via the Python API. All other acceptance criteria met.
