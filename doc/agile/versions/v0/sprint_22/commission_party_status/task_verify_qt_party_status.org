* Result

Verified party_status Qt UI end-to-end post-NATS migration: list,
detail round-trip (create/edit), history, delete, and cross-session
eventing all pass. Two regressions found and fixed inline:

- History window's close button was not wired to
  =connect_dialog_close=, so it did nothing.
- party_status had no NATS event mapping/subscription in
  =ores.refdata.service=, so changes never propagated to other
  sessions; added registration and publish subscription for
  =party_status_changed_event=.
