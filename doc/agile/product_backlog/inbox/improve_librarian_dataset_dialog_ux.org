#+title: Improve UX of Librarian dataset detail dialogs
#+description: The Data Librarian's dataset detail dialogs (Overview, Provenance, Methodology, Dependencies tabs) are hard to read and make sense of - a rigid single-column property-sheet table layout with no visual hierarchy, per a Gemini UI/UX review.

This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *inbox* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

Redesign the Data Librarian's dataset detail view away from its
current single-column, alternating-row property-sheet table (the same
layout across the Overview, Provenance, Methodology, and Dependencies
tabs) toward a layout that actually uses widescreen real estate and
gives the reader visual hierarchy — a persistent identity header
(name/code/ID, currently duplicated per-tab), a primary/sidebar split
(rich text + methodology on one side, classification/governance/audit
metadata chips on the other), and interactive elements (copyable IDs,
clickable URLs) instead of flat text. Full audit, anti-pattern list,
and a proposed mockup are in [[id:211CB159-F633-4897-A899-56CD91625396][OreStudio Data Librarian UI/UX Review]].

* Why

Every DQ-published dataset (currency_pairs, fx_spot_configs, the new
FX driver rates dataset, ...) is inspected through this same dialog,
so the friction compounds across the whole Librarian surface as more
datasets get added. Flagged directly: "it is very hard to read and
make sense of it."
