Database setup is very noisy. Key offenders:

- Per-row =NOTICE: Created change reason category: system= (and similar) for every seeded row.
- Per-permission =NOTICE: Created permission: iam::accounts:create= — dozens of lines.
- TimescaleDB detection output.

Fix:
- Replace per-row raise with a single post-block summary: =NOTICE: Seeded N change reason categories.=
- Wrap permission seeding similarly.
- Suppress or redirect TimescaleDB internal notices using =SET client_min_messages = WARNING;= around the relevant sections.
