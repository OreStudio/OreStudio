* Goal

Pull the messaging-type definitions and the framing/transport
helpers out of both client and server code into a shared =comms=
library. Prevents the messaging surface from leaking into domain
code on either side.

* Acceptance

- A =comms= library target exists in CMake.
- Both server and client link against it; messaging types removed
  from their private headers.
- A basic message round-trip test covers the library.

* Notes

- The library boundary is the messaging-types layer, not transport
  internals — cobalt-specific code stays in the server.

* Result

- =ores.comms= library committed; client and server consume it.
