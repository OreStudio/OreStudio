* Summary

A standard =ores.refdata= entity spans eleven layers: DB, domain, repository, service,
messaging (JSON IO), Qt UI, shell REPL, CLI, HTTP routes, Wt web UI, and manual. Each
layer has a fixed set of criteria that determines whether the entity is fully commissioned.
The checklist below lists those criteria with an "Applies" column indicating whether the
criterion applies to every entity (Always), most entities (Usually), or only specific ones
(If applicable). Use this document alongside the [[id:A3F8C2D1-7E4B-4F9A-B2C5-8D1E6A3F0B7C][Entity Coverage Matrix]] when
commissioning a new entity or auditing an existing one. The currency entity was used as
the reference specimen to derive these criteria.

* Detail

** DB layer

| Criterion | Applies | Notes |
|-----------+---------+-------|
| Table exists in =refdata_<entity>_create.sql= | Always | |
| =tenant_id= column present (FK soft-ref to tenants) | Usually | Omit only for truly global records (e.g. currencies, countries) |
| =version= column present for optimistic locking | Always | |
| =modified_by= column present | Always | |
| =performed_by= column present | Always | |
| =change_reason_code= column present | Always | |
| =change_commentary= column present | Always | |
| =valid_from= and =valid_to= columns present (temporal history) | Always | No separate history table; same table with GIST exclusion |
| GIST exclusion constraint prevents temporal overlaps | Always | Requires =btree_gist= extension |
| Unique constraint or index on natural key | Always | e.g. code + tenant_id + valid_from |
| FKs or soft-references to related entities documented | Always | List expected soft-FKs in Result section |
| =party_id= soft-reference present | If applicable | Only for party-scoped entities |
| =workspace_id= soft-reference present | If applicable | Not applicable to tenant-global reference data |
| Appropriate non-temporal indexes (code, tenant_id) | Usually | |

** Domain layer (C++)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| Domain struct in =ores.refdata.api/domain/<entity>.hpp= | Always | |
| All DB columns represented as struct fields | Always | |
| Fields use canonical types (=std::string=, =bool=, =std::optional=, etc.) | Always | |
| =valid_from= / =valid_to= as =boost::posix_time::ptime= or equivalent | Always | |
| No logic in domain struct (plain data) | Always | |

** Repository layer (C++)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| Entity class in =repository/<entity>_entity.hpp= | Always | Holds domain + ODB metadata |
| Mapper class in =repository/<entity>_mapper.hpp= | Always | DB row ↔ domain struct |
| Repository class in =repository/<entity>_repository.hpp= | Always | |
| Repository supports: list, count, save, delete, get by ID | Always | |
| Repository supports: get_history (list all versions) | Always | |
| Repository supports: list_for_party | If applicable | Only for party-scoped entities |
| Repository uses tenant-scoped queries where applicable | Usually | |

** Service layer (C++)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| Service class in =service/<entity>_service.hpp= | Always | |
| Service exposes: list, count, save (upsert), delete, get, get_history | Always | |
| Service exposes: list_for_party | If applicable | Only for party-scoped entities |
| Service performs authorization checks | Always | |
| Service fires NATS events on mutation (save, delete) | Always | Post-NATS migration requirement |

** Messaging / JSON IO layer (C++)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| JSON struct in =json/<entity>_json.hpp= | Always | rfl-based; mirrors domain struct |
| IO handler in =json/<entity>_json_io.hpp= | Always | Converts domain ↔ JSON |
| NATS message types defined: =save_<entity>=, =delete_<entity>=, =<entity>_saved=, =<entity>_deleted= | Always | |
| List / count / get / history NATS message types defined | Always | |
| JSON round-trip tested (or verifiable via NATS) | Always | |

** Qt layer

| Criterion | Applies | Notes |
|-----------+---------+-------|
| MDI list window: =<Entity>MdiWindow= exists | Always | Shows list of records in MDI |
| Detail dialog: =<Entity>DetailDialog= exists | Always | Edit / view a single record |
| History dialog: =<Entity>HistoryDialog= exists | Always | Shows temporal history for a record |
| Controller: =<Entity>Controller= wires windows to service | Always | |
| Client model: =Client<Entity>Model= for Qt model/view | Always | |
| Import dialog: =Import<Entity>Dialog= | If applicable | For bulk-import entities |
| List window loads and displays records post-NATS | Always | Verify manually |
| Detail dialog: edit + save round-trips without error | Always | Verify manually |
| History dialog shows correct history | Always | Verify manually |
| Delete command removes record; history is preserved | Always | Verify manually |
| NATS eventing: change in one session appears in second session | Always | Verify with two connected clients |

** Shell layer (=ores.shell= interactive REPL)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| Submenu registered for entity | Always | |
| =list= command implemented and working | Always | |
| =add= / =save= command implemented and working | Always | |
| =remove= / =delete= command implemented and working | Always | |
| =history= command implemented and working | Always | |
| Commands use correct NATS-based service post-migration | Always | |

** CLI layer (=ores.cli= non-interactive)

| Criterion | Applies | Notes |
|-----------+---------+-------|
| =add_<entity>_options.hpp= config struct exists | Always | |
| =list= subcommand implemented | Always | |
| =add= / =save= subcommand implemented | Always | |
| =remove= / =delete= subcommand implemented | Always | |
| Commands use correct NATS-based service post-migration | Always | |

| Criterion | Applies | Notes |
|-----------+---------+-------|
| GET list route exists | Always | |
| POST save route exists | Always | |
| DELETE (batch) route exists | Always | |
| GET history route exists | Always | |
| Routes registered in routes file | Always | |
