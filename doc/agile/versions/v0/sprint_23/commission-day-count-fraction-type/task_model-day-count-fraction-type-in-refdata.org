* Plan

Followed the =business_day_convention_type= move precedent (commit
195fed986) end to end:

1. Scaffold the entity model via =compass add entity_org= under
   =projects/ores.refdata/modeling/=, filled in from the
   =business_day_convention_type.org= template (aux-type shape).
2. Generate all layers with =compass codegen entity generate <uuid-prefix>=
   (disambiguated by UUID since both the old trading model and new
   refdata model shared the entity name during the transition). This
   produced SQL create/drop/notify scripts, domain/repository/service/
   protocol/generator C++, the NATS event_registrar, and — since no Qt
   layer pre-existed for this entity (unlike business_day_convention_type)
   — the full Qt Controller/MdiWindow/DetailDialog/HistoryDialog set too.
3. Wired the new entity into =ores.refdata.core='s NATS =registrar.cpp=,
   the service's generated =event_registrar.cpp= (LISTEN/NOTIFY-to-NATS
   relay), the SQL create/drop manifests, a hand-written populate script
   (seeded from the old trading value set), and
   =ores_iam_provision_tenant_fn= (copy into new tenants at provisioning).
4. Checked whether any =ores.refdata= =*_convention= tables validated
   =day_count_fraction= against a function — none did (plain unvalidated
   text columns), so no repointing was needed there.
5. Removed the =ores.trading= original in full: SQL create/drop/notify/
   populate scripts and manifest entries, the .org model, all generated
   C++ (domain/repository/service/generator/protocol), the hand-written
   NATS wiring in =instrument_ref_handler.hpp=/=registrar_trades.cpp=,
   and the hand-written CLI wiring (options, parser, =application.cpp=
   handlers, =entity.hpp= enum). Repointed =trading_swap_legs='s
   =day_count_fraction_code= validation at the new =ores.refdata=
   function.
6. Found =ores.qt/refdata/RefdataPlugin.{hpp,cpp}= already had
   speculative forward-declarations/wiring for a
   =DayCountFractionTypeController= — fixed one latent bug: the
   constructor call was missing the =change_reason_cache= argument that
   the generated controller (since the model sets
   =has_change_reason_cache: true=) actually requires.
7. Validated: =projects/ores.sql/utility/validate_schemas.sh= (added an
   =RLS_001= ignore-list entry for the new lookup table, matching
   =business_day_convention_types='s existing entry) and
   =projects/ores.codegen/validate_docs.sh= (remaining warnings are
   pre-existing repo-wide drift, filed separately as capture
   =9297CB27-F58B-4AF7-9F56-5E077778773A=). Built
   =ores.refdata.core/api/service=, =ores.trading.core/api=,
   =ores.cli.exe=, and =ores.qt.refdata.lib= clean with no warnings.
