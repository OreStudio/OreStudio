Add a =tenants= entity to the =ores.cli iam= domain so that:

=code=, =name=, and =tenant_id= (the UUID developers need for
=ORES_TENANT=).

* Notes

Tenant table: =ores_iam_tenants_tbl=

Relevant columns:
| Column      | Type | Notes                                            |
|-------------+------+--------------------------------------------------|
| =tenant_id= | uuid | The UUID required for =ORES_TENANT= / =--tenant= |
| =code=      | text | Short code, e.g. =barclays=, =acme=             |
| =name=      | text | Display name                                     |
| =valid_to=  | timestamptz | =ores_utility_infinity_timestamp_fn()= = active |

Active-tenant SQL (confirmed working):
#+begin_src sql
SELECT code, name, tenant_id
FROM ores_iam_tenants_tbl
WHERE valid_to = ores_utility_infinity_timestamp_fn()
ORDER BY code;
#+end_src
