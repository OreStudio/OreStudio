#+title: Always use compass to create agile artefacts
#+description: Never write task/story org files manually — always scaffold with compass add.
#+type: memory
#+memory_subtype: feedback

#+filetags: :memory:feedback:

#+updated: 2026-05-25

Always use =compass add= (or the appropriate compass subcommand) to scaffold
new agile artefact files — tasks, stories, sprints, epics. Never write them
manually using the Write tool or generate_v2_doc.sh.

*Why:* Compass handles UUID generation, file placement, correct frontmatter,
and wiring into the project graph automatically. Manual creation (with Write
or generate_v2_doc.sh) was explicitly corrected by the user during sprint 18
dashboard work (2026-05-25). The generate_v2_doc.sh codegen recipe remains
correct for knowledge, recipe, and memory files — compass is specifically for
agile artefacts.

*How to apply:* Whenever a new task or story needs to be created, run
=compass.sh add task "Title" --parent-dir <path/to/story>= or the equivalent
compass command for the artefact type. Only fall back to Write/codegen if
compass fails or the artefact type is not supported by compass.
