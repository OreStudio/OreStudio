* Question

How do I run the ORE Studio C++ code generator?

* Answer

1. /Set up the venv/ — once per checkout:

   #+begin_src sh
   cd projects/ores.codegen
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   #+end_src

2. /Run the generator/ on a JSON model:

   #+begin_src sh
   ./run_generator.sh models/path/to/model.json
   #+end_src

   Or with a profile (set of templates for one facet, e.g. =qt=, =sql=):

   #+begin_src sh
   ./run_generator.sh models/path/to/model.json --profile qt
   #+end_src

   Or with a single template:

   #+begin_src sh
   ./run_generator.sh models/path/to/model.json --template cpp_protocol.hpp.mustache
   #+end_src

3. /Inspect the output/ — by default written to =output/= alongside the
   script. Override with a second positional argument:

   #+begin_src sh
   ./run_generator.sh models/path/to/model.json custom_output/
   #+end_src

4. /List available profiles/ when you need to discover them:

   #+begin_src sh
   ./run_generator.sh --list-profiles
   #+end_src

* Script

The wrapper itself is =projects/ores.codegen/run_generator.sh=. It
sources the venv and invokes =src/generator.py= with the given
arguments.

* Tested by

The generator's own test suite (under =projects/ores.codegen/=) and
its use in the project build: when models change, regenerated output
is committed and CI builds the result.
