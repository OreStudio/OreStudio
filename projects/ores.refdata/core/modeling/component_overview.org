* Summary

=ores.refdata.core= manages the reference-data domain in ORE Studio: over 150
domain types covering currencies, countries, counterparties, books, business
units, business centres, and trading structure entities. It provides temporal
ORM repositories, domain services, NATS message handlers in the 0x3000–0x3FFF
range, CSV export for selected types, and synthetic-data generators. Reference
data is consumed by all other domain components (trading, assets, IAM) and
drives UI drop-downs and validation.

* Inputs

- NATS request messages from Qt clients and peer services (query/create/update
  for currencies, counterparties, books, etc.) in the 0x3000–0x3FFF range.
- PostgreSQL connections to =ores_refdata_*= tables with temporal versioning.
- DQ population events for bulk reference-data loads.

* Outputs

- Reference-data records persisted to the =ores_refdata= schema with full
  temporal history (=valid_from= / =valid_to=).
- NATS response messages returned to callers.
- CSV exports of selected entity types (e.g., currencies).

* Entry points

- =include/ores.refdata.core/ores.refdata.core.hpp= — aggregate include.
- =include/ores.refdata.core/messaging/registrar.hpp= — registers all NATS
  handlers with the service host, including a currency history_provider
  (pilot) on [[id:9562F9CA-DCEB-44C6-8DB1-F3FC56D7703A][ores.history]]'s generic =history.v1.get= subject —
  see =src/messaging/registrar.cpp= for the wiring.
- =include/ores.refdata.core/service/= — per-entity service headers.

* Dependencies

- =ores.refdata.api= — shared domain types and NATS protocol schemas.
- =ores.dq= — ORM base classes and data-quality infrastructure.
- =ores.iam.core= — identity and authorisation context.
- [[id:9562F9CA-DCEB-44C6-8DB1-F3FC56D7703A][ores.history]] — generic entity-history dispatch: registrar.cpp

- =fort= — formatted table output.
- =soci= — SQL ORM for PostgreSQL persistence.
- =nats.c= — NATS messaging client.
