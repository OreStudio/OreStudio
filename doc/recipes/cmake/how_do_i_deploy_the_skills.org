* Question

How do I build the Claude Code skills bundle locally?

* Answer

The =deploy_skills= CMake target invokes Emacs in batch mode against
[[proj:projects/ores.lisp/src/ores-build-skills.el][=projects/ores.lisp/src/ores-build-skills.el=]], which exports each skill under [[proj:doc/llm/skills][=doc/llm/skills/=]] to
the format Claude Code expects:

#+begin_src sh :results verbatim
cmake --build --preset linux-clang-debug-ninja --target deploy_skills
#+end_src

Output lands under =build/output/= in the skills directory configured
by =CMakeLists.txt= (=ORES_SKILLS_DIRECTORY=).

** Prerequisites

- Emacs on PATH.
- The project must be configured (=cmake --preset ...=).

* Script

[[proj:projects/ores.lisp/src/ores-build-skills.el][=projects/ores.lisp/src/ores-build-skills.el=]] is the Emacs script the target invokes. The CMake
target definition lives in =CMakeLists.txt= near
=add_custom_target(deploy_skills ...)=.

* Tested by

No dedicated CI workflow yet — the target is exercised by manual
deployment when a skill is updated.
