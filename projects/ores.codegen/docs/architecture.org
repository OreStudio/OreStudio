* Summary

This document records how =ores.codegen= is built and how its parts
fit together: where its code and data live, how the generator picks
templates for models, how it handles modelines and licence headers,
and the conventions it enforces.

For /what/ the component is and /how to operate it/, see
[[id:08FBD248-257A-A474-DD43-DB08949978F1][ores.codegen]] (the component model) and the recipes it links to.

** Internal modules

| File                                  | Purpose                                                                  |
|---------------------------------------+--------------------------------------------------------------------------|
| =src/generator.py=                    | Main generator: JSON models + Mustache templates → SQL / C++ / etc.      |
| =src/doc_generate.py=              | v2 information-architecture document generator (task/story/sprint/...)   |
| =src/fpml_parser.py=                  | FPML Genericode XML → JSON models                                        |
| =src/iso_generate_metadata_sql.py=    | ISO standards → SQL                                                      |
| =src/images_generate_sql.py=          | Image artefacts (flags, crypto icons) → SQL                              |
| =src/lei_extract_subset.py=           | LEI dataset subset extractor                                             |
| =library/data/=                       | Static data files (licences, modelines)                                  |
| =library/templates/=                  | Mustache templates                                                       |
| =models/=                             | JSON model files                                                         |
| =output/=                             | Default destination for generated files                                  |

** Main generator functions

In =src/generator.py=:

- =load_data(data_dir)= — load JSON and text files from a data directory.
- =format_comment_block(text, lang)= — format text as a language-specific
  comment block.
- =generate_license_with_header(license_text, modeline_info, lang)= —
  build a licence header with modeline and copyright.
- =render_template(template_path, data)= — render a Mustache template
  with the given data.
- =get_template_mappings()= — the model-filename → template mapping.
- =generate_from_model()= — orchestrate generation for one model.

** Template system

- Mustache via the =pystache= library.
- Templates live in =library/templates/=.
- Output is SQL, C++, or org-mode depending on the template family.

** Data files

- =library/data/licence-GPL-v3.txt= — full GPL v3 licence text used in
  generated headers.
- =library/data/modeline.json= — per-language editor modeline strings.

** Features

- /Licence generation/. Generated files carry a licence header with
  editor modelines, a copyright with the current year, and the
  appropriate per-language comment formatting.
- /Multi-language comment support/. SQL and C++ use =/* ... */= with a
  =* = line prefix; Python uses ="""=, JavaScript uses =/** */=.
- /Flexible output/. Default output directory is =output/=; overridable
  via the second positional argument; created automatically if absent.
- /Overall models/. A =model.json= can orchestrate generation of
  multiple artefacts in dependency order.
- /Dynamic prefixing/. A =model_name= property on an overall model
  prefixes every output file (for example =solvaris_=).
- /Automatic sibling loading/. JSON models in the same directory are
  loaded together so a template can cross-reference them.
- /Enhanced data context/. Subject-area datasets (such as
  =currencies_dataset=, =countries_dataset=) are surfaced as named
  variables to templates for direct access.

** Example model structure

From =models/slovaris/catalogs.json=:

#+begin_src json
[
    {
        "name": "Slovaris",
        "description": "Imaginary world to test all system functions.",
        "owner": "Testing Team"
    }
]
#+end_src

The =sql_catalog_populate.mustache= template generates SQL that:

1. Includes the enhanced licence header.
2. Sets the schema to =ores=.
3. Generates SQL calls to =metadata.upsert_dq_catalogs()=.
4. Includes summary queries.

** Extending

To add a new model-template pair:

1. Add the JSON model to the =models/= directory.
2. Add the Mustache template to =library/templates/=.
3. Update the mapping in =src/generator.py=
   in =get_template_mappings()=.
