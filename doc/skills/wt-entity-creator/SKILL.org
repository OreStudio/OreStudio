* When to use this skill

When you need to add a Wt web UI for a new entity in =ores.wt.service=.
Prerequisites: the domain type must exist ([[id:b1450696-8c51-4cc5-8910-84912a411ab6][domain-type-creator]]) and the
service layer for the entity must be implemented. See [[id:608675e1-4284-4902-99ac-4da565390afb][Entity lifecycle]]
for the full layer ordering.

* How to use this skill

1. /Gather requirements/: entity name, service class, display fields,
   editable fields, primary key.

2. /Phase 1 — list widget/ (PR: =[wt] Add {Entity} list widget=):
   create ={entity}_row= struct and ={entity}_list_widget= class with
   toolbar, table, and signals.

3. /Phase 2 — detail dialog/ (PR: =[wt] Add {Entity} dialog=):
   create ={entity}_dialog= with form editors and =save_requested= signal.

4. /Phase 3 — application integration/ (PR: =[wt] Integrate {Entity} into application=):
   add page/controller, connect signals, register in navigation.

Use [[id:6DAFFC2F-7F94-403B-87D0-2A56C4669B7F][feature-branch-manager]] between phases and [[id:545D6B43-17FC-0234-FF2B-AB6565695616][pr-manager]] to open each PR.
Build after every step: =cmake --build --preset linux-clang-debug-ninja --target ores.wt.service.lib=.
