* Goal

ORE example data is naturally grouped (an example has Input,
ExpectedOutput, etc). Introduce a top-level container that
represents this grouping in the project so a user can load/save an
example as a unit.

* Acceptance

- A container type exists representing an ORE example.
- The container can be loaded from disk and persisted to disk.
- A test exercises loading a sample container.

* Notes

- Carried over from sprint 01 where it was POSTPONED. The v0 record
  has design notes about naming (environment / workspace / sandbox)
  and the inner-container choice (namespace / package).
- The chosen name was =workspace= for the outer container.

* Result

- Containing structure committed; ORE examples loadable as units.
