* Goal

Absorb =scripts/sprint_metrics.py= into compass as =compass sprint charts=
— a new subcommand under the existing =sprint= pillar. The command
generates the CSV data files that feed the sprint's gnuplot charts
(=sprint_activity.csv=, =sprint_progress.csv=, =pr_cycle_times.csv=)
and is invoked in place of the standalone script wherever it is called
today. The standalone script is deleted on hard cutover.

* Acceptance

- =compass sprint charts= is registered as a subcommand of =cmd_sprint=
  in =compass.py= and appears in =compass sprint --help=.
- Interface: =compass sprint charts [--sprint N] [--start-date YYYY-MM-DD]
  [--end-date YYYY-MM-DD] [--output-dir PATH]=. Without =--sprint= the
  command auto-detects the current sprint number from =compass where=
  state (the same =_locate_sprint= lookup the rest of the sprint pillar
  uses).
- Output: =build/output/sprint_NN/sprint_activity.csv=,
  =sprint_progress.csv=, =pr_cycle_times.csv= — identical to what the
  standalone script produces today.
- =scripts/sprint_metrics.py= is deleted and all callers (the sprint
  health review recipe, any CI or Makefile references) are updated to
  call =compass sprint charts=.
- The "How do I update the sprint charts?" recipe (or the sprint health
  review recipe, whichever documents this command) is updated to use the
  new invocation.
