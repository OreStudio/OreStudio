* Goal

Regression coverage in =ores.iam.core= proving the new subscription
actually works: publishing =ores.refdata.party_changed= for a given
tenant causes =party_cache= to reload that tenant's partition, and
=lookup_party=/=compute_visible_party_ids= reflect the updated data
without any restart. Without this, the previous silent-staleness bug
could regress unnoticed the same way it went unnoticed originally.
