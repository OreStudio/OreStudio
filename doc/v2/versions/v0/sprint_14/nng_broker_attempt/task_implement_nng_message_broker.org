* Goal

Stand up an NNG-based broker with JWT-in-frame so services can self-register and clients can be routed independently.

* Acceptance

- Stand-alone =ores.mq.broker= executable on NNG.
- JWT tokens embedded in message frames (new =jwt_size= field); CRC includes JWT bytes; protocol 50.0.
- Broker protocol messages: =register_service_request= / =_response=, =token_refresh_request= / =_response=.
- =auth_session_service= refactored for JWT validation with =jwt_validator_fn= + secondary =session_id_index_=.
- =ores.comms.service= self-registers handled message-type ranges at startup.
- =nng_service_runner= moves broker registration + message processing into a separate thread.
- Asynchronous dispatch via =boost::asio::co_spawn=.
- Minimal error response frames generated when NNG message parsing fails.
