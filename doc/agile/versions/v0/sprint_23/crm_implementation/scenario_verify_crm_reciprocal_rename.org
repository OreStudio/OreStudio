** Connect to tenant Barclays Plc

Log in against the =clever_dijkstra= environment as
=tenant_admin@barclays_plc= / =Secure-Password-123= and select
*BARCLAYS PLC*. If the DB has been recreated since Barclays was last
provisioned, re-run
=compass shell -f projects/ores.shell/scripts/library/provisioning/barclays_system_provision.ores=
first.

** Open the CRM Cross Rates Matrix

Market Data menu > Cross Rates Matrix (or the MDI window if already
open). Pick a CRM with at least one pair lacking a direct quote in one
direction, so the reciprocal-backfill path is exercised (e.g. one of
the majors/scandies/exotics CRMs used in the shell recipes).

** Confirm the toggle reads "Show Reciprocal"

The footer button that used to read "Show Inverted" now reads *Show
Reciprocal*; hover it and confirm the tooltip text also says
"reciprocal" rather than "inverted".

** Toggle "Show Reciprocal" on and check the backfilled cells

With the toggle off, note a cell with no direct quote shows a dash.
Turn "Show Reciprocal" on and confirm that cell now shows a
pastel-tinted computed 1/rate value instead of a dash, and that its
sparkline/history still populates correctly.

** Toggle "Show Reciprocal" off and confirm the cell reverts

Turn the toggle back off; the previously-backfilled cell should
return to showing a dash (no direct quote), confirming the toggle
still round-trips correctly.

** Export CSV with "Show Reciprocal" on and check the header

With the toggle on, use the matrix's CSV export. Open the exported
file and confirm the reciprocal cells are present in the export, and
that the filename/metadata reflects "reciprocal" rather than
"inverted" (matching the renamed filter state).
