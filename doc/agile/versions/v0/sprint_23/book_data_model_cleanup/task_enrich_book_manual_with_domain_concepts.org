* Result

Added a PlantUML book lifecycle diagram (multi-party opening workflow,
live-book loop, flat-check closing gate) to both
doc/knowledge/domain/book_lifecycle.org and the Books manual chapter,
with a departments-involved section (Desk/Controller/Finance/Market
Risk/Operations) and the FRTB trading/banking classification expanded
into full prose. Turned the previously bare book-fields list into
contextual paragraphs.

Dogfooding the new screenshot-capture test_scenario surfaced three real
bugs, all fixed: regulatory_book_type was never copied down to new
tenants during provisioning; book_status_repository was stale and
leaked rows across tenants (missing the tenant_id filter every sibling
lookup entity has); BookStatus's Qt stack was still living under
ores.qt/trading with a hidden-visibility crash from a missing
has_export_macro. PR review then found a fourth: book_status was also
missing has_pagination, so its list model silently never sent
offset/limit to the server.

Also found and killed ~47 orphaned service processes for this
environment (3 full generations never reaped by `services stop`),
which were both exhausting the shared Postgres connection pool and
serving stale pre-rebuild responses over NATS during debugging.
