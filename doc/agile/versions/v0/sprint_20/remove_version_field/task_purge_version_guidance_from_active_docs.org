* Goal

Remove every instruction that tells a developer or LLM to add =#+version: 2= or
to run the audit script. These are the live source of the contradiction the
user hit. Known sites:

- =doc/orientation.org= (~line 117): "Every new =.org= file must carry
  =#+version: 2=. The audit script =doc_version_audit.py --check= gates this."
  — delete the bullet.
- =doc/llm/memory/memory.org= (~line 83): "always =#+version: 2= for docs".
- =doc/llm/memory/version_field_is_schema_version.org= — *delete the whole
  memory*. Its premise (the field is a kept schema version) is exactly the
  decision this story reverses.
- =doc/meta/document_types.org= — remove the =* Version tagging= section.
- Recipes: =how_do_i_create_a_recipe.org=, =how_do_i_create_a_memory.org=,
  =how_do_i_open_a_new_sprint.org=, =how_do_i_create_a_new_doc.org= (the
  "including =#+version: 2=" clause) — strip the version/audit references.
- =doc/recipes/documentation/how_do_i_audit_doc_versions.org= — retire (delete)
  the recipe; the thing it documents is gone.
- Skills: =doc/llm/skills/skill-review/SKILL.org= (lists =#+version: 2= as
  required frontmatter), =doc/llm/skills/doc-sync-recipes/SKILL.org=.

Grep for =#+version= and =doc_version_audit= across =doc/= at the end; the only
survivors should be historical sprint-17/18 records describing the migration.
