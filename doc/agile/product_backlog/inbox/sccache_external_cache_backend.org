#+description: sccache's cache is on GitHub's built-in actions/cache, capped at a fixed, non-purchasable 10 GB per repository shared across every OS/compiler/buildtype variant (linux-gcc/clang, windows-clang, macos-clang x debug/release) plus vcpkg/Qt caches. Confirmed via live experiment (sprint 22, story canary_ctest_ignores_cancellation): a PR with essentially zero source diff got a 0.00% sccache hit rate because the Linux-gcc-debug cache had been evicted entirely by GitHub's LRU policy to make room for other platforms' caches touched more recently by continuous-*.yml runs. Recommended fix: move sccache to an external cache backend it natively supports (S3/GCS/Azure Blob, or a self-hosted Redis/webdav endpoint) so the cache is bounded by provisioned storage, not GitHub's shared 10GB quota, and isn't subject to cross-platform eviction or the 7-day-unused cleanup. Needs bucket/credentials/IAM setup and workflow changes across every platform's workflow file (canary-linux.yml, continuous-linux.yml, continuous-macos.yml, continuous-windows.yml) — a real infra project, not a quick fix.
