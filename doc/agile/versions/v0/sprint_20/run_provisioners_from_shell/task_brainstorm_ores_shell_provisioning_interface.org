* Goal

Using the wizard-to-shell coverage mapping from the analysis task, brainstorm what provisioning should feel like in the medium of ores-shell: command vocabulary and grammar (verbs, submenus, flags vs positional args), sensible defaults that remove inputs the wizards must ask for, where one command should span several wizard pages, error reporting and progress for long-running steps, and how scripts (.ores via load) shape the design — including what the end-to-end provisioning script would read like. Produce concrete candidate command surfaces and weigh them; this is a design exercise, not implementation.
