* Result

New manual chapter /Provisioning from the Shell/
(=chapter_shell_provisioning.org=), included in =user_manual.org=
directly after /Initial Setup/ so the wizard and shell paths sit side
by side. Coverage: the shell-and-provisioning model, the three
=provision= commands with their wizard parallels, defaults and
options, the supporting-commands table, =.ores= script semantics
(stop-on-error, =--continue-on-error=) and the script library, and
=provision_all= as the complete worked example.

Per product direction the ores-shell blocks are executable org-babel:
each stage is a named =ores-shell= block with a =sh= runner
(=:noweb yes :results output :wrap example :exports results=) feeding
=compass shell= via heredoc — the recipe-library pattern, modernised
from the recipes' =./ores.shell ${log_args}= to =compass shell= so
the connection resolves from =.env=. Every =#+RESULTS:= block holds
output from the live verified run (2026-06-07), so re-execution
refreshes the captured output in place. PDF build verified
(=deploy_manual=).
