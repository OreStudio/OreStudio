* Acceptance

- =compass search <story_folder_name>= (underscores, e.g.
  =user_manual_pdf_build=) finds the story without manual
  underscore-to-space rewriting.
- The story itself ranks first, ahead of its tasks and other docs
  that merely carry the filetag.
- Commands that resolve stories by slug (e.g. =capture promote
  --story=) disambiguate same-slug folders across sprints, or report
  the candidates instead of a bare failure.

* Plan

Two changes in =compass.py=:

1. *=cmd_search= — folder-slug query expansion and re-ranking.*
   =re.findall(r"\w+", query)= treats underscores as word characters,
   so =user_manual_pdf_build= tokenises to one FTS5 term and the
   unicode61 tokeniser turns it into a phrase, killing recall. Add a
   =_is_folder_slug= predicate (all =[a-z0-9_]=, contains =_=); when
   true, split on =_= and build an OR query from the parts. After
   deduplication, boost results whose =file_path= sits inside
   =/<slug>/=, with =story.org= ranked above task files within that
   folder.

2. *=resolve_story= — current-sprint preference for slug ambiguity.*
   When a folder-slug resolves to multiple candidates (same slug in
   different sprints), prefer the story in the current sprint — the
   same approach =_cmd_task_start= already uses for tasks. Report
   candidates if still ambiguous after that.
