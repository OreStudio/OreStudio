* Goal

Collapse =create_currency_request/response= and
=update_currency_request/response= into a single =save_currency=
message pair. Bitemporal repositories always insert new versions, so
the create/update distinction was never semantically necessary —
removing it simplifies the protocol and the UI.

* Acceptance

- Protocol exposes only =save_currency_request/response= (0x1009/0x100A).
- =update_currency_request/response= removed.
- Qt =CurrencyDetailDialog= sends one message type regardless of add/edit mode.
- Protocol version bumped to signal the breaking change.
