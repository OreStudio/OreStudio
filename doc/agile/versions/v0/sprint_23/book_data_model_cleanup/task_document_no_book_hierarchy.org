* Goal

State explicitly, in the Book domain knowledge docs and the manual,
that ORE Studio does not model an accounting-side book-to-book
hierarchy: =book= has no self-referencing parent field, and all
hierarchy is expressed through the portfolio tree instead. This is a
design choice, not an open gap.
