* Goal

The knowledge graph can be filtered to show only LLM-related documents
(memories, skills, runbooks, settings) by entering =llm= in the existing
org-roam-ui tag allowlist. The graph export correctly populates the ~tags~
field from node properties, and all documents under =doc/llm/= carry the
=:llm:= filetag. The compiled graph output lives in =external/= rather than
=projects/=, and carries no local patches.

* Decisions

- Tag data is in =properties.ALLTAGS=; the fix is in the exporter, not the graph UI.
- Compiled output stays in =external/= without a Node build step in CI.
- No patches on compiled binaries — the reverted image-path patch belongs in the upstream TypeScript source when images are next needed.
- A dedicated preset button in the fork is not worth the upstream complexity; a single tag entry covers the use case.

* Out of scope

- A fully separate LLM dashboard page (deferred; tag-filtered graph is sufficient).
- Adding a Node/npm build step to the site CI pipeline.
- Upstream image-path fix in =mcraveiro/org-roam-ui= (deferred to when image rendering in static mode is actually needed).
