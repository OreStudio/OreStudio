* Result

Abandoned the manual-chapter half of this task on user direction:
=contact_type= is an auxiliary/lookup type (code/name/description/
display_order), not a primary domain entity, and the
[[id:897A7156-504D-4581-AD4C-026D643C0838][Domain entity evaluation checklist]] is explicit that aux types don't
get their own manual chapter — they get a subsection in the manual
chapter of the primary entity that consumes them ("Aux type documented
in parent chapter | ... | Subsection in parent entity chapter"). For
=contact_type= that consuming entity is conceptually "Contact" /
"Contact Information" (party and counterparty contact records), which
has no manual chapter of its own yet — writing a standalone
=contact_type= chapter would have been documenting the lookup without
the thing it looks up into, which reads wrong to a user. Filed a
capture ([[id:EC281267-A651-4995-B2F8-AF0023E1923A][Document Contact/Contact Information with contact_type as a
subsection]]) to write the real parent chapter later, with
=contact_type= folded in as its "valid values" subsection at that
point, per the checklist's own convention — rather than doing it out
of order here.

The backlog-captures half of this task (shell, CLI, Wt, HTTP support)
is unaffected by this and split off into
[[id:2AFB3102-B9AB-452C-AEF5-7EFECA039BA0][File shell/CLI/Wt/HTTP backlog captures for contact_type]] instead of
being finished here, so this task's abandonment reason stays legible
in isolation.
