* When to use this skill

When the user asks to create a new Claude Code skill, port an
existing v1 skill into the v2 thin-index shape, or rebuild the
shipped bundle so the skill changes propagate to Claude Code.

* How to use this skill

1. /Confirm the work is skill-shaped/. A skill is /task-shaped/ — a
   thin index over recipes plus a reference doc, not the recipes
   themselves. If the doc describes /what someone does/ across
   many tasks, it is a [[id:66f2a9e8-0385-4d56-b4b1-0957208990dc][function]], not a skill. See
   [[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document types]]§=skill= for the contract.

2. /Identify the recipes the skill will index/. Each operational
   how-to belongs in its own =how_do_i_<thing>.org= file. If a
   needed recipe doesn't exist yet, create it first via
   [[id:d4e65ef1-54f4-47e1-9388-58be9e36fbe3][How do I create a recipe?]].

3. /Scaffold the skill/ via the codegen. Use kebab-case for the
   slug; it doubles as the Claude Code =name:= field and the
   folder name (=doc/skills/<slug>/SKILL.org=). See
   [[id:3153aa4b-23d4-4111-9710-5a4971102edc][How do I create a new v2 doc?]] §"New Claude Code skill" for
   the exact invocation.

4. /Fill in the skeleton/, in this order:
   - =* When to use this skill= — one or two sentences.
   - =* How to use this skill= — numbered steps that mostly link
     to recipes (do not embed procedural detail).
   - =* Recipes= — id-links to every recipe the skill indexes.
   - =* Reference= — id-link to the knowledge document or
     component model that backs the skill.

5. /Check the markdown export block/. The =name:= must match the
   folder slug and the =#+title:=' kebab-case stem. The
   =description:= is what Claude Code shows in its skill selector;
   make it crisp and actionable.

6. /Verify against the v2 contract/. The [[id:44d7a328-f69c-4ee9-83a5-74bd3634ad69][skill-review]] skill is
   the canonical checklist — frontmatter, required sections,
   task-shape, no embedded inventories.

7. /Cross-reference the new skill from the relevant [[id:1a5e0079-5771-4213-b624-62fa9ac947b4][function]] docs/.
   Functions list the skills each cybernetic level invokes (see
   the =* Skills= section of each =doc/functions/s*.org=). A new
   skill that isn't linked from at least one function is
   effectively invisible to LLM sessions that load by function.
   Pick the function(s) the skill belongs to — usually one, often
   two — and add an id-link with a one-line "when to use" hint.

8. /Update the [[id:2003934B-ED05-D114-404B-08C0E0D844BB][Skills catalogue]]/ — add the new skill under the
   appropriate area heading. This is the canonical inventory.

9. /Rebuild the shipped bundle/ so Claude Code picks up the new or
   updated skill — see [[id:b5f231a9-5403-482c-8f2a-12d6917c58cd][How do I deploy the skills?]].
