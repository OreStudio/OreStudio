definition here. Other documents link to these entries with =[[id:...]]= so
meaning is defined once and reused everywhere. Each entry says what the term is

[[id:926b916e-a57e-498e-99ab-720b583c0362][cybernetic systems]], how to generate one, a representative example, and the
inventory page that lists every instance.

* Release
:PROPERTIES:
:ID: fabb50e5-518c-42d6-901e-848644e56ca2
:END:

The artefact a [[id:0820b7fd-147c-4832-ac25-c043d38d5b61][sprint]] produces when it closes — a tagged build plus release
notes capturing the sprint's landed work. The release is the upward signal that
work happened (per [[id:926b916e-a57e-498e-99ab-720b583c0362][cybernetic levels]] information flow). Every sprint cuts a
release; not every release ships a [[id:ca4d4bde-2eab-4f0a-bc8b-7c0bcb17bc82][version]] — version-level release decisions sit
with [[id:6ca8b181-bd81-4ef4-af27-64f5491d62f0][System 4]].

* Function
:PROPERTIES:
:ID: 66f2a9e8-0385-4d56-b4b1-0957208990dc
:END:

A named role that operates the system — /what someone (or some LLM session) does
when they are wearing a given hat/. Each function doc is the priming context an
LLM session loads when it takes on the role: its purpose, what to pre-read,
which [[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skills]] to invoke, what /not/ to do. Functions are role-shaped
(descriptive); skills are task-shaped (operational). Functions compose skills;
skills do not own functions.

* Skill
:PROPERTIES:
:ID: 4f6384fe-cde9-40f6-b13d-8a84b6cd77f7
:END:

A concrete, executable capability — /how to do a kind of work/ — in Claude
Code's skill format (=doc/skills/<slug>/SKILL.org=). Each skill is
self-contained: it describes when to use it, the procedure, and any artefacts it
expects. A skill is /task-shaped/: invokable in one session to perform a unit of
work. Contrast with [[id:66f2a9e8-0385-4d56-b4b1-0957208990dc][function]], which is /role-shaped/: a description of what
someone does, listing the skills that role draws on.

* Plan
:PROPERTIES:
:ID: 58d491a6-e4bd-4b0d-86bd-bb8465dfd4c7
:END:

A transient implementation strategy for a single task or story.
Lives next to the work as the =* Plan= section inside that document.
Distilled into the parent story's =* Decisions= when the work
closes; the =* Plan= section is then cleared. Plans do not outlive
their task.

* Capture
:PROPERTIES:
:ID: 671f18e4-e09c-4b3b-bd24-d33df8ae38a6
:END:

A pre-sprint idea: something we may want to do but have not yet pulled into a
sprint as a story. Captures live at
=doc/agile/product_backlog/{near,far}/<slug>.org=. The bucket is the source of
truth: =near/= = candidate work for the next version, =far/= = longer-term
wishlist. Captures are /not stateful/ — no TODO keyword, no =* Status= table.
They have only two effective states: /pending/ (the file exists in a bucket) and
/promoted/ (the file has been converted into a =story=). On promotion the file
is moved into the appropriate sprint folder and its =#+type:= flips from
=capture= to =story=; *the =:ID:= is preserved* so existing links keep working.

* Memory
:PROPERTIES:
:ID: 32b3a055-63fa-4edf-8dcc-901e9d883f22
:END:

A short, durable note an LLM session writes for the benefit of future
sessions — feedback the user gave, a fact about them, a fact about
the work in flight, or a pointer to an external system. One memory
per file under =doc/memory/=; carries =#+type: memory= and a
=#+memory_subtype:= of =feedback=, =user=, =project=, or
=reference=. Memory captures /how to work together/; code-derivable
facts and architecture belong in [[id:bd1f3458-f991-46be-abeb-5f06c2633375][knowledge]] instead.

* TODO state
:PROPERTIES:
:ID: 8a246d60-6a87-4b54-a023-564ec1965c61
:END:

The [[https://orgmode.org/manual/TODO-Items.html][TODO keyword]] on a stateful document's =* Status= headline. The full
vocabulary is =DISCOVERED BACKLOG STARTED BLOCKED | DONE ABANDONED= (active
states left of the bar, terminal states right). Documents that are not stateful
(knowledge, recipes, functions, meta) have no Status headline. See [[id:8810e0f9-c5b9-4cfa-a710-774ad6c771dd][lifecycle]] for
transitions.

* Status headline
:PROPERTIES:
:ID: 26af1bc4-8b83-4aa4-a324-6fbebc2de960
:END:

The =* Status= headline inside a stateful document. Carries the TODO keyword
(state) and four subheadings (=** Now=, =** Waiting on=, =** Next=, =** Last
touched=) that describe the live operational detail. Updated every time an
orchestrator or agent acts on the document.

* Predecessor story
:PROPERTIES:
:ID: cf9b011e-7ff1-4fdb-8b5f-148adf43a75f
:END:

A story in a prior sprint that the current story continues. Captured as
=#+predecessor: <id>= frontmatter plus a =Continued from: [[id:...][...]]= line
in the body. See cross-sprint stories in [[id:8810e0f9-c5b9-4cfa-a710-774ad6c771dd][lifecycle]].

* Successor story
:PROPERTIES:
:ID: 65976df1-57a0-4ba3-b6a5-a947886c20bd
:END:

A story in a later sprint that picks up where this one left off. Captured as
=#+successor: <id>= frontmatter plus a =Continued in: [[id:...][...]]= note in
=* Decisions=. Audit verifies bidirectional linkage.

* Agent
:PROPERTIES:
:ID: 366d6521-79e0-42ce-97f4-5fd0f443a923
:END:

A subprocess forked by a [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]] orchestrator LLM to execute one task. Operates
at [[id:dbfd530b-4b56-48d1-9f03-3a0d0da16552][System 1]]. See [[id:f58ca9e7-c65f-4d17-a9a7-95fde25b9ebd][execution model]] for the role and [[id:b544713a-ba23-4d5a-9cee-084212d7c8ba][S1 Agent function]] for the
priming context.

* Orchestrator
:PROPERTIES:
:ID: 32fbc685-b81d-4af8-9b10-294f9055bc75
:END:

The [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]] LLM session the user has assigned to a story. Forks agents, reviews
their output, decides what to do with discovered work. See [[id:eeca6981-3240-44d9-ada7-4b8c64a76b44][S2 Orchestrator
