* Result

Delivered on =feature/fix-heading-task-order-stale-started=.

Two fixes in =cmd_heading= (=compass.py=):

*Task order*: Added =_task_uuids_from_story(story_file)= which reads the
UUID sequence from the =* Tasks= org table via regex.  After collecting
=task_states= from the filesystem, they are re-sorted by that sequence
(unknown UUIDs sort to the end).  Filename order is used as a fallback
when the story has no Tasks table or no UUIDs.

*Stale STARTED detection*: When a task is STARTED and has a =#+pr:= value,
=pr_state()= is called to check the live GitHub state.  If the PR is
MERGED the task is emitted as a =close= suggestion (score 95) with action
=compass task done <slug>=, surfacing it at the top of the recommendation
list rather than hiding it as an ordinary "in-flight" item.
