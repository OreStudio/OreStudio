#+title: How do I view a CRM as a grid matrix from the shell?
#+description: Render one CRM's rates as an NxN grid (base x quote) from ores.shell, mirroring the Qt Cross-Rates Matrix window.
#+type: recipe
#+level: cross
#+filetags: :recipe:shell:marketdata:
#+created: 2026-07-16
#+updated: 2026-07-16

Reproduce, from the command line, the Qt Cross-Rates Matrix window's
own NxN grid layout -- base currencies as rows, quote currencies as
columns -- via [[id:F4D45153-D81E-464D-8D9B-2DBFEE7CB670][the crm_commands shell command]]'s =--matrix= flag.
The =ores-shell= block below is tangled into the runnable script
library as =crm_rates_matrix.ores=.

* Question

How do I view a CRM as a grid matrix from ores.shell?

* Answer

=crm rates <crm-name> --matrix= renders the same rows the flat
=crm rates <crm-name>= table would, reshaped into a grid keyed by
(base, quote) instead of one row per pair -- the diagonal shows a
dash, an unconfigured pair shows blank. =--matrix= requires a named
CRM: mixing rows from more than one CRM into a single grid would be
ambiguous (a (base, quote) cell could exist in two different CRMs at
once), so =crm rates --matrix= with no CRM name is rejected with an
error rather than silently picking one.

#+name: crm-rates-matrix
#+begin_src ores-shell :exports none
crm rates majors --matrix
logout
#+end_src

#+begin_src sh :noweb yes :results output :wrap example
./ores.shell ${log_args} ${connect_args} ${login_args} << 'EOF'
<<crm-rates-matrix>>
EOF
#+end_src

recreate -y -k= then the Barclays provisioning recipe), then
=compass shell -u tenant_admin@barclays_plc -p Secure-Password-123 -f
