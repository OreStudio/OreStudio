* Goal

Add an export toolbar action to the CRM cross-rates matrix UI panel
(see [[id:C685A8BE-0DF9-4D23-8AAB-E9DF9069E482][the panel task]]) that saves the
currently displayed matrix (whatever the current Base Currency/Hide
Empty/Show Inverted filters produce -- what the user sees, not
necessarily the full unfiltered response) to a file, in two formats:

- *CSV*: one row per (base, quote, rate, status, as_of) -- straightforward
  tabular dump for spreadsheet use.
- *ORE XML*: matching whatever ORE market data export convention the
  app already uses elsewhere (=Icon::ExportOre= exists in
  =IconUtils.hpp=, and other panels presumably already implement an ORE
  export -- reuse that existing convention/helper rather than
  inventing a new one).

* Plan

- *Which "ORE export" convention.* The app actually has two distinct
  ones: =ore::xml::exporter= (paired with =Icon::ExportOre=, used by
  e.g. =CurrencyMdiWindow=) produces ORE *XML* config files for
  reference-data entities (currencies, calendars, conventions,
  portfolios); =ore::market::serialize_market_data= produces ORE's
  real market-data format, flat tab-delimited text
  (=DATE\tKEY\tVALUE=, e.g. =20260101\tFX/RATE/EUR/CHF\t1.10=), the
  format ORE actually consumes for FX spot rates
  (=market::market_datum{date, key, value}=). CRM rates are quote
  data, not a config entity, so the fitting convention is the latter
  (text, keyed =FX/RATE/<BASE>/<QUOTE>=) even though the task's own
  title says "ORE XML" -- confirmed with the story owner before
  implementing.
- *Export scope.* Only cells actually rendered with a
  =CrmRateCellWidget= (i.e. a real quote/derived rate, direct or
  inverted) go into the export; the diagonal and dash ("no data")
  cells are plain =QTableWidgetItem=s with nothing to export and are
  skipped, for both CSV and market-data output. Whatever the current
  Base Currency / Hide Empty / Show Inverted filters produced on
  screen is exactly what gets exported -- no separate unfiltered
  fetch.
- *CSV*: one row per exported cell -- base, quote, rate, status,
  as_of -- RFC 4180-ish plain write (this data has no fields needing
  escaping, so no need to pull in
  =ores.refdata.api/csv/exporter='s escaping helper for this).
- *Market data text*: builds one =ore::market::market_datum= per
  exported cell -- =date= parsed from the =as_of= ISO-8601 date
  portion, =key= = ="FX/RATE/" + base + "/" + quote=, =value= = the
  rate formatted full precision -- then
  =ore::market::serialize_market_data()=.

* Result

Added two toolbar export actions to =CrmCrossRatesMatrixMdiWindow=
(=Export CSV=/=Export ORE=, using the existing
=Icon::ExportCsv=/=Icon::ExportOre= icons), backed by
=displayedRates_= -- exactly the cells actually rendered in the
current grid (respecting Base Currency/Hide Empty/Show Inverted),
captured at the end of each =reload()=, so export never re-fetches
or shows data the user isn't currently looking at.

- *CSV*: plain =base,quote,rate,status,as_of= rows, one per exported
  cell.
- *ORE market data*: builds one =ore::market::market_datum= per
  fresh/stale exported cell (=unavailable= cells skipped -- no rate
  value to export), keyed =FX/RATE/<BASE>/<QUOTE>=, via the real
  =ore::market::serialize_market_data()= convention -- not the
  unrelated config-entity XML exporter the task's own title
  originally implied (see =* Plan= for that scoping decision).
- Default export filenames encode the CRM name, active Base Currency
  filter, and which of Show Inverted/Hide Empty are on, plus a
  filesystem-safe ISO-8601 UTC timestamp (e.g.
  =crm_rates_majors_inverted_hideempty_20260715T131751Z.csv=) --
  added after initial manual testing showed two exports taken under
  different filters were otherwise indistinguishable in a file
  listing.
- =ores.qt.marketdata= gained a private link to =ores.ore.core.lib=
  for =ore::market::serialize_market_data=/=market_datum=.

Verified manually against live Barclays Plc CRM data (majors/exotics,
seeded via the standard provisioning script) through the QA
Validation Runner -- see the linked scenario, all 7 steps PASSED,
including inspecting the actual exported CSV/text file contents
(row counts, reciprocal inverted rates, Hide Empty narrowing) outside
the app.
