* Goal

Both collections' feeds seed their initial spot from imported
reference-vintage data (per the story's Decisions). Before such a feed
starts,
=ores.synthetic= should confirm the data actually exists in
=ores.marketdata= (queryable via =source=/=point_id=/
=observation_datetime=) and fail with a clear, actionable error — "No
vintage data found for source=ore.reference, date=2016-02-05 — run the
reference-data import first" — rather than silently defaulting to a
wrong value or failing deep inside feed-start logic. Provide this two
ways: an implicit check at feed-start, and an on-demand "validate all"
toolbar action in Market Simulator so a user can audit an entire
collection proactively.

* Acceptance

- Starting a feed with missing vintage data (either collection — both
  seed initial spot from the same vintage, per the story's Decisions)
  fails with a clear, actionable message naming the missing
  source/date and pointing at the import step.
- A Market Simulator toolbar action validates every configured entry's
  vintage-data availability on demand, surfacing results without
  requiring each feed to be started individually.
