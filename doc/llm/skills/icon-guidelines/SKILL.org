* When to use this skill

When selecting icons for a new UI element, reviewing icon usage for
consistency, or deciding between Regular and Filled variants.

* How to use this skill

The icon catalogue is the source of truth. This skill exists to
give it a task-shaped entry point.

1. /Identify the semantic category/ — action, status, navigation,
   content, etc.

2. /Find the icon/ in the [[id:6EF42145-CEE7-4035-806A-A4200A3E003A][Icon guidelines]] catalogue. Each entry
   has its Regular and Filled file paths, plus the usage rules
   that distinguish the variants.

3. /Pick the variant/:
   - /Regular/ — default / passive / secondary actions.
   - /Filled/ — active / hovered / selected / emphasis / primary
     actions.

4. /Pick the size/:
   - 16px — compact UI, inline with text, dense tables.
   - 20px — standard UI elements, buttons, navigation.
   - 32px — headers, emphasis.
   - 48px — large feature icons, empty states.

5. /Mind the spacing/ — at least 4px between icon and adjacent
   text or controls.

6. /If no existing icon fits the semantic/ — propose a new entry
   for the catalogue (a Fluent UI System Icons name plus the
   Regular/Filled SVG pair). Add it to [[id:6EF42145-CEE7-4035-806A-A4200A3E003A][Icon guidelines]] under the
   right semantic category /before/ using it in code, so the
   catalogue stays the single source of truth.
