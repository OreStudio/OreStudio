* Plan

1. =parties_commands=: =parties list [--category <c>] [--status <s>]=
   — wizard-style limit-1000 listing with case-insensitive
   client-side filters; party_table_io output.
2. =account_parties_commands=: =account-parties add <account_id>
   <party_id>= — party looked up to validate existence and source
   tenant_id (as the wizard does); =system.new_record= change reason;
   plus =account-parties list= for verification.
3. =tenants complete-provisioning= added to the existing tenants
   group over =iam.v1.tenants.complete-provisioning=.
4. =reports_commands=: =reports templates [--bundle <code>]= (default
   =ore_analytics=) — name, type, schedule, description.
5. Register all groups; no new library dependencies.
