* Result

Delivered via PR #1376 and PR #1379. Three synthetic config entity types
(market_data_generation_config, fx_spot_generation_config, gmm_component)
were fully generated via codegen across all layers: SQL DDL (with soft-FK
triggers and CHECK constraints), domain, JSON/table IO, repository,
messaging, generators, and Qt UI. The codegen templates were extended to
support soft-FK validations, CHECK constraints, default_value on natural
keys, and correct integer natural key handling — all reusable for future
entities. The new marketdata feed-config/binding type will be scaffolded
via codegen in the successor task (D8008A82).
