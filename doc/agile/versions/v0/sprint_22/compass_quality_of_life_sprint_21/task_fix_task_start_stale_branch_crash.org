* Result

Delivered in commit =f6371eed1= on =feature/fix-task-start-stale-branch-crash=.

Root cause: =_cmd_task_start= resolved =task_path= from the caller's branch
index, switched to the task branch with =git switch=, then called
=task_path.read_text()= — crashing with =FileNotFoundError= when the task
file had moved (e.g., the branch predated a sprint transition that
relocated the sprint directory).

Fix: after =git switch= succeeds, reload the doc index and match by task
UUID to re-resolve =task_path= (and =story_path=) on the new branch.  If
the UUID is not found in the post-switch index the original path is kept as
a fallback, preserving the existing behaviour for branches where the file
has not moved.
