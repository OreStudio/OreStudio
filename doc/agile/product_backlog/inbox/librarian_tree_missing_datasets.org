This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *inbox* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

The Data Librarian window's navigation tree no longer lists datasets
— observed during sprint 23 screenshot review of the accession-card
redesign (=DatasetViewDialog=, [[id:E2FCDE38-9EF7-434D-BAB0-CA34DFD3E5CC][Improve Data Librarian accession card
UI/UX]]). The status bar still reports datasets as loaded (e.g.
"Loaded 64 datasets"), so this looks like a tree-population/display
regression rather than a data-loading failure — the datasets exist
and are queryable (accession cards open and render correctly), but
the tree used to browse to them is empty or not showing entries.

* Why

The tree is the primary way a user browses to a dataset's accession
card; without it, datasets are only reachable some other way (if at
all), which defeats the point of the Librarian window. Given the
underlying data and detail-dialog rendering both work, this is likely
a small, isolated regression (tree model population, a stale filter,
or a signal wiring gap) rather than a deep architectural issue —
worth triaging early before it's forgotten.
