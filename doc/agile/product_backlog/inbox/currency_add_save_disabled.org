#+title: Currency add dialog: Save button stays disabled
#+description: Save button remains disabled even after filling in all required fields when adding a new currency, blocking creation.
#+type: capture
#+level: cross
#+filetags: :bug:qt:refdata:currency:
#+created: 2026-07-13

This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *inbox* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

In the Qt client, Reference Data > Currencies > Add opens
=CurrencyDetailDialog= in create mode. Filling in all fields (ISO
code, name, numeric code, symbol, fractions per unit, change reason,
commentary) does not enable the Save button — it stays disabled and
the currency cannot be created. Found while manually testing the
generic =HistoryDialog= widget (task
[[id:7D8659AB-220E-4077-BD17-3C4DCA1937D8][Single generic HistoryDialog widget in ores.qt]]); unrelated to that
change — needs its own repro against =main= to confirm scope and find
the root cause in =CurrencyDetailDialog='s create-mode dirty/
validation tracking.

* Why

Blocks adding new currencies from the Qt client entirely, which is a
basic CRUD operation.
