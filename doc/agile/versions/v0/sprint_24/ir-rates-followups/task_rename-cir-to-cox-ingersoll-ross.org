* Goal

Every other short-rate process in this codebase is spelled out
(=vasicek_process=, =hull_white_process=); only =cir_process= is a bare
acronym. QuantLib itself spells it out
(=CoxIngersollRossProcess=/=ExtendedCoxIngersollRoss=). Rename
=cir_process= (class, file, test file, header, doc) to
=cox_ingersoll_ross_process= for consistency.

The acronym also appears well beyond the process class itself — the
=synthetic_yield_curve_process_types= catalog/enum, the
=ir_curve_generation_config.process_type= stored value, Qt's
=IrCurveEditor= dropdown, =process_factory=/
=process_parameter_validation=, and doc references across sprint_23/24.
Scope this task to identifier/type renames the codebase controls
directly; whether the *stored* =process_type= string value (=cir=
today) also changes, or stays as a stable wire/DB identifier while only
the C++ symbol changes, is a decision to make explicit in the Plan —
changing a stored enum value is a data-compatibility concern the class
rename alone is not.
