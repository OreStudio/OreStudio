** Party Create shows a required Parent Party combo

Menu: *Reference Data > Parties*, click *Add*. Confirm a new *Parent
Party* combo is present, populated with existing parties (e.g.
BARCLAYS PLC and its GLEIF-imported subsidiaries). Leave it unset and
try to Save -- confirm Save stays disabled (no error dialog, matching
the existing required-field pattern for Short Code/Full Name).

** Create a second party succeeds with a parent selected

Fill in Short Code =TESTPTY2=, Full Name =Test Party Co 2=, pick a
*Parent Party* (e.g. BARCLAYS PLC), and fill the other required
fields. Save. Confirm the new row appears in the Parties list --
this is the case that previously always failed with a duplicate-key
error on =parties_root_party_uniq_idx=.
