* Notes

** Phase A — shared field-group models (done)

- =ores.dq.audit_record_field_group.org= and
  =ores.trading.instrument_identity_field_group.org= scaffolded via the
  new =compass add field_group= and generating clean,
  clang-format-stable headers into the post-regroup layout.
- Tooling built en route (all per the "How do I add a new document
  type?" recipe): =field_group= doc type (template + doc_generate
  registration + compass help), one-shot memory body args
  (=--statement/--why/--how-to-apply=), generator fixes (trailing
  whitespace, stray blank before closing brace), and the field-group
  output pattern in profiles.json fixed for the component regroup
  (the rest of profiles.json is still stale — captured separately).
- The generated headers replace the handwritten ones from #1047/#1070:
  per-field doc comments added; =final= dropped (consistent with the
  five generated trade field groups).

** Phase B — entity model annotations (done)

Contract recorded in every rates instrument model under =* C++=:

- C++ Flags gain =:domain_identity_group:= and =:domain_audit_group:=
  naming the composing field-group models.
- Columns folding into identity carry =:group: identity=
  (trade_type_code, party_id, trade_id); implicit scaffolding
  (version, tenant_id, workspace_id, primary key) and the implicit
  audit columns fold via the group declarations.
- SQL/entity/mapper column lists unchanged — only the domain class
  and mapper domain-side accesses nest.
- The entity templates do NOT consume the annotations yet (Phase C);
  each model carries a do-not-regenerate warning for the domain and
  repository profiles until then.

Findings:
- =rpa_instrument= has no trade_type_code anywhere (SQL, entity,
  mapper) — domain identity.trade_type_code is unmapped for rpa.
  Pre-existing narrowing, left as is.
- Models for fx/equity/bond/credit/commodity/scripted/composite do
  not exist yet; the in-flight migration tasks should adopt this
  contract when creating them (bond/credit/commodity additionally
  need their type-specific sub-groups from #1083).

** Phase C — follow-up

Teach org_loader + the entity templates to honour the group contract
and achieve zero-diff regeneration against the decomposed code; then
delete the dead generate_trading_instruments.sh (its JSON inputs are
already gone).
