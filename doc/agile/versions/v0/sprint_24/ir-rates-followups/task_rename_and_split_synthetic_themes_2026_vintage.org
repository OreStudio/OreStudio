* Goal

Give the synthetic FX/IR data themes clearer, self-explanatory
names and add a genuinely current real-vintage theme:

- Rename the "basic" theme to "Uniform Volatility Demo" (exaggerated
  uniform volatility, for visual demo purposes) and "realistic" to
  "2016 ORE Samples" (calibrated to plausible 2016 real FX
  volatility, aligned to ORE's own bundled 2016-02-05 reference
  vintage).
- Add a new "2026 Realistic" theme sourced from a real
  2026-05-05 Federal Reserve H.10 FX driver-rates vintage
  (=marketdata_fx_driver_rates_2026_populate.sql=,
  =tools/fetch_fed_h10_rates.sh=), so there is a current-vintage
  option alongside the 2016 one.
- Add the =MarketSimulatorWindow= Start-at-Root guard
  (=promptThemeAndStart()=) so starting from the tree root prompts
  for which theme to run instead of cascading all mutually-exclusive
  vintages at once.
- Fix =QaValidationRunnerWidget::captureScreenshot()= to grab the
  app's own top-level window via =QWidget::grab()= instead of
  =QScreen::grabWindow(0)= on the X11 root window, which returned
  black frames under WSLg's compositing XWayland setup -- needed to
  actually run the manual test scenario for this work.
- Manually test the above via a new =test_scenario= doc and file the
  three follow-up bugs/gaps it surfaced (duplicate per-asset-class
  theme folders, 2026 Realistic missing its vintage FX driver rates
  on provision, and a =find_series= false-negative in vintage
  validity) as separate tasks under this story rather than fixing
  them inline.
