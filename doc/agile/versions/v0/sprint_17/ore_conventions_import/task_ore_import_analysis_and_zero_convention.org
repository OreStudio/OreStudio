** Summary

Delivers Phase 1 of the ORE import conventions plan: a working end-to-end
=zero_convention= refdata type produced via =ores.codegen=, establishing
the pattern to be rolled out to the eight remaining ORE convention types.
Adds the plan document itself, a validator script for the ORE example
corpus, and a clutch of codegen fixes that the pilot surfaced.

*** Zero convention pilot (Phase 1)
- New =zero_convention= domain type across refdata.api / refdata.core / qt.refdata / sql (~50 generated files).
- NATS handler registered with the refdata service; Qt controller registered in the refdata plugin; entry in the new "Conventions" submenu.
- Bi-temporal history via the standard repository + generated =HistoryDialog= path.

*** Codegen infrastructure fixes
=zero_convention= is the first refdata type with nullable non-string fields (=compounding: optional<string>=, =spot_lag: optional<int>=, =end_of_month: optional<bool>=), which exposed several gaps:
- *Entity/mapper templates* double-wrapped =std::optional<T>= cpp_types in another =std::optional<>=; mapper then used =.value_or("")= and =.empty()= (invalid for =optional<int>= / =optional<bool>=). Added =is_already_optional= flag so the entity uses the cpp_type directly and mapper does plain assignments.
- *Table template* streamed =std::optional<T>= into libfort (no =operator<<= for optional). Added =opt_str<T>= helper emitted when any displayed column is optional.
- *Generator profile output path* hardcoded =/generator/= (singular) while template included from =/generators/= (plural); zero_convention files ended up split across both. Path now respects ={generator_facet_name}= placeholder.
- *Qt client model and detail/history dialogs* called =QString::fromStdString()= directly on =std::optional<std::string>=. Added =is_optional_string= flag cross-referenced from domain columns, with unwrap branches in all three templates.

*** Handler method rename
Renamed =del()= → =remove()= across 34 legacy NATS handlers (22 refdata, 5 iam, 4 reporting, 3 marketdata) to align with the name codegen already emits.

- [[proj:scripts/validate_ore_examples.sh][=scripts/validate_ore_examples.sh=]] — xmllint-based XSD validator for the ORE example corpus.
