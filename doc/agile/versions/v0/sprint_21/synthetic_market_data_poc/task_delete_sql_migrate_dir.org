* Goal

Remove =projects/ores.sql/migrate/= and its seven ALTER TABLE scripts. The
directory implies a migration workflow that does not exist: ORE Studio has no
migration runner and the only supported schema upgrade path is =compass db
recreate -y -k=. Its presence misled the reviewer during PR #1315 (observation_datetime
migration) into asking why ALTER scripts had not been provided for the new
column. Deleting it eliminates the confusion and makes the recreate-only policy
unambiguous. Optionally, document the policy in the SQL component overview.
