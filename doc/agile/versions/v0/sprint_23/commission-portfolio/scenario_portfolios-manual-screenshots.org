* Steps

** Connect to tenant Barclays Plc

Log in against the =prime_origin= environment as
=tenant_admin@barclays_plc= / =Secure-Password-123= and select
*BARCLAYS PLC*.

** Capture portfolios_main_dialog.png

- Open /Reference Data > Portfolios/.
- Confirm the list shows several rows with a mix of purpose types,
  statuses (as colour-coded badges), and aggregation currencies (with
  flag icons) — if the seeded data doesn't already vary enough, edit
  a couple of rows first so at least three different Status values
  and two different Purpose Type values are visible.
- Click the QA Validation Runner's *Screenshot* button with the full
  Portfolios list window in frame, then *Pass*.

** Capture portfolio_details_general.png

- Double-click any portfolio row to open its Details dialog.
- On the General tab, ensure every field is populated — if
  Description is empty, type a sentence into it and leave the dialog
  in edit mode (do not save yet).
- Click *Screenshot* with the General tab showing Id, Name, Purpose
  Type, Status, Aggregation Currency, Virtual, and Description all
  filled in, then *Pass*.

** Capture portfolio_status_combo.png

- Still in the Portfolio Details General tab from the previous step,
  click the Status combo box to open its drop-down.
- Click *Screenshot* with the open combo showing all five values
  (Active, Inactive, Closed, Frozen, Pending), then *Pass*.
- Close the combo and cancel out of the dialog without saving (or
  save if you'd rather keep the edited description — either is fine
  for the next step).

** Capture portfolio_history_dialog.png

- Open a portfolio's Details dialog, change its Status or Purpose
  Type, and Save — enter a change reason when prompted.
- Open History on that same portfolio (history icon in the dialog's
  title bar, or right-click the row and choose History).
- Select the two most recent versions in the Compare drop-downs and
  toggle *Only Changes* on, so the field list narrows to just the one
  field that differs.
- Click *Screenshot* with the History dialog in this state, then
  *Pass*.

** Read the manual in the Qt client

- Rebuild the Qt-embedded help (=compass build --direct help= then
  =compass build deploy_help_qch --preset <preset>=) and restart the
  client so it picks up the new chapter.
- Open the Portfolios chapter in the client's own manual viewer
  (Help → User Manual) and read it top to bottom.
- Confirm the prose reads correctly, the segues between sections
  work, and every screenshot appears where expected and actually
  shows what its caption claims.

** Check the built PDF

- Rebuild the manual (=compass build --direct manual=) and open the
  rebuilt =user_manual.pdf= at the Portfolios chapter's pages (e.g.
  via =zathura= or =evince=).
- Confirm layout, figure numbering, and captions look correct in the
  PDF's own rendering, and that all four new figures appear in the
  List of Figures.
