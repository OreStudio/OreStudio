** MASD framing notes

MASD (Model-Driven Architecture Software Development) is a software
engineering method that applies OMG's Model-Driven Architecture (MDA)
principles at the process level. The three key abstractions to cover:

- *Meta-model* — the abstract grammar of valid models. In ORE Studio the
  JSON schema for entity models (field names, types, constraints) is the
  implicit meta-model.
- *Profile* — a named binding of a meta-model to a concrete platform.
  =profiles.json= encodes ORE Studio's profiles (=sql_dq=, =v2_doc=, etc.)
  mapping model elements to Mustache templates.
- *Template* — a parameterised artefact skeleton. Each =.mustache= file
  is a template; the generator applies model data to produce a concrete
  file.

** Backout strategy framing notes

The term /backout/ refers to extracting generative structure from an
existing implementation rather than designing it top-down. The workflow:

1. Hand-write the =currency= entity across all layers (the reference
   implementation).
2. Identify which parts are mechanical and which contain genuine
   domain-specific logic.
3. Express the mechanical parts as Mustache templates parameterised by
   the JSON model.
4. Add a profile entry mapping =entity= model type to those templates.
5. Verify the generator reproduces the reference exactly (zero diff on
   regeneration).
6. Decommission the hand-written files; the generator owns the output.

This is distinct from a top-down MDA approach (model first, generate
always) because it grounds the generator in a proven implementation
rather than an idealised design.
