#+title: ores.refdata entity evaluation checklist
#+description: Per-layer criteria for evaluating whether an ores.refdata entity is fully commissioned. Each criterion shows whether it applies to domain entities, auxiliary (lookup) types, or both.

* Summary

A standard =ores.refdata= entity spans eleven layers: DB, domain, repository, service,
messaging (JSON IO), Qt UI, shell REPL, CLI, HTTP routes, Wt web UI, and manual. Each
layer has a fixed set of criteria that determines whether the entity is fully commissioned.

Two meta-types exist with different expected coverage:

- *Domain entity* — a primary entity with its own full CRUD stack, Qt MDI window, shell
  and CLI commands, HTTP routes, and a manual chapter. Example: =currency=.
- *Auxiliary type* — a small lookup table referenced by one or more domain entities as a
  soft-FK. It has the same DB and domain/service layers but a narrower UI surface:
  typically a dropdown in the parent entity's detail dialog rather than its own MDI
  workflow. Example: =rounding_type=, =monetary_nature=, =currency_market_tier=.

Each checklist table below has a *Domain* column and an *Aux* column. Values:
=Always= / =Usually= / =If applicable= / =N/A=.

Use this document alongside the [[id:A3F8C2D1-7E4B-4F9A-B2C5-8D1E6A3F0B7C][Entity Coverage Matrix]] when commissioning or auditing
any ores.refdata entity. The =currency= entity (and its three auxiliaries) was used as the
reference specimen to derive these criteria.

** DB layer

| Criterion | Domain | Aux | Notes |
|-----------+--------+-----+-------|
| Table exists in =refdata_<entity>_create.sql= | Always | Always | |
| =tenant_id= column (soft-ref to tenants) | Usually | Always | Domain: omit only for truly global entities (currencies, countries). Aux: always tenant-scoped. |
| =version= column for optimistic locking | Always | Always | |
| =modified_by= column | Always | Always | |
| =performed_by= column | Always | Always | |
| =change_reason_code= column | Always | Always | |
| =change_commentary= column | Always | Always | |
| =valid_from= and =valid_to= columns (temporal history) | Always | Always | No separate history table; GIST exclusion on same table. |
| CHECK constraint: =valid_from < valid_to= | Always | Always | |
| CHECK constraint: natural key column not empty | Always | Always | |
| GIST exclusion constraint prevents temporal overlaps | Always | Always | Requires =btree_gist=. |
| Primary key on (tenant_id, natural_key, valid_from, valid_to) | Always | Always | |
| Version unique index (where =valid_to = infinity=) | Always | Always | |
| Code / natural key unique index (where =valid_to = infinity=) | Always | Always | |
| Tenant index (where =valid_to = infinity=) | Always | Always | |
| =display_order= column | N/A | Always | Aux types are shown in dropdowns; ordering matters. |
| =description= column | N/A | Always | Human-readable description of the lookup value. |
| Insert trigger: version management + tenant/account validation | Always | Always | |
| Delete rule: soft-delete (set =valid_to = now=) | Always | Always | |
| Validation function =ores_refdata_validate_<entity>_fn= | Always | Always | Used by parent entities' insert triggers to validate soft-FKs. |
| Entity-specific soft-FK validations in trigger | Always | N/A | Domain: validate rounding_type, monetary_nature, etc. Aux: no downstream references. |
| =party_id= soft-reference | If applicable | N/A | Domain only, for party-scoped entities. |
| =workspace_id= soft-reference | If applicable | N/A | Domain only, for workspace-scoped entities. |
| =coding_scheme_code= optional column | If applicable | N/A | Domain only; links to DQ coding scheme. |

| Criterion | Domain | Aux | Notes |
|-----------+--------+-----+-------|
| Domain struct in =ores.refdata.api/domain/<entity>.hpp= | Always | Always | |
| All business columns represented as struct fields | Always | Always | |
