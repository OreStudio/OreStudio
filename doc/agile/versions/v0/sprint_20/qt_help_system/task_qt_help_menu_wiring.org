* Result

Wired =HelpViewer= into the Help menu with a User Manual action (F1 shortcut) that opens it
in an MDI sub-window, following the existing =onAboutTriggered= single-instance pattern
(raise if already open, otherwise create and show). Pre-built =user_manual.qch= checked into
=projects/ores.qt/application/resources/help/= and deployed to =publish/bin/help/= via a
CMake POST_BUILD step. Qt SQLite plugin likewise deployed to =sqldrivers/= next to the binary.

Key discoveries: Qt 6.7+ defaults =QHelpEngine= to read-only — must call =setReadOnly(false)=
before =setupData()= or the =.qhc= stays 0 bytes and all namespace registrations fail silently.
=.qhc= is co-located next to the =.qch= so each build tree keeps its own independent index.
