* When to use this skill

When the user asks to provision, create, or set up a new environment or
worktree — e.g. "provision a new environment", "create a festive-hawking
worktree", "set up a new dev env".

* How to use this skill

1. *Check the genesis environment exists* at =~/Development/OreStudio/OreStudio.genesis=.
   If it is missing, tell the user to clone the repo there first (see the provision recipe).

2. *Provision the worktree* from the current checkout:

   #+begin_src sh
   # Auto-generated adjective-noun name:
   ./projects/ores.compass/compass.sh env provision

   # Or a specific name:
   ./projects/ores.compass/compass.sh env provision <adjective-noun>

   # Light environment (no C++ stack):
   ./projects/ores.compass/compass.sh env provision --type light

   # Provision + configure in one step (requires PGPASSWORD):
   ./projects/ores.compass/compass.sh env provision \
       --preset linux-clang-debug-ninja -y
   #+end_src

3. *If =--preset= was NOT given*, run =env configure= in the new worktree:

   #+begin_src sh
   cd ~/Development/OreStudio/ores_dev_<name>
   ./projects/ores.compass/compass.sh env configure \
       --preset linux-clang-debug-ninja -y
   #+end_src

4. *For full environments*, complete setup:

   #+begin_src sh
   ./projects/ores.compass/compass.sh db recreate -y
   ./projects/ores.compass/compass.sh services start
   #+end_src
