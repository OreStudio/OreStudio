#+title: Write org-timestamps to the build output directory
#+description: Configure org-publish to write .org-timestamps/ under the build output directory rather than ~/.org-timestamps, supporting clean-build semantics and keeping the home directory uncluttered.

* What

Set =org-publish-timestamp-directory= in =.build-site.el= to a path
under the CMake build output directory (e.g.
=build/output/.org-timestamps/=) rather than the default
=~/.org-timestamps=. This is a one-line change in the Elisp build
configuration.

* Why

Currently, =org-publish= writes its change-tracking timestamps to
=~/.org-timestamps= in the user's home directory. This has two problems:

1. *Home-directory pollution*: =~/.org-timestamps= accumulates entries
   for every project on the machine; entries from other projects can
   interfere with ORE Studio's incremental rebuild logic.
2. *Clean-build semantics*: deleting the build directory should give a
   fully clean rebuild. With timestamps in =~/.org-timestamps=, a
   =cmake --build ... --target deploy_site= after wiping [[proj:build][=build/=]] may
   skip files because the old timestamps still exist in the home dir.

Directing timestamps into the build tree means =rm -rf build/output/=
gives a guaranteed clean rebuild with no hidden state.
