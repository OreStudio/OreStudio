* Goal

Make the currency table bitemporal: each row carries both validity
time (when the fact is true in the world) and transaction time
(when we recorded it). Establishes the pattern every later
reference-data type uses.

* Acceptance

- Currency rows have validity_start, validity_end, tx_start,
  tx_end columns.
- An "as of" query returns the row that was current at a given
  point.
- Updates produce new versions rather than overwriting.

* Notes

- Postgres 18's native temporal support was investigated but
  deferred (separate investigation task) — the project pins to a
  current LTS, so we do bitemporality in application code.

* Result

- Currency table bitemporal; pattern propagated to other reference
  types in later sprints.
