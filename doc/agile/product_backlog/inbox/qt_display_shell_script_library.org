* What

A Qt UI surface for the ores-shell script library: browse the
available =.ores= scripts (name, description and step commentary
sourced from their literate org files), view a script's contents, and
run one against the current session — likely alongside the embedded
shell widget.

* Why

The provisioning epic creates a script library in
=projects/ores.shell/scripts/= (org-mode tangled). Operators using
the Qt client should be able to discover and execute those scripts
without leaving the UI or knowing the filesystem layout. Idea
recalled from earlier discussions; the backlog had no capture for it
(verified 2026-06-06).
