* Help System

#+begin_src markdown

I have a user manual written in Emacs Org-mode that exports to HTML files (e.g.,
index.html, intro.html, advanced.html, along with a CSS file and an images/
folder).

I want to use the Qt Help Framework (QHelpEngine) to consume this documentation
natively inside my Qt C++ application.

Please act as a Senior C++ and Automation Engineer and implement the following
User Story.

---

### User Story
As a Developer,
I want to automate the compilation of my exported HTML documentation into a Qt Compressed Help (.qch) file and display it inside my Qt application,
So that users have a native help system with a Table of Contents and a text browser.

### Technical Requirements

#### Part 1: Automation Script (Python)
Write a Python script (e.g., generate_help.py) that:
1. Scans a directory of exported HTML files.
2. Automatically generates the required Qt Help Project (.qhp) XML file.
   - It should dynamically populate the <files> section with all .html, .css, and image files it finds.
   - For the Table of Contents (<toc>), it can either parse the HTML titles or accept a simple hardcoded structure I can edit later.
3. Automatically executes the system command `qhelpgenerator manual.qhp -o manual.qch` to compile the final binary.

#### Part 2: Qt C++ Implementation
Provide a clean, modern C++ (Qt 6 preferred, but Qt 5 compatible) implementation showing how to load this `manual.qch` file. Please provide:
1. The necessary initialization code for `QHelpEngine`.
2. How to extract and display the Table of Contents (`QHelpContentWidget`) in a sidebar (like a QDockWidget or QTreeView).
3. How to connect that sidebar to a `QTextBrowser` so that clicking a topic opens the correct HTML page.

Please provide well-commented, production-ready code for both parts.
#+end_src
