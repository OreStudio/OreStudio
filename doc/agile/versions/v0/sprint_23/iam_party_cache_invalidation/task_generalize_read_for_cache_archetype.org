* Goal

Turn the bulk "read all active entities for a tenant" RPC (today
hand-written once, for party, as =read_parties_for_cache=) into a
standard generated archetype: a =read_for_cache= model flag that any
tenant-scoped entity can opt into to get its protocol structs, handler
method, and registrar wiring generated for free, the same way
list/save/remove/history already are.

Party itself stays on its hand-written implementation for now — its
messaging layer is separately flagged as drifted from codegen and not
yet regenerable (see the blocked "Party messaging layer has drifted
from codegen" capture) — so this task builds and verifies the
archetype without migrating party onto it.

* Acceptance

- A new =read_for_cache= C++ Flags knob exists, gated on =has_tenant_id=.
- Setting it on an entity model generates, with no hand-written code:
  - =read_{{entity_plural}}_for_cache_request/response= protocol structs
    (tenant_id in, full unpaginated vector out).
  - A =read_for_cache= handler method that swaps tenant context and
    reuses the entity's existing paginated =list_*= service method.
  - Registrar wiring subscribing the new subject to that handler method.
- Entities that don't set the flag are generated byte-for-byte
  unchanged (regression check against an existing entity).
- Archetype docs (protocol, nats-handler, nats-sub-registrar) describe
  the new flag.
- party's model/messaging layer is untouched by this task.

* Plan

1. Add =read_for_cache= flag to the C++ Flags model schema, defaulting
   to false; assert/document its dependency on =has_tenant_id=.
2. Extend =cpp_protocol.hpp.mustache= with a =read_for_cache= block
   generating the request/response structs.
3. Extend =cpp_nats_handler.hpp.mustache= with the =read_for_cache=
   method (tenant_context swap + call into the existing paginated
   =list_*= service method with a large limit), plus the conditional
   =tenant_context.hpp= include.
4. Extend =cpp_nats_registrar.cpp.mustache= with the subscribe block.
5. Update the three archetype =.org= docs describing the new flag.
6. Verify: temporarily flip the flag on a real tenant-scoped entity,
   regenerate, confirm it builds; revert the flag afterwards unless a
   real second consumer is wanted. Confirm unrelated entities regenerate
   unchanged.

* Result

Added a generic =read_for_cache= codegen archetype, gated on
=has_tenant_id=:

- =core.py=: new flag, default false, validated against =has_tenant_id=.
- =ores.cpp.protocol.protocol_header.org= (→ =cpp_protocol.hpp.mustache=):
  emits =read_<entity_plural>_for_cache_request/response=.
- =ores.cpp.nats-handler.nats_handler_header.org=
  (→ =cpp_nats_handler.hpp.mustache=): emits the =read_for_cache= handler
  method (tenant-context swap + reuse of the entity's existing paginated
  =list_*= service method with an unbounded limit).
- =ores.cpp.nats-sub-registrar.nats_registrar_implementation.org=
  (→ =cpp_nats_registrar.cpp.mustache=): wires the subject to the handler.
- All three archetype docs updated to describe the new flag.

Verified via =compass codegen entity generate --diff= against
=counterparty= (a real tenant-scoped entity): unaffected when the flag
is unset (no regression), and renders correct C++ when temporarily set
(flag reverted, not merged in — no second consumer wired up by this
task). party keeps its hand-written =read_parties_for_cache=
unchanged, pending the separate messaging-drift fix.
