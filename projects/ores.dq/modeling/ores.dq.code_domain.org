A code domain is a classification registry that names the context in
which a code value exists. It disambiguates identical codes used in
different entity types — e.g. 'ACTIVE' in party_status vs 'ACTIVE'
in book_status.

Code domains are reusable beyond badges: any future system needing
to namespace code values (validation rules, audit customisation, etc.)
can reference this table.

** Custom badge mappings tab

The live =CodeDomainDetailDialog= (=projects/ores.qt/refdata=) is itself
hand-maintained, not generated from this model -- there is a separate,
unused generated scaffold under =projects/ores.qt/dq= left over from an
earlier component layout (tracked as drift; see the codegen drift-catalog
story). Paste blocks don't apply here since the live file isn't
tangled/generated. Instead, a read-only "Badge Mappings" tab (showing
every entity_code -> badge_definition mapping registered for the
displayed code_domain) is wired directly into
=projects/ores.qt/refdata/src/CodeDomainDetailDialog.cpp= by hand,
mirroring the paste-block pattern used by =PartyHierarchyTab= in shape
(see =BadgeMappingsTab=) even though no actual paste block is involved.
