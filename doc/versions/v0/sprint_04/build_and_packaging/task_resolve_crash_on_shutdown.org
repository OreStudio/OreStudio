* Goal

Diagnose and fix the crash on Qt UI shutdown. Root cause:
the MDI detach/reattach logic was creating /two/ owners for each
detachable sub-window, which led to a double-delete at exit.

* Acceptance

- The application shuts down cleanly with no crash.
- Detach / reattach can be repeated any number of times in one session before shutdown without state corruption.

* Plan

(Plan was distilled into the parent story's =* Decisions= and cleared when the task closed.)

* Notes

- Total time spent in v0: ~0:50.
- Design rule adopted: keep detachable widgets parent-less; =MainWindow= is the sole owner; attaching to MDI uses =QMdiArea::addSubWindow()= which reparents only the internal viewport, not the widget; deletion happens once, from =MainWindow='s destructor, with =QObject::destroyed= cleaning the lookup maps as widgets die.

* Result

Shutdown crash fixed; detach/reattach ownership rule documented in the codebase.
