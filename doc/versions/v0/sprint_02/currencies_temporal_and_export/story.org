This page documents a [[id:699a8d45-b67e-4d3e-9206-24fa17c51ada][story]] in *Sprint 02*. It captures the goal, current status, acceptance criteria, and the tasks that compose it.

* Goal

Build on the sprint-01 currencies-end-to-end story: add bitemporal
versioning so changes can be tracked over time, add a CLI command to
dump currency data, and create a containing structure for the ORE
example files so they can be loaded as a group.

* Acceptance

- Currency records carry validity and transaction time columns.
- The CLI dumps currencies to a file in either XML or JSON.
- ORE example data is organised under a top-level container that
  supports load/save.

* Decisions

- /Bitemporal in application code, not Postgres 18/ :: Postgres 18's
  native temporal support was investigated but deferred because the
  project pins to a current LTS Postgres. We do bitemporality in
  application code for now.
- /CLI dump goes through reflect-cpp/ :: avoids a separate
  serialisation path for the dump output.

* Out of scope

- Temporal support for other reference types (deferred — each gets

- A general-purpose ORE-data viewer (deferred).
- Full Postgres write path for currencies — completed in sprint 03's
