** Summary

- MSVC =fatal error C1202= fires when =rfl= reflects too many complex types in a single translation unit — =ClientManager.cpp= accumulated =rfl::json::read= instantiations for IAM, session, HTTP, trading, and eventing types in one TU, hitting the constexpr depth limit in =rfl/internal/no_duplicate_field_names.hpp=
- Split into four focused TUs, each carrying only the rfl instantiations relevant to its domain: =ClientManager.cpp= (connect/login/session), =ClientManagerSetup.cpp= (testConnection/signup), =ClientManagerTrades.cpp= (listTrades/getTradeDetail — the 25-field =trade= struct was the primary trigger), =ClientManagerEvents.cpp= (subscribeToEvent/unsubscribeFromEvent)
- =GLOB_RECURSE= in =src/CMakeLists.txt= picks up the new files automatically; no CMakeLists.txt change needed
- No logic changes; all method signatures and behaviour are identical

Landed.
