* Goal

Settle the design questions before any mass migration: how the 140
templates map onto the three-level literate hierarchy (template ←
facet doc ← group doc ← groups overview) — what counts as a facet
group, e.g. =cpp_domain_type_*= — where the org sources live, how
tangling runs (org-babel via
the existing =ores-babel.el= / tangle tooling), what the generated-file
header looks like inside a =.mustache= artefact, and how the zero-diff
drift check is wired (CI target and/or compass command). Prove the
design with a small pilot batch of templates and record the decisions
in the parent story's =* Decisions= section.

* Result

Design settled and proven with the cmake pilot (PR [[https://github.com/OreStudio/OreStudio/pull/1123][#1123]], merged).
The five decisions — facet granularity, co-location, emacs/CMake
tangle tooling, output-neutral header, zero-diff drift check — are
recorded in the story's =* Decisions=. The pilot migrated all eight
=cmake_*= templates into =cmake.org=: tangled artefacts byte-identical
to HEAD modulo the header, pystache renders byte-identical. The
three-level hierarchy is scaffolded (five group docs + overview) and
facet inventories are generated by
=regenerate_facet_inventories.py= (=--check= for CI). Remaining
migration is sliced into the rollout task; tooling support landed via
the add-facet-doc-types task in the same PR.
