* Goal

Register each Catch2 test /suite/ as a single CTest entry rather
than registering every individual test case. Now that the test database
is isolated per-suite, the per-case granularity is too slow.

* Acceptance

- CTest dashboard shows one entry per Catch2 suite.
- Total CTest configure + run time measurably lower than before.
- Per-case results are still visible in the suite's own output.

* Notes

- Total time spent in v0: ~0:13 for the registration change; the deeper rework continues as the BLOCKED carry-over /CTest shows only one test per test suite/.

* Result

Suite-level CTest registration adopted. The follow-up issue (CTest only
running the first case of a suite under certain conditions) carries
forward to a successor sprint as a =BLOCKED= predecessor link.
