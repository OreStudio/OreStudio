A catalog entry for a market data series — it records what is being observed:
a yield curve, vol surface, spot rate, fixing index, or similar. Standard
temporal reference data; changes infrequently so a regular table with GIST
exclusion is appropriate.

Every ORE market data key follows the skeleton =TYPE / METRIC / QUALIFIER=;
=asset_class= and =series_subclass= carry the coarse taxonomy for filtering.

* Natural keys

** series_type
:PROPERTIES:
:type:     text
:cpp_type: std::string
:END:

ORE market data type token (e.g. =FXSpot=, =YieldCurve=, =FXVolatility=).

** metric
:PROPERTIES:
:type:     text
:cpp_type: std::string
:END:

Metric within the series type (e.g. =SPOT=, =DISCOUNT=, =FLAT_FWD_VOLATILITY=).

** qualifier
:PROPERTIES:
:type:     text
:cpp_type: std::string
:END:

Free-text qualifier disambiguating the series within type+metric (e.g. =EUR=,
=EUR-EURIBOR-3M=, or an empty string for scalars).

* Columns

** asset_class
:PROPERTIES:
:type:     text
:cpp_type: std::string
:nullable: false
:END:

Coarse asset class taxonomy: =FX=, =RATES=, =CREDIT=, =EQUITY=, =COMMODITY=,
=INFLATION=, =BOND=, =CROSS_ASSET=.

** series_subclass
:PROPERTIES:
:type:     text
:cpp_type: std::string
:nullable: false
:END:

Subclass within the asset class (e.g. =SPOT=, =VOLATILITY=, =YIELD=, =SPREAD=).

** is_scalar
:PROPERTIES:
:type:          boolean
:cpp_type:      bool
:nullable:      false
:default_value: false
:END:

True when the series has no point dimension (e.g. an FX spot rate or a single
fixing), false when it is curve/surface/matrix data.
