* Goal

Every newly-created party ends up with a usable synthetic FX spot
config with no manual step, by adding a new automatic Phase 3.5
to =PartyExecutePage= (=projects/ores.qt/application/src/PartyProvisioningWizard.cpp=)
that publishes the =synthetic_market_data= bundle. Mirror Phase 2's
=startOrgPublish=/=onOrgWorkflowComplete= structure exactly (build a
=publish_bundle_request= with =bundle_code = "synthetic_market_data"=,
run it via =QtConcurrent=, show progress via the existing
=WorkflowStepsWidget=, connect =instanceReachedTerminalState=) rather
than the Phase 3 reports pattern, since there's no per-item selection
to offer yet. Insert it between =startReportInstall='s completion and
=startActivate()=.
