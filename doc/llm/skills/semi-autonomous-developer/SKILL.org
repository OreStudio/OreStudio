* When to use this skill

When the user asks you to work autonomously on a specific sprint story
with minimal back-and-forth.  The story must already exist in the
current sprint under =doc/agile/versions/v0/sprint_NN/<story>/story.org=.

* How to use this skill

** Step 1 — Find and read the story

Locate the current sprint (highest-numbered directory under
=doc/agile/versions/v0/=).  Read the story file and its tasks.  If
no tasks exist yet, create them with =TaskCreate= before coding.

** Step 2 — Create a feature branch and mark STARTED

Follow [[id:DA003A23-1604-43E7-A98F-A305148FBCF4][How do I start work on a story?]] — branch from =main= using
=feature/concise-lowercase-description=, mark the story =STARTED=,
and commit the status change.  The full state machine is in
[[id:8810E0F9-C5B9-4CFA-A710-774AD6C771DD][Lifecycle]]; the execution-phase context is in [[id:227958A5-0F78-4CC6-A116-B83988592B68][Sprint execution]].

** Step 3 — Implement incrementally

For each task, follow this cycle without exception:

1. *Implement* — make the minimal change for the task.
2. *Compile* — use [[id:654D4A70-E63D-4C18-B5A5-886066E36314][cmake-runner]] for build and test commands.
   Fix errors before proceeding.
3. *Test* — run the affected component's tests.  Fix failures before
   proceeding.
4. *Commit* — one atomic commit per task using the =[component] Description=
   format from =CLAUDE.md=.
5. *Update task* — mark the =TaskCreate= task completed.

** Step 4 — Handle ambiguity

When the right approach is unclear, make a reasonable assumption,
document it in the story's =* Decisions= section, and proceed.  Do
not pause for confirmation — the PR review is the feedback gate.

** Step 5 — Open the PR

When all tasks are green: push the branch and use [[id:545D6B43-17FC-0234-FF2B-AB6565695616][pr-manager]] to open
the PR.  Mark the story =DONE= and add the PR number to its =* Status=
table.
