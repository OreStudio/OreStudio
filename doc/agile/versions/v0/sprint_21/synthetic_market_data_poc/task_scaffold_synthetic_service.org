* Notes

The =ores.synthetic.service= CMake project pre-existed (created in an earlier sprint for the
organisation data generation use case). Steps 1–4 of the plan were already complete. This
task therefore focused on adding the new FX spot tick generation infrastructure that will
host the PoC implementation:

- =ores.marketdata.api= gained the three shared types: =fx_spot_tick=, =IStochasticProcess=,
  =IFxSpotFeed=. They live in =ores.marketdata.api= (not =ores.synthetic=) so a future
  calibration service can use them without depending on the synthetic service binary.
- A service-level registrar stub (=src/registrar.hpp= + =src/registrar.cpp=) was added to
  =ores.synthetic.service= and wired into =application.cpp= alongside the pre-existing
  org-generation registrar. The stub currently registers no NATS handlers; they will be
  added in step 2 (tick loop).
