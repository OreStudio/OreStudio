* Goal

Discovered while modelling [[id:04A121FA-00D6-43EB-9B21-04EDC1FA493D][Currency pair support in reference data]]:
=day_count_fraction_type= lives in =ores.trading=
([[id:7856FDEB-E15B-4A72-BFA5-57ECF053AFFF][ores.trading.day_count_fraction_type]]), with exactly one consumer
(=trading_swap_legs=), while every =*_convention= entity that would
also need it (=ibor_index_convention=, =zero_convention=,
=fx_convention=, etc.) lives in =ores.refdata=. Move it to
=ores.refdata=, commission it fully — SQL, codegen model, domain,
repository, service, messaging, Qt UI — following the
[[id:897A7156-504D-4581-AD4C-026D643C0838][Domain entity evaluation checklist]], update =trading_swap_legs='s one
soft-FK to point at the new location, and delete the =ores.trading=
original. This unblocks =currency= soft-FK'ing =day_basis= to it
same-component, matching every other refdata soft-FK.
