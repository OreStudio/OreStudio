Component names like =ores.qt.api= and =ores.compute.core= use =api= and =core= as sub-component qualifiers but these words are not C++ namespaces — the actual C++ namespace is =ores::qt= or =ores::compute=. This causes confusion when reading include paths vs namespace paths.

Options:
1. Accept the mismatch and document the convention (component directory name ≠ C++ namespace).
2. Rename components so the directory structure mirrors the namespace hierarchy.

Resolve the naming convention in a design note before touching any includes.
