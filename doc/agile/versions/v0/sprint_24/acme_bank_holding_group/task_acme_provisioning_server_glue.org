* Notes

Two real, general (not Acme-specific) bugs surfaced by the
orchestrator looping over three companies in one transaction:

- =on commit drop= temp tables in
  =refdata_publish_from_dq_create.sql= collided when the same publish
  function ran more than once per transaction -- fixed with a =drop
  table if exists= guard before each =create temp table=.
- Sibling-dataset ref-map lookups (=bu_ref_map=,
  =portfolio_ref_map=) matched by name only, unscoped by party --
  identical unit/portfolio names across different companies (e.g. a
  shared top-level division name) collided -- fixed to scope the
  match by party.

The initial local build under =linux-gcc-debug-ninja= needed a
one-off =cmake --preset= reconfigure after adding new source files;
switched to building/testing under the environment's actual preset
(=linux-clang-debug-make=) throughout.

A session crash mid-build left =publish/lib= with a truncated
=libores.logging.so= (8MB, missing section headers) and ~26
zero-byte/broken-symlink artefacts from concurrent contention;
cleared and relinked before the final green =rat= run.
