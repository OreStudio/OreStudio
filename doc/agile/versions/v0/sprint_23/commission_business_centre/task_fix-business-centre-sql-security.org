~not exists (select 1 from ... limit 1)~ to filter on

fix). Investigated why the fixed =sql_schema_create.mustache=

file was tangled from a different, older template
(=sql_schema_table_create.mustache=) that was never updated when
=sql_schema_create.mustache= was patched in sprint 21 -- consistent
with the discarded capture's prediction that pre-fix entities would
be swept "naturally through the commissioning pipeline" rather than
via a proactive sweep. Did not touch either template: business_centre
has no live codegen model to regenerate from (appraisal finding 3),
and touching a shared template mid-story risks regressing every other
entity still generated from it.
