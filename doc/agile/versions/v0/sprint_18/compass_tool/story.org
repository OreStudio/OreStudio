* Goal

The LLM can answer "where is X?" and "what does Y link to?" over the
org-roam graph without exhaustive manual exploration — using a
=compass.py= CLI that queries a pre-built FTS/SQLite index of all
=.org= files and returns ranked matches with UUID and path.

* Acceptance

- =scripts/compass.py= accepts a query string and returns ranked
  matches (UUID, title, path, snippet) from the org-roam SQLite DB.
- =scripts/compass.sh= wraps the Python script for convenient CLI use.
- The tool is documented in a recipe under =doc/recipes/llm/=.
- A Claude Code skill (=compass=) exposes the tool to the LLM with
  a clear when-to-use and invocation pattern.
- The =.compass.db= index file is in =.gitignore=.

* Decisions

- /Use the existing org-roam SQLite DB./ org-roam already maintains
  =~/.emacs.d/org-roam.db= (or project-local). Querying it directly
  avoids maintaining a separate index. =compass.py= connects read-only.
- /Python over shell./ FTS ranking and JSON output are cleaner in Python
  (sqlite3 stdlib). The =compass.sh= wrapper provides a one-line entry
  point for users who prefer not to invoke Python directly.

* Out of scope

- Full semantic/vector search (plain FTS is sufficient for orientation).
- Integration with the CI build.
- Real-time index updates (org-roam handles that).
