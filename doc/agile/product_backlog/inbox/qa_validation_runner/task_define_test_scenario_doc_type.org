* Plan

Followed the existing =task=/=story= registration path in
=ores.codegen/src/doc_generate.py= rather than inventing a parallel
mechanism: added =test_scenario= to =TYPE_TO_TEMPLATE=,
=DEFAULT_INITIAL_STATE= (=PENDING=, since a scenario's lifecycle is
pending/passed/failed, not backlog/started/done),
=PARENT_OF_TYPE= (parent type =task=, since a scenario verifies a
task), =derive_ancestor_slugs= (same shape as =task= — lives directly
in the story folder), title-prefixing (=Test Scenario: ...=), and the
output-path rule (=scenario_<slug>.org=, mirroring =task_<slug>.org=
but under its own prefix so the two sort separately in a listing).

Two things needed beyond copying =task='s pattern:

- *Mandatory dual link.* A task's parent (the story) is
  auto-detectable by convention: =parent_dir / "story.org"= always
  exists in the standard layout, so no explicit flag is needed. But a
  =test_scenario='s task parent has no fixed filename
  (=task_<slug>.org= — the slug varies), so =--parent-id=/
  =--parent-title= must be passed explicitly, same as when scaffolding
  a task's own children would. The *story* link, though, is trivially
  auto-detectable exactly like a task's own parent-story lookup — since
  a =test_scenario= lives directly in the story folder alongside its
  task, =parent_dir / "story.org"= resolves it with no extra flags in
  the common case. Added =--story-id=/=--story-title= as optional
  overrides for the rare case (e.g. scaffolding into a mismatched
  directory) where auto-detection guesses wrong.
- *New template.* Wrote
  =library/templates/doc_test_scenario_org.org= (literate source) and
  hand-tangled =doc_test_scenario.org.mustache= (normally
  =compass build --direct tangle_codegen_templates= would do this from
  Emacs org-babel, not available in this session) — mirroring
  =doc_task_org.org='s structure but with scenario-specific sections
  instead of Goal/Plan/PRs/Review.

Verified end-to-end: scaffolded a real doc under
=doc/agile/versions/v0/sprint_22/qa_validation_runner/=, confirmed both
=--parent-id= (explicit) and the story (auto-detected, no flag needed)
