* When to use this skill

Immediately after a PR is opened, when you want to drive it to a
green state with minimal user intervention — automatically polling
CI, addressing review comments, and fixing failures as they appear.

* How to use this skill

1. /Start the watcher/. Prefer the =pr-watch.sh= script — it polls
   cheaply via =gh= and only invokes Claude when there is something
   actionable. See [[id:F6E88DCC-9240-4632-A9A3-9F36F79C9262][How do I monitor a PR until green?]] for both the
   script and the manual fallback.
2. /On each notification/, branch on what changed:
   - /A check went red/ → run [[id:62D39F43-5D91-4A82-B1C3-62C2AF21BEFC][How do I fix a failing CI check?]];
     push the fix and return to monitoring.
   - /A reviewer opened a new thread/ → run [[id:26C1DC06-82D2-46C4-94D6-DF9231BA171B][How do I address PR
     review comments?]]; reply + resolve each thread before
     returning.
3. /Stop when terminal/. The loop exits when CI is fully green and
   no review thread is unresolved. Inform the user; hand off to
   [[id:545D6B43-17FC-0234-FF2B-AB6565695616][PR Manager]]'s merge step.

Never amend commits during this loop — fixes always go in as new
commits so the review history is preserved.
