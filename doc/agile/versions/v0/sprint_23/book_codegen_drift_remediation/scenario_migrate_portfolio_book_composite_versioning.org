* Steps

Each step is its own heading — the title should be short (it's shown
as a single list entry in the QA Validation Runner); put any longer
instructions in the body below the title. The panel writes each
step's PASS/FAIL/PENDING outcome and notes back as a =*** Result=
child heading directly under it.

** Log in and select Barclays Plc

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123=, select
=BARCLAYS PLC=. Confirm all menus enable with no errors.

** Note a portfolio's current version

Open Trading > Portfolios, pick any portfolio, open its History
dialog, and note the current (highest) version number.

** Add a book under that portfolio and confirm the portfolio's version bumps

Open Reference Data > Books, add a new book with Parent Portfolio set
to the portfolio from the previous step. Save. Re-open that
portfolio's History dialog: confirm a new version row appears (one
higher than before), even though no field on the portfolio itself was
directly edited.

** Edit the book and confirm the portfolio bumps again

Edit the new book (e.g. change its description) and Save. Re-check the
portfolio's History dialog: confirm another new version row appeared.

** Delete the book and confirm the portfolio bumps once more

Delete the book. Re-check the portfolio's History dialog: confirm yet
another new version row appeared (the delete-side bump).

** Normal Book and Portfolio CRUD still work unaffected

Confirm ordinary Book and Portfolio list/detail/history/save flows
(unrelated to the parent_portfolio_id FK) behave exactly as before —
no regressions from the composite wiring.

** No error toast or log error during the whole flow

Check the client's log tail
(=build/output/<preset>/publish/log/ores.qt.<colour>.log=) for any
ERROR-level line.
