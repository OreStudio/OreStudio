* When to use this skill

Before writing UI code — for any new screen, dashboard, dialog, or
significant refactor of an existing surface. Don't default to
generic styling; commit to a direction that matches what the
specific interface is for.

* How to use this skill

The design system lives in [[id:6743f6e3-578c-4e4b-9dd6-d5a0df6734c8][UI design principles]] (Jony Ive-level
precision with intentional personality). This skill is the
task-shaped entry that walks you through applying it.

1. /Pick a design direction/ before any code:
   - What does this surface /do/?
   - Who uses it (power user vs. occasional)?
   - What's the emotional job (trust / efficiency / delight /
     focus)?
   - Which personality (precision & density / warmth /
     sophistication / boldness / utility / data) fits?

2. /Commit to a colour foundation/ (warm / cool / pure neutral /
   tinted), a light-or-dark stance, and one accent colour that
   means something.

3. /Pick a layout approach/ (dense grid / generous spacing /
   sidebar nav / top nav / split panels) driven by the content.

4. /Pick typography/ (system / geometric sans / humanist sans /
   monospace influence). Don't default.

5. /Hold the craft floor/ — 4px grid, symmetrical TLBR padding,
   consistent border-radius family, one depth strategy
   (borders-only / single shadow / layered / surface-color
   shifts), motion that aids comprehension. The full rulebook is
   in [[id:6743f6e3-578c-4e4b-9dd6-d5a0df6734c8][UI design principles]]§"Core Craft Principles".

6. /Pick icons via [[id:2A167638-8B5A-4691-90CB-4A6B55785108][icon-guidelines]]/. The icon vocabulary sits on
   top of the design direction; consult the catalogue rather than
   inventing.

7. /Self-review/. Before submitting:
   - Every spacing value lands on the 4px grid.
   - Border radii are in one family.
   - One depth strategy is in play, not three.
   - The accent colour is used /sparingly/, not everywhere.
   - Motion has a reason.
