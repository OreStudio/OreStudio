* What

Add a recipe =how_do_i_build_the_site_locally.org= that documents the
=deploy_site= CMake target: run =cmake --build --preset ... --target
deploy_site=, point out the =.build-site.el= entry point, and describe
what the output directory contains. Mirror the structure of the existing
=how_do_i_serve_the_site_locally= recipe and add a cross-reference
between the two so readers understand the full build-then-serve workflow.

* Why

The serve recipe exists but the build recipe does not. Someone new to the
project can start the server but doesn't know how to regenerate the HTML
from Org sources. The two operations are tightly coupled — you build first,
then serve — so they should be documented together and cross-link each other.

- CMake target: =deploy_site= (see =CMakeLists.txt=)
- Build entry point: =.build-site.el=
