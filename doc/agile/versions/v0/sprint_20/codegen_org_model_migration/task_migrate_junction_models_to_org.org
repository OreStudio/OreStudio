* Goal

Convert the 7 =*_junction.json= files (one per junction table:
compute/app_version_platform, dq/badge_mapping, dq/dataset_bundle_member,
iam/account_party, refdata/{party_counterparty, party_country, party_currency})
into literate =ores.<component>.<name>_junction.org= files under
each owning component's =modeling/= dir. Extend the codegen
pipeline with a new ``ores.codegen.junction`` kind: dedicated
``load_org_junction_model``, filename-suffix dispatch on
``_junction.org``, and exclusion from the generic entity match so
routing is unambiguous.

* Acceptance

- =migrate_junction_to_org.py= one-shot converter committed.
- 7 =ores.<component>.<name>_junction.org= files under each
  owning component's =modeling/=; 7 JSON files removed.
- =org_loader.load_org_junction_model= produces a model dict
  matching the JSON shape consumed by
  =sql_schema_junction_create.mustache=.
- =is_junction_model= accepts the new =_junction.org= suffix;
  =is_domain_entity_model= excludes the same suffix.
- =_is_codegen_entity_org= accepts =ores.codegen.junction= as a
  recognised model type for the per-component =modeling/=
  discovery filter.
- Both =generator.load_model= and =codegen.generate._generate_single=
  dispatch on the new suffix to the junction loader.
- =codegen.sh generate --model X_junction.org --profile sql=
  produces byte-identical output to the JSON path for all 7
  junctions.

* Plan

A small parser fix landed alongside the loader: bullet items
(``- foo``) inside a node's body are now tracked both as a
structured =bullet_lists= entry AND as raw =body_lines=, so
=_strip_body= preserves them in their original position inside
description prose. Two junction descriptions (dq/badge_mapping,
dq/dataset_bundle_member) carry bullet examples that the regen
output needs to render verbatim.
