* Notes

** Summary

- Applies =-fvisibility=hidden= to all 18 shared library targets and removes the =CMAKE_WINDOWS_EXPORT_ALL_SYMBOLS= global flag, fixing the Windows PE 65,535 named-export limit
- Adds =export.hpp= headers (with =ORES_X_EXPORT= / =ORES_X_IMPORT= macros via =BOOST_SYMBOL_EXPORT=) to every library that lacked one
- Annotates all classes and free functions that cross =.so= boundaries with the correct =ORES_X_EXPORT= macro, covering foundation, infrastructure, variability, IAM, DQ, refdata, analytics, assets, compute, controller, marketdata, trading, reporting, scheduler, workflow, synthetic, eventing, telemetry, and ORE domain layers

** Motivation

Windows DLLs cap named exports at 65,535 symbols. The existing =CMAKE_WINDOWS_EXPORT_ALL_SYMBOLS= workaround exports every symbol, which hits that limit as the codebase grows. The fix is explicit visibility control: hide everything by default, export only what crosses library boundaries.
