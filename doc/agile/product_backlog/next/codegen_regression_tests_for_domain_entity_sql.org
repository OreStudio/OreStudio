* What

Add unit/golden tests in =projects/ores.codegen/tests/= covering two
fixes made to =ores.sql.schema.domain_entity_create.org= /
=core.py= while migrating =purpose_type=: (1) a model with an explicit
=change_reason_code= row in its =insert_trigger= validations table
should emit that assignment exactly once, not twice (guarded by
=change_reason_code_declared=); (2) a model with a tenant-scoped
=validation_fn= (=tenant_scope: system/tenant/both=) should regenerate
a =validate_<entity>_fn= body that includes the bootstrap-passthrough
and FK-existence check, not a no-op stub (guarded by syncing
=validation_fn= scope flags from =table= back onto =domain_entity= in
=core.py=).

* Why

Both bugs were silent: the SQL still compiled and ran, it just did the
wrong thing (duplicate assignment; no real validation). They were only
caught by manually diffing regenerated output against a hand-written
file during the =purpose_type= entity migration (PR #1451). Nothing in
=ores.codegen='s test suite currently exercises either code path, so a
future refactor of =generate_from_model= / =normalise_sql_table_context=
could silently reintroduce either regression with no test failure to
catch it.
