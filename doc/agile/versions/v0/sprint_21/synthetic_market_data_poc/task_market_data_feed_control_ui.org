* Goal

Provide the synthetic-specific Qt UI to manage the persisted synthetic market
data configs built in the foundation task — without this UX a config cannot be
created. Scoped to the logged-in tenant/party.

- List the party's synthetic config containers and their typed sub-configs
  (FX spot now; vol surface, IR later).
- Create / edit / delete a container and its FX spot sub-config: source name,
  ore_key, GMM means/stdevs/weights, initial price, ticks/hour, enabled — with
  validation (weights sum, positive ticks/hour, unique source name).
- Lives in the synthetic UI (e.g. =ores.qt.synthetic= / under the Market Data
  or a Synthetic menu), wired to the synthetic config NATS protocol.

This is config authoring only — starting/stopping/monitoring feeds is the
separate, source-agnostic *Market data control panel* task. Depends on the
foundation task (the persisted config entity + protocol).
