* Goal

Apply the gray-is-inactive rule across the badge system. Audit findings:

- =account_type_service= shipped gray in PR #1038 — recolour teal =#14b8a6=.
- =account_unlocked= (positive) and =state_done= (completed) were gray —
  recolour green =#22c55e=.
- Every badge resolver fell back to gray when no definition resolves,
  conflating "definition missing" with "inactive" — introduce
  =color_constants::badge_fallback= (orange =#f97316=) and use it in the
  8 MDI-window resolvers, TenantMdiWindow, AccountItemDelegate and
  DatasetItemDelegate. Semantic grays (No/disabled/Pending/Unsent and the
  =inactive=/=login_never=/=fsm_draft=/=treatment_raw= definitions) stay.
- Account detail dialog showed Online/Locked as plain text — add
  =BadgeLabelUtils= (widget counterpart of the delegate badge painter) and
  render both as DB-driven badges; new =account_online= code domain reuses
  =login_online=/=inactive= definitions.
