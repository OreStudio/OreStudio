* Goal

Add a /revert/ action to the currency history dialog so the user can roll back to a prior version.

* Acceptance

- Revert button on each history row.
- Click prompts for confirmation and then writes a new version equal to the selected past one.
- Audit trail preserved (revert creates a new row rather than deleting subsequent history).

* Plan

Captured during execution; cleared into the parent story on close.

* Notes

Revert-by-write-new-version keeps history monotonic and matches how the repository handles updates.

* Result

Users can roll back any currency without database surgery.
