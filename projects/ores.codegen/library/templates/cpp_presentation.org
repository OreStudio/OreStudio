This page is the literate source for the =cpp_presentation= facet of
the codegen template library. Each section documents one template and
holds its source in a =mustache= block tangled to the sibling
=.mustache= file consumed by =generator.py=. The =.mustache= files are
generated artefacts — edit this document, then run the tangle
(=compass build --direct tangle_codegen_templates= or
=projects/ores.lisp/src/ores-build-codegen-templates.el= directly).
Output paths and profile membership come from
[[id:A8145FD7-0651-40FD-857E-BC37C9320020][facet_catalogue.org]].

* Summary

One template pair: the history field mapper (=presentation= profile),
rendering a domain entity to an ordered, human-labelled field list —
per [[id:85892C17-BB81-4724-9B84-20E21D41C499][History Diff Architecture]] layer 1. Codegen'd, plain code: one
call per field, dispatched on the same per-column type flags every
other domain facet already computes. No runtime reflection.

* The cpp_presentation facet

Server-side rendering, distinct from the domain class ([[id:F1AB97F6-5F1F-4C73-9A2C-B994495B2D22][C++ domain
templates]]) and the repository mapper ([[id:8C52C48E-9661-452F-A951-136EFC7B6B22][C++ repository templates]]) —
this facet's mapper converts a domain value to *display* strings
(=ores::diff::domain::field_value=), not database rows.
