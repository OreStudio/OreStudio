#+title: Promote cross-tenant read to a codegen flag
#+description: read_latest_all_tenants is a generic pattern — promote it to a generated optional via a flag in the entity org model.

This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *next* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

The pattern of reading all records across all tenants (no =tenant_id=
filter, only =valid_to == max=) appears in every entity that has a
system-level background process that must operate across tenant
boundaries.  Currently this is implemented as a paste block in each
entity org model individually.  It should instead be controlled by a
boolean flag in the entity org model (e.g. =#+has_cross_tenant_read:
true=) that causes codegen to emit the method automatically, removing
the boilerplate paste block from each entity.  The generated method
would follow the same naming convention as the other generated reads
(=read_latest_all_tenants=) and appear alongside the other =read_latest=
overloads.

* Why

Paste blocks exist for truly entity-specific logic.  Cross-tenant reads
are structural — the implementation is identical for every entity.
Having the same code in three paste blocks (one per entity that needs
it) violates DRY, introduces copy-paste drift risk, and adds friction
when onboarding new entities that need the same capability.  A codegen
flag is the correct abstraction.
