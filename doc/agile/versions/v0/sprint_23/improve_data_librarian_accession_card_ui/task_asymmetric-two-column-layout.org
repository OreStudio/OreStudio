* Acceptance

- Overview tab: QSplitter (primary ~65% / sidebar ~35%). Primary side:
  Description and Commentary prose cards. Sidebar: Classification,
  Data Governance (badges), Audit cards.
- Provenance & Methodology tab: same split. Primary side: Methodology,
  Business Context, Commentary prose cards, Lifecycle Timeline card,
  Implementation Details text browser. Sidebar: Source, Lineage
  Metrics, Methodology Info, Audit cards.
- Cards are QGroupBox + QFormLayout — the same grouping widget
  ProvenanceWidget's "Record Provenance" box already uses elsewhere,
  not a new bespoke "card" widget.
- Governance badges use =BadgeLabelUtils::apply= on a plain QLabel
  (the AccountDetailDialog precedent for form-based badges), not a
  QTreeWidget delegate — that machinery is removed along with the
  QTreeWidgets.
- Dependencies tab (interactive lineage graph) untouched.
- Manually verified in the running Qt client against a real dataset.

* Plan

Replaced overviewTree_/provenanceTree_/methodologyTree_ (QTreeWidget
property grids) with QSplitter(Qt::Horizontal) in both tabs, ~65/35
stretch factors. Added small card helpers (addPropertyCard,
addProseCard, addFormRow, addBadgePlaceholder) reused across both
tabs. Removed the QTreeWidget-based OverviewBadgeDelegate entirely —
badges are now plain QLabels styled via BadgeLabelUtils::apply.

* Result

Reworked the Overview and Provenance & Methodology tabs into the
asymmetric two-column "Card & Canvas" layout: QSplitter (~65/35),
QGroupBox/QFormLayout cards on the sidebar, prose cards on the
primary side. Removed the QTreeWidget-based property grids and the
=OverviewBadgeDelegate= entirely.

Two review rounds with the running client found real defects:
a near-black header banner background (palette(alternate-base) in
this dark theme) and oversized badges — the latter's true cause
(QFormLayout stretching the field column) only surfaced after a
first attempt at tightening padding/font-size didn't fix it. Also
filed a follow-up task for Dependencies-tab zoom/pan, raised during
QA but out of scope here.

QA Validation Runner scenario now PASSES all 7 steps with no
outstanding notes. The story's original two-phase scope (header/
badges/tab-merge in Phase 1, two-column layout in Phase 2) is
complete; the story stays open only for the zoom/pan follow-up task.
