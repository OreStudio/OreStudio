surfaced, without pre-empting commission stories:

   =refdata_rounding_types_notify_trigger_create.sql= is generated but
   was never in production. Adopt it deliberately or suppress it.

parties, counterparties, etc.) is owned by the individual commission
stories and is explicitly out of scope here.

* Result

Investigation confirmed both originally-reported structural issues were already resolved
before this task started:

- *Channel naming*: production SQL, template, and C++ subscribers all use
  =ores_refdata_<entity>= consistently. The =application.cpp= subscriber cited in the
  task description (=ores_currency_market_tiers=) was corrected in prior work.
- *Net-new rounding_types trigger*: =refdata_rounding_types_notify_trigger_create.sql=
  is already in production HEAD; it was adopted during the currency commissioning work.

Regeneration (=--component refdata --profile sql=) surfaced 16 drifted files in two classes:

1. *Structural trigger improvements* (7 files, adopted): =AUTO-GENERATED= header added;
   =changed_id text= corrected to =changed_id uuid= with direct assignment instead of
   =::text= cast. Both are type-correctness improvements with identical runtime behaviour.
   Channel names unchanged throughout.
2. *Per-entity DDL drift* (8 create + 1 drop file, reverted): missing soft-FK check
   constraints, indexes, and column defaults in books, parties, counterparties, and
   related entities. These are owned by the individual commission stories and were
   reverted without modification.
