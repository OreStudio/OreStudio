* Plan

Created =ores.refdata.book_status.org= (type: =ores.codegen.entity=) following the =ores.refdata.rounding_type.org= pattern with book_status-specific fields, NATS subjects, and Qt metadata. Ran =python3 projects/ores.codegen/codegen.py generate --model projects/ores.refdata/modeling/ores.refdata.book_status.org --profile all-cpp=.

Initial codegen (round 1) generated files into ghost directories:
- =projects/ores.refdata.api/= — standalone project: =domain/=, =generators/=, =messaging/= files (wrong: no CMakeLists.txt)
- =projects/ores.refdata.core/= — standalone project: =repository/= and =service/= files (wrong: no CMakeLists.txt)

Root cause: =generator.py= =resolve_output_path()= substituted ={component_include}= and ={component_core}= literally into file paths, so =refdata.api= produced =ores.refdata.api/= instead of =ores.refdata/api/=. Facet catalogue patterns used ={component_include}= for both the project-root directory segment (where slash form is needed) and the C++ include directory name (where dots must be preserved).

Round 2 (after review): fixed four template/codegen defects:
1. =generator.py=: added =_component_to_path()= helper; introduced separate ={component_include_path}= / ={component_core_path}= placeholders for the project-root segments; updated =facet_catalogue.org= to use them. ={component_include}= and ={component_core}= remain unchanged (dot form) for the C++ include directory names.
2. =cpp_service.cpp.mustache=: added =#include "ores.service/messaging/handler_helpers.hpp"= and =using stamp;= before namespace; introduced =auto t = v; stamp(t, ctx_); repo_.write(ctx_, t);= pattern in single-save; stamping loop in batch-save.
3. =cpp_domain_type_generator.cpp.mustache=: captured =counter.fetch_add= once as =const auto idx= inside the ={{#primary_key.is_text}}= block; replaced all natural-key =counter.fetch_add= calls with =idx= so PK and natural keys share the same counter increment.
4. =cpp_domain_type_repository.cpp.mustache=: changed all 8 =static auto max(...)= to =static const auto max(...)=.

Each =.mustache= fix was applied in tandem to its canonical =.org= archetype source.

Deleted all ghost files, re-ran codegen: all 18 files land in =ores.refdata/api/= and =ores.refdata/core/= (correct CMakeLists.txt trees). Reconciled diffs against existing hand-written files — accepted generated output in full: include ordering, =static const= max, stamp calls, single counter, and =tenant_id= removal are all improvements. Updated =book_status_handler.hpp= to align with changed protocol field names (=book_statuses= → =statuses=, =status= field → =code=/=codes=, =history= → =statuses=). Second codegen run confirms idempotency (zero further diff).
