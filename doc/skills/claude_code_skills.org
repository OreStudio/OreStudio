Every Claude Code [[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skill]] used with ORE Studio lives at
=doc/skills/<slug>/SKILL.org= and is listed below, grouped by area. Skills are
/task-shaped/: each one is the playbook an LLM session loads to perform a
specific unit of work end-to-end. Compare with [[id:66f2a9e8-0385-4d56-b4b1-0957208990dc][function]], which is /role-shaped/.
See [[id:654d4a70-e63d-4c18-b5a5-886066e36314][CMake Runner]] for the deployment workflow or return to [[id:7474935C-95D1-3034-B833-83DCD9FB3A09][Knowledge]].

  prioritisation, sprint planning.

- [[id:b0b99088-d88e-4980-a4ee-7d6a9a8e96a5][Component Model Creator]] — component model documentation in

  schema.
- [[id:C5EEF144-BB9D-2F44-453B-4B7338D8A457][New Sprint]] — create new sprint backlogs.
- [[id:DD7857D9-068E-D724-5C63-A67BD5F1C009][Release Notes Generator]] — generate release notes for a sprint.
- [[id:28aaf9a6-b1e4-402a-83bb-289090990fb8][Recipe Author]] — create new recipe files and recipe-updating
  skills.
- [[id:b5766b0e-24f0-45c3-b35d-e762bbab5206][ORES Recipe Sync]] — keep recipe documentation in sync with its
  source (CLI parsers, HTTP routes, shell commands, etc.).
  Replaces the per-topic recipe-sync skills.

* Rebuilding skills

When skills change, redeploy them via the =deploy_skills= CMake
target. See [[id:b5f231a9-5403-482c-8f2a-12d6917c58cd][How do I deploy the skills?]] for the recipe and the
CI hook that calls it.
