** Stop every feed, then "enable all" from the root/folder

Open the Market Simulator window. If any feeds are already running,
select each and stop it (or select the root node and use the
folder-scoped stop) so every pair starts from a clean stopped state.
Then select the root/collection node covering all of Barclays' FX
pairs and start it (folder-scoped "enable all").

*** Result

| Field  | Value |
|--------+-------|
| Status | PENDING |

** Confirm every pair, not just one, is bound

After "enable all" completes, open Market Data > Feed Bindings and
confirm there is one feed_binding row per FX pair that was started
(not just a single pair like USD/AUD from before the fix). Cross-check
against the CRM Cross Rates Matrix: every started pair should show a
live rate, not a dash, within a few ticks.

*** Result

| Field  | Value |
|--------+-------|
| Status | PENDING |

** Confirm the running-status badge colours are distinct from validation badges

With some feeds running and some stopped/paused, look at the tree's
per-node badges: the bottom-left running-status badge should be
*blue* for fully running and *yellow* for paused/stopped -- clearly
different from the bottom-right vintage-validity badge, which stays
green (valid) or red (invalid). Confirm the two badge families are no
longer confusable (previously running=green clashed with
vintage-valid=green).
