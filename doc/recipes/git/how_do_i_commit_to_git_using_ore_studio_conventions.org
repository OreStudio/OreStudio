ORE Studio uses a structured commit format to ensure that the project history
remains readable, greppable, and attributable.

* Question

How do I format my git commits to follow the ORE Studio project conventions?

* Answer

Every commit should consist of a structured title, an optional but encouraged
detailed body, and a mandatory co-author attribution when generated by an LLM.

** 1. Commit Title

The title (first line) must follow this pattern:

: =[component] <Imperative summary>=

- **Component**: A short, lowercase slug in square brackets identifying the
  primary area of change (e.g., =[doc]=, =[agile]=, =[site]=, =[qt]=,
  =[vcpkg]=).
- **Imperative Mood**: Use the imperative ("Add feature", "Fix bug") rather than
  descriptive ("Added feature", "Fixed bug").
- **Length**: Keep the title under 72 characters.

Examples:

- =[doc] Tidy-up documentation around sprint 16 to 18=
- =[site] Inline org-roam export utilities into ores.lisp=
- =[agile] Open Sprint 18=

** 2. Commit Body

The body is separated from the title by a blank line. It should explain the
*what* and *why* of the change, especially for complex implementation details.

- Wrap text at 72 characters.
- Use bullet points for multiple distinct changes within a single commit.

** 3. Co-Author Attribution

When a commit is produced by an LLM (directly or via an agent), it must include
a =Co-Authored-By:= trailer at the end of the commit message.

- **Self-Identification**: The LLM must use its own name and provide its own
  associated email address (do not hard-code "Claude" if you are not Claude).
- **Format**: =Co-Authored-By: <Model Name> <email>=

Examples:

- **Gemini**: =Co-Authored-By: Gemini 2.0 Flash <noreply@google.com>=
- **Claude**: =Co-Authored-By: Claude 3.7 Sonnet <noreply@anthropic.com>=
- **GPT**: =Co-Authored-By: GPT-4o <noreply@openai.com>=

Example commit message:

#+begin_src text
[site] Serve real page content in knowledge graph node panel

- ws-shim: intercept node/{id} and return HTML content.
- .build-site.el: reset body flex/padding.

Co-Authored-By: Gemini 2.0 Flash <noreply@google.com>
#+end_src

* Script

#+begin_src sh :results verbatim
# Typical commit command
git commit -m "[comp] Title" -m "Detail body." -m "Co-Authored-By: Your Identity <your@email.com>"
#+end_src

* Tested by

- Manual review of `git log` output.
- CI linting of commit messages (where applicable).
