* Question

How do I see a summary of the last test run — pass/fail counts, failure
details, and any ERROR/WARN log lines?

* Answer

1. /Use the preset from =.env=/ (auto-detected):

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh test results
   #+end_src

2. /Specify a preset explicitly/:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh test results --preset linux-clang-debug
   #+end_src

   =compass test results= reads =ORES_PRESET= from =.env= when =--preset=
   is omitted. It finds all =test-results*.xml= files under
   =build/output/<preset>/publish/bin/=, prints per-suite statistics
   (total, passed, failed, skipped, duration), details of each failure
   (name, location, exception text), and any =ERROR=/=WARN= lines from
   the matching log files.

3. /If you see no log output/ — test logging is disabled by default.
   To enable it, reconfigure CMake:

   #+begin_src sh :results verbatim
   cmake --preset linux-clang-debug-make -DORES_TEST_LOG_LEVEL=debug
   ./compass.sh build rat --preset linux-clang-debug-make
   ./projects/ores.compass/compass.sh test results
   #+end_src

   For trace-level output also printed to the console:

   #+begin_src sh :results verbatim
   cmake --preset linux-clang-debug \
     -DORES_TEST_LOG_LEVEL=trace \
     -DORES_TEST_LOG_CONSOLE=ON
   #+end_src

* Script

=compass test results= lives in [[proj:projects/ores.compass/src/compass.py][=projects/ores.compass/src/compass.py=]]
(=cmd_test= + =_cmd_test_results= + =_tr_*= helpers).

* Tested by

Manual: build and run tests with =--target rat=, then run
=compass test results= and verify suite stats and failure details appear.
