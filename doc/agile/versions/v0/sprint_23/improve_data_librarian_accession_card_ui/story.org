* Goal

Redesign the Data Librarian's dataset detail dialog
(=DatasetDetailDialog=, the "accession card") away from its current
single-column, alternating-row property-sheet table toward a
persistent-header, asymmetric two-column layout, per
[[id:211CB159-F633-4897-A899-56CD91625396][OreStudio Data Librarian UI/UX Review]]. Phased: cosmetic wins
(header banner, UUID copy, status pills) before the structural
column/tab rework. The interactive lineage graph proposed for
*Dependencies* is out of scope — a separate, bigger undertaking.

* Acceptance

- A persistent header banner replaces the per-tab repeated
  =Name=/=Version=/=ID=/=Code= fields, with a one-click UUID copy
  action.
- Status identifiers (=Primary=, =Actual=, =Raw=, etc.) render as
  visual pills instead of plain text.
- *Provenance* and *Methodology* tabs are merged into a single tab;
  lineage primitives (=Lineage Depth=, =Upstream Derivation=) are
  visually prominent rather than buried at the bottom of a property
  grid.
- The dialog uses an asymmetric two-column layout (primary content +
  metadata sidebar) instead of a single-column table, for *Overview*
  and the merged *Provenance & Governance* tab.
- =DetailDialogBase= (shared by 78 other entity dialogs) is
  untouched; the new layout lives in =DatasetDetailDialog= only.
