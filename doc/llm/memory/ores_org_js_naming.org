The browser-based org-mode UI project lives at projects/ores.org-js/. Do not name anything 'dashboard' in this repo — ores-dashboard.el (an emacs mode in projects/ores.lisp/src/) already owns that name.

*Why:* Marco chose ores.org-js (2026-06-06) over ores.agile because the org parsing/rendering part may be generalised beyond agile. 'dashboard' is already taken by an emacs mode.

*How to apply:* When referring to or extending the browser UI: use ores.org-js. Never introduce a file or component named dashboard.
