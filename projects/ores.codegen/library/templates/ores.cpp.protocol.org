* Summary

One template: request/response message types for an entity's NATS
API. =protocol= profile. Protocol class names arrive as explicit model
fields rather than being derived from naming conventions.

* The ores.cpp.protocol facet

The wire-format seam between an entity and the messaging fabric:
protocol message types are what travels between the client and the
per-entity service. Sibling to [[id:4B13E23E-FD4D-439D-A8BF-73D0C0FE0415][ores.cpp.nats-handler]] (which dispatches
these messages) and [[id:FC3A81FE-54F6-4119-8369-A7B8EF423E47][ores.cpp.service]] (which executes them against the
repository).
