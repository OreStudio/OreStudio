* Goal

Establish a documented, developer-ready approach for generating a single consistent synthetic market data environment that covers all ORE-supported asset classes, can price all products in the ORE Examples suite, and can be evolved through time in a way that maintains cross-asset consistency. The work is primarily analysis and documentation: summarise three academic papers in developer language, catalogue the ORE sample data requirements, and synthesise the findings into an actionable approach document.

* Decisions

- Three papers chosen as the primary input: Gaussian GenAI (SSRN 5050372), mixture-preserving arbitrage-free vol-surface interpolation (arxiv 2606.12717), GJR-GARCH NN option pricing (arxiv 2606.15502).
- Scope of the ORE Examples suite: =external/ore/examples/= — all subdirectories. The Products example is the broadest single portfolio and covers the widest asset-class spread.
- Output is documentation only in this story; no implementation code.
- Different generation techniques per asset class are acceptable, but cross-asset constraints are mandatory: e.g. FX vol surfaces must be consistent with FX spot rates; IR curves used in discounting must be consistent with those used in FX forward pricing (covered interest parity); equity vol surfaces must be consistent with equity spot levels. The approach document must enumerate these consistency constraints explicitly, not just describe per-asset generation methods in isolation.

* Out of scope

- Actual implementation of any generation algorithm.
- Calibration to real market data or historical time series.
- Full stochastic-vol model calibration (e.g. Heston, SABR parameter fitting).
- Coverage of ORE scripted trade types beyond what is needed to illustrate the approach.
- Any UI or database changes.
