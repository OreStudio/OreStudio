* Goal

The flat nav bar in =ores-build-site.el= currently lists twelve items in a
single row. As the number of sections grows, the bar becomes cluttered and
new pages cannot be exposed without making it worse. Replace the flat row
with a hierarchical dropdown menu — existing items reorganised into logical
groups, no page content changed — so future sections can be added cleanly.

* Acceptance

- [x] =site-html-preamble= in =ores-build-site.el= is replaced with a
  dropdown menu structure using semantic HTML (=<ul>= / =<li>= with
  =hover= or =focus= driven submenus).
- [x] All existing nav items (Home, Orientation, Product, Architecture,
  LLMs, Manuals, Downloads, Agile, Developer, Roadmap, Knowledge Graph,
  GitHub) are present under the new menu, reorganised into logical groups.
- [x] Dropdown CSS lives in =assets/style.css=; no inline styles added.
- [x] The graph-page injection in =ores-inject-site-nav= is updated to use
  the same markup.
- [x] Site builds and deploys cleanly (=cmake ... --target deploy_site=).
- [x] Menu is usable without JavaScript (CSS-only =:hover= / =:focus-within=).
