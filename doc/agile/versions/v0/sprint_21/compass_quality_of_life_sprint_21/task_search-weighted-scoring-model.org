* Goal

Replace the ad-hoc =core=/=full=/=body= label system with a formal,
transparent scoring pipeline. A single =score_document()= function
takes a pre-computed query plan and a set of document signals, applies
explicit named weights, and returns both a percentage score and a
per-signal breakdown. Results below a configurable threshold are
suppressed. The scorer must be independently testable against a JSON
corpus fixture so the algorithm can be evaluated and tuned without
running a full search.

* Result

Delivered across commits =fcd2bcb7d=–=89419d7ca= on
=feature/compass-quality-of-life-sprint-21=.

The ad-hoc =core=/=full=/=body= label system was replaced by a formal
relevance pipeline in =search_scorer.py= (no live-DB dependency, 21
unit tests):

- =QueryPlan= tokenises the query, strips stopwords, and splits core
  words (question verbs removed) from full words.
- =DocSignals= names every raw signal: core-words title rank, full-words
  title rank, body BM25 proxy (pool position), inbound-link count,
  recipe-question bonus.
- =Weights= holds every coefficient and normalisation constant — one
  place to tune the algorithm.
- =score_document()= applies RRF (k=60) normalisation and returns
  =ScoreResult= with =.pct= (0–100) and =.breakdown= dict.
- =global_floor()= computes =max(absolute_floor, max_score × dropout_ratio)=
  as a single global threshold applied before bucketing so a strong
  Recipes match suppresses weak Knowledge hits.
- =precision_at_k()= / =ndcg_at_k()= enable corpus evaluation without
  any external deps.

=ui.py= gained =BUCKET_AFFINITY=: a doctype → bucket mapping read by
=cmd_search= so new doctypes are automatically classified.

=compass.py= changes:
- Score shown as =[74%]= per hit.
- Global relative dropout (=0.25=) replaces fixed threshold.
- =--floor PCT= and =--dropout RATIO= flags expose the constants for
  live tuning without editing source.
- =--history= excludes past-sprint stories/tasks by default.
- =--all-buckets= disables the relative dropout.
- Heading-level anchors (=olp= non-empty, no =#+type:=) excluded by
  default — parent body FTS covers their content.
- Configuration block at top of output shows active sprint, floor,
  dropout, and override flags.
- Tags line (=🏷=) and parent-path line (=📍=) added to each hit.
- Doc index loaded once; sprint prefix reused across passes.
