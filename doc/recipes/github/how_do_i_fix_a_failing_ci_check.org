* Question

A check on my PR has gone red. How do I find what failed, fix it,
and push the fix?

* Answer

1. /Identify the failing check/. =gh pr checks= shows the overall
   status; =gh run list= drills into the runs for the branch:

   #+begin_src sh :results verbatim
   gh pr checks
   gh run list --branch "$(git branch --show-current)"
   #+end_src

2. /Read the failure log/. The =--log-failed= flag prints only the
   failing step's output:

   #+begin_src sh :results verbatim
   gh run view <run-id> --log-failed
   #+end_src

3. /Diagnose by category/:

   - /Compilation error/ → fix the source file.
   - /Test failure/ → fix or update the failing test.
   - /Linting / formatting/ → apply the required formatting.
   - /Schema or doc validation/ → run the local validator
     (=projects/ores.codegen/validate_docs.sh=,
     =doc_version_audit.py --check=, etc.) and fix what it flags.

4. /Apply the fix as a new commit/:

   #+begin_src sh :results verbatim
   git add <modified-files>
   git commit -m "[component] Fix CI failure: <brief description>"
   git push
   #+end_src

   *Never amend* — preserves review history.

5. /Wait for the rerun/. CI re-triggers on push; return to [[id:F6E88DCC-9240-4632-A9A3-9F36F79C9262][monitor
   until green]].

* Script

Bare =gh= CLI plus the language-specific local validators that
mirror what CI runs.

* Tested by

Manual. Validators that ship with the repo are exercised in CI;
running them locally is the fastest way to reproduce.
