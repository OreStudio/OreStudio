* Goal

Make =compass heading= environment-aware in two ways:

1. *Visibility* — show a summary of what each other environment is
   currently working on (story title + task title, read from
   =#+environment:= frontmatter stamped at =task start=).  This gives
   every environment a live map of who is where without running =compass
   fleet= across worktrees.

2. *Scoring* — penalise recommendations that would conflict with other
   environments:
   - *Direct ownership*: a story already stamped with another
     environment's label gets a heavy score penalty (or is suppressed
     outright with a note).
   - *Topical overlap*: even if a story is not directly owned, if its
     tags/title share significant keywords with what another environment
     is currently working on, apply a softer penalty and surface a
     warning — the environments may be heading toward the same area and
     coordination is warranted.

The current environment is identified from =ORES_CHECKOUT_LABEL= in
=.env=.  Other environments are discovered by scanning sprint stories for
=#+environment:= values that differ from the current label.

* Result

Delivered on =feature/heading-respect-environment-ownership=.

Two additions to =cmd_heading= in =compass.py=:

*Visibility section*: Before the ranked list, an "Other environments in
this sprint" block shows each non-current environment alongside the story
(and current task if STARTED) it has stamped with =#+environment:=.  Only
non-DONE stories with an =#+environment:= field appear.  When no other
environment is active the section is omitted.

*Environment scoring*:
- *Direct ownership* (story's =#+environment:= ≠ =ORES_CHECKOUT_LABEL=):
  multiplier 0.15 — the story is effectively off-limits.  It still appears
  in the list with a "[owned by X]" note so the conflict is visible.
- *Topical overlap* (story title shares tokens with any other env's active
  story): multiplier =max(0.5, 1.0 − 0.1 × overlap_count)= — soft penalty
  per shared keyword, with a "[possible overlap with other env: ...]" note.
- Both cases use a per-story =_env_multiplier= / =_env_note= pair applied
  uniformly to all suggestions from that story.

Also on this branch: =_cmd_task_start= now restores the task file (and
=story.org=) from =origin/main= when it is missing from the task branch,
handling the case where =task new= created the branch before the file was
merged to main.
