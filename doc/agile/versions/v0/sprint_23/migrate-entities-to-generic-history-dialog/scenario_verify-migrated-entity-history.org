* Steps

** Log in and select the Barclays Plc party

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123= and
select "BARCLAYS PLC" as the active party if prompted. If the
database has been recreated since this was last provisioned, first
run =./compass.sh shell -f
projects/ores.shell/scripts/library/provisioning/barclays_system_provision.ores=
to reprovision it.

** Check book history

Reference Data > Books. Open an existing book's history (or create
one, edit it, then open history). Confirm the History Timeline shows
its versions and the diff pane renders without a "No history provider
registered" error.

** Check CDS convention history

Reference Data > CDS Conventions. Open an existing convention's
history. Confirm it loads correctly — this entity was one of the ones
needing an explicit =history_response_data_field= override
(=cds_conventions=, not the default =history=).

** Check tenor history

Reference Data > Tenors. Open an existing tenor's history. Confirm it
loads correctly.

** Check currency group history

Reference Data > Currency Groups. Open an existing currency group's
history. Confirm it loads correctly.

** Check calendar history

Reference Data > Calendars. Open an existing calendar's history.
Confirm it loads correctly.

** Check Open and Revert still work on one entity

Pick any one of the five entities above. With a single version
selected in its History Timeline, click Open — confirm the read-only
detail view opens correctly. Close it, select an older version, and
click Revert — confirm the confirmation dialog appears and (on
confirming) the editable revert dialog opens pre-populated.
