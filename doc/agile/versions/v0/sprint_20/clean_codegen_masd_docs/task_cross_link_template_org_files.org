* Goal

The codegen template org docs (=projects/ores.codegen/library/templates/=)
and the modeling facet reference docs (=projects/modeling/*_facet.org=)
described the same facets but were not linked, so the org-roam graph had
two disconnected islands. Cross-link them bidirectionally: each template
facet/group doc links the MASD concept, Applied MASD, and its modeling
facet reference; each modeling facet links back to the literate
templates that generate it. Where a template facet had no modeling
facet doc, create one with a suitable name.

The template library and the modeling facet docs are now one connected
graph.

- *New modeling facet docs* for the four template facets that lacked
  one: [[id:FC0E2DCC-AF2E-4C48-8984-651F5D9A7EB1][Build facet]] (CMake), [[id:FE5C921E-ADA2-4009-BD96-D66C86AEB207][Assets facet]], [[id:1A82F89B-E7B6-4DDA-8C9B-A144C4DC8F04][Documentation facet]] and

  Applied MASD and its template; all wired into the Applied MASD facet
  catalogue.
- *Template → facet links*: every template facet doc (14) and group doc
  (5) gained See-also links to the MASD concept, Applied MASD, and its
  modeling facet reference (Type definition / Qt / SQL / Build / Assets
  / Documentation / Service application).
- *Reciprocal facet → template links*: the modeling facets link back to
  the literate templates; =type_definition_facet= gained a See-also
  section it previously lacked.
- All inserted ids were verified to resolve (no dangling links).
