A /facet/ is a named capability or role that a [[id:4F24EEB9-58F1-42FD-BF4F-121D890F2D85][logical entity]] plays in
[[id:787A7429-71D5-4295-829B-A9B56275B1F8][physical space]]. Each facet generates a coherent set of artefacts serving
a specific, bounded purpose. Enabling or disabling a facet via the
[[id:CE95C751-2125-4BA9-A78D-C30F9540FC5E][variability model]] controls whether those artefacts are produced at all —
and for which entity types.

In the four-level physical-space address:

: [technical space].[part].[facet].[archetype]

the facet is the third segment. A facet is always owned by exactly one
[[id:E7B74138-9ADA-4B4B-A983-23B3FDB55F44][technical space]] and contains one or more archetypes (the individual file
templates).

See [[https://mcraveiro.github.io/dogen/docs/masd_reference_implementation.html#mri-dogen-architecture][Dogen Architecture — facets ↗]] for the formal treatment.

* SRPP: what a facet encodes

Before writing any template, the analyst identifies a /Schematic and
Repetitive Pattern in Physical Space/ (SRPP) — a class of infrastructure
code that varies predictably across entities but is otherwise mechanically
identical. An SRPP is what a facet encodes; each Mustache template in the
facet is the mechanical extraction of one such pattern.

In ORE Studio, commissioning the =currency= entity produced the canonical
SRPP catalogue for reference-data entities: the C++ domain struct with
rfl-based JSON I/O, the repository entity/mapper/CRUD trilogy, the NATS
service and protocol types, the SQL DDL and trigger scripts, and the full
Qt UI (12 files: MDI window, controller, detail dialog, history dialog,
client model).

* Archetypes

Each facet contains one or more /archetypes/ — templates for individual
files. An archetype produces exactly one output file per [[id:4F24EEB9-58F1-42FD-BF4F-121D890F2D85][logical entity]].
An archetype is parameterised by the logical model (entity name, fields,
namespace, component path). In ORE Studio, every Mustache =.mustache=
file is an archetype.

In ORE Studio the literate template system wraps each archetype in an
org-mode document. The =.mustache= files are tangled from these documents;
the prose surrounding the tangle block is the archetype's documentation.
