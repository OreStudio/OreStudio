#+level: cross

Every =.org= file in the repo must carry =#+version: 2=. See [[id:8B29A8D8-E004-4E87-B81D-ABEAF68F2B98][Document types]]
for the frontmatter contract.

* Question

How do I check which org files are missing a version tag?

* Answer

#+begin_src sh :results verbatim
# Summary: counts and % tagged
python3 projects/ores.codegen/scripts/doc_version_audit.py

# CI gate — exits non-zero if any .org file is missing #+version:
python3 projects/ores.codegen/scripts/doc_version_audit.py --check

# Stamp newly-untagged files as #+version: 2 (idempotent)
python3 projects/ores.codegen/scripts/doc_version_audit.py --stamp
#+end_src

Run =--check= before committing. Run =--stamp= after adding files that bypass
=generate_doc.sh= (the script already stamps them =2=).

* Script

=projects/ores.codegen/scripts/doc_version_audit.py= — standalone Python
script; no external dependencies.

* Tested by

CI gate: =validate_docs.sh= calls =doc_version_audit.py --check= as its
second phase.
