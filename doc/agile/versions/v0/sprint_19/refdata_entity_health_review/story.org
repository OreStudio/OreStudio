* Goal

All =ores.refdata= Qt entities are verified as working correctly after the NATS microservices
migration (load, edit, save, event notifications, history). Regressions found during
verification are fixed inline. A reusable entity evaluation process doc is produced — a
recipe or knowledge page describing the standard steps for evaluating any entity's health —
so the same procedure can be applied to other components in future sprints.

* Acceptance

- All =ores.refdata= entities with Qt support exercised: load list, open detail, edit a field, save, confirm change in history.
- Event notification verified: change in one client session appears in a second connected session.
- All regressions found during verification are either fixed inline or filed as captures.
- A process doc (recipe or knowledge page) exists that documents the standard entity evaluation steps.
- Process doc is referenced from [[id:A3F8C2D1-7E4B-4F9A-B2C5-8D1E6A3F0B7C][Entity Coverage Matrix]] as the evaluation procedure.
