This page is part of the [[id:D773166D-0C91-8CB4-3323-42166BC07687][System Model]]. Infrastructure owns the transport and
testing contracts. Everything above this layer speaks [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]]; this layer
provides the client, the service runner, the event bus, and the test fixture
that wires them together. See [[id:608675E1-4284-4902-99AC-4DA565390AFB][Entity lifecycle]] for how [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] entities flow
across request/reply pairs.

* Detail

** ores.nats

[[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] messaging backbone — client connection management, subject naming
conventions, queue groups, and JetStream configuration and lifecycle. The
primary inter-service transport used by every domain and application component.
See [[id:D9B0988E-D783-43D6-BCCD-CE2F70022B9C][NATS]] for the technology overview.

|---------------+--------------------------------------------------+---------------|

** ores.service

Shared [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] domain service runner — the =domain_service_runner= template
that gives every service a standard request/reply lifecycle: connect, subscribe,
handle, reply, reconnect on failure.

** ores.testing

Catch2 integration that isolates every test into its own PostgreSQL schema. Each
test process creates a unique schema, wires up a preconfigured logger, and tears
down cleanly — making tests safe to run in parallel without touching production
data.

| Sub-component | Role                                 | Dependencies                |
|---------------+--------------------------------------+-----------------------------|
| [[id:7F58FF13-5A85-49C4-09BB-EED86B874ABB][ores.testing]]  | Per-test schema isolation for Catch2 | [[id:3A8F2E91-D4C7-8AB4-5E23-91D6F8C04A72][ores.database]], [[id:1FBCC86B-8E04-4F81-8558-85C0B6C54836][ores.logging]] |

Related documents:

** ores.eventing

In-process publish/subscribe event bus with PostgreSQL-backed delivery and
optional [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] integration. Decouples components within the same process
from the [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] transport layer.

| Sub-component | Role                                         | Dependencies             |
|---------------+----------------------------------------------+--------------------------|
| [[id:DDBC94C4-F350-478B-9E4C-643DEEA803B0][ores.eventing]] | In-process event bus with PostgreSQL backing | [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]], [[id:3A8F2E91-D4C7-8AB4-5E23-91D6F8C04A72][ores.database]] |

** ores.connections

Client-side server connection bookmarks with hierarchical organisation and
credential management. Stores [[id:C9C24C99-F16C-45BE-A262-1C0F4502765E][ores.nats]] connection parameters — host, port,
credentials — in an encrypted local SQLite database.
