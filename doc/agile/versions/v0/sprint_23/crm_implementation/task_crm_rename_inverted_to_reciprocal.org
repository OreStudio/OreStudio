* Goal

CRM's 1/rate backfill (a pair with no direct quote served from its
reverse pair's computed 1/rate) is currently named "inverted"
everywhere -- =crm_rate_item.inverted=, the Qt "Show Inverted"
toggle, the =--inverted= shell flag, =rate_inverter=, tooltip text
"Computed inverse (1/rate)". "Reciprocal" is the correct FX-market
term for this (the reciprocal of EUR/USD is USD/EUR) and cleanly
distinguishes it from *derived*/triangulated rates (this codebase's
existing term for cross-rate computation via an intermediate
currency -- see =crm_rate_item='s own doc comment) -- a different
concept "inverted" was at risk of being confused with. Rename the
concept end-to-end: user-visible strings, code identifiers, the
=get_crm_rates_request=/=crm_rate_item= protocol field, and every
doc/recipe/task that names it.

Scope note: the =inverted= field is on a NATS wire protocol
(=marketdata.v1.crm.rates=) with a boost/rfl JSON payload -- renaming
it is a breaking wire-format change for anything that has already
captured a payload sample (docs, recipes with =#+RESULTS= blocks),
but not a DB schema change (nothing persists an "inverted" column).
No external/third-party consumer of this subject exists yet, so a
straight rename is safe rather than needing a deprecation shim.
