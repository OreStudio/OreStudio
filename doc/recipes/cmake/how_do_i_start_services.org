#+title: How do I start the ORE Studio services?
#+description: Start NATS and all backend services for a local checkout with compass services start; stop and check status with the companion verbs.
#+type: recipe
#+level: cross
#+filetags: :cmake:build:services:nats:controller:recipe:
#+created: 2026-05-31
#+updated: 2026-07-12

Requires =.env= and built binaries (see [[id:49F7DA81-ADB7-44ED-BE7E-06DBFBB32A84][How do I set up a development environment?]]).
Start order: NATS server → controller; the controller spawns all other
services (IAM, domain services, HTTP, Wt, compute wrappers) in dependency order.

* Question

How do I start all ORE Studio backend services for a local checkout?

* Answer

1. /Ensure the environment is initialised/ and the project is built (see
   [[id:1794B513-877B-4B93-A818-53F080C55E47][How do I initialise the checkout environment?]] and [[id:F176FCA1-68D5-420D-B076-75A65FEE5EBB][How do I build the system?]]).

2. /Start services/:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh services start
   #+end_src

   The preset defaults to =ORES_PRESET= from =.env=. Override with =--preset=:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh services start --preset linux-clang-debug-make
   #+end_src

   Change log verbosity with =--log-level= (default: =trace=):

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh services start --log-level info
   #+end_src

   The script blocks until the controller logs =All services started=, then
   exits. Its own progress output (not the per-service logs below) also
   streams live to =/tmp/ores_<env>_services_start.log= (overwritten each
   run) — =tail -f= that path from another shell to watch startup without
   waiting on this command's own output.

** Check status

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh services status
   #+end_src

** Stop all services

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh services stop
   #+end_src

** Logs

Logs land in =build/output/<preset>/publish/log/=. Each service writes its own
=<name>.log= file; the NATS server writes =nats-server.log=.

** Prerequisites checklist

- =.env= exists (run =compass env configure= if not — see [[id:1794B513-877B-4B93-A818-53F080C55E47][How do I initialise the checkout environment?]]).
- Database initialised (=compass db recreate -y=).
- Project built (=./compass.sh build --preset <preset>=).
- =nats-server= on =PATH= or in =/usr/sbin=.

* Script

=compass services start= — starts NATS then the controller; blocks until all services are up.
=compass services stop= — stops all services cleanly.
=compass services status= — shows running/stopped state per service.

* Tested by

Manual, each development session. No CI path — CI uses pre-configured runners.
