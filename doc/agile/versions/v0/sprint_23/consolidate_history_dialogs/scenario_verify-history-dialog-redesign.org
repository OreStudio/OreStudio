* Steps

** Log in and select the Barclays Plc party

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123= and
select "BARCLAYS PLC" as the active party if prompted. If the
database has been recreated since this was last provisioned, first
run =./compass.sh shell -f
projects/ores.shell/scripts/library/provisioning/barclays_system_provision.ores=
to reprovision it.

** Create a currency with three versions

Reference Data > Currencies > Add. Add a currency with ISO code
=TST=, then edit it twice more (e.g. change the Name, then the
Symbol) so it has three versions. Use change reason code
=system.test=.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Check the master-detail layout

Open =TST='s history. Confirm the dialog shows a left-hand version
timeline (~25-35% width) and a right-hand diff pane (~65-75% width)
— no dead space, and no separate Changes/Full Details tab row.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Check the live diff pane on selection

Click the newest version in the timeline. Confirm the diff pane
header reads "Comparing: Version X -> Version Y" and immediately
shows that version's changes, coloured (red old / green new) with
the changed characters highlighted darker.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Check the All Fields / Only Changes toggle

With a version selected, click "Only Changes" — confirm unchanged
fields disappear from the diff pane, leaving only the rows that
actually changed. Click "All Fields" — confirm unchanged fields
reappear as plain (uncoloured) rows alongside the changed ones.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Check two-version compare mode

Click the oldest version, then Ctrl-click the newest version (with
at least one version in between). Confirm the header updates to
"Comparing: Version 1 -> Version 3" (or similar) and the diff pane
shows the net change across both versions directly, not just the
adjacent single-step diff.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Check Open/Revert are disabled during compare

With two versions selected (from the previous step), confirm Open
and Revert are disabled in the toolbar. Click back down to a single
version and confirm they re-enable.

** Check Open and Revert still work

With a single version selected, click Open — confirm the read-only
detail view opens correctly. Close it, select an older version, and
click Revert — confirm the confirmation dialog appears and (on
confirming) the editable revert dialog opens pre-populated.
