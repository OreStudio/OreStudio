* Result

Delivered in commit =a8aa6517a= on =feature/fix-task-new-mid-scaffold-branch-switch=.

Root cause: =_scaffold_and_branch= always called =git switch -c branch base=,
leaving the worktree on the newly created branch regardless of whether it was
a =story new= or a =task new= on an existing story.

Fix: split the branch-creation step by call site.  =story new= still does
=git switch -c= (the scaffold files must land on the story branch and
=_cmd_task_start= runs immediately after).  =task new= on an existing story
now calls =git branch branch base= instead — creating the branch pointer
without switching — so the worktree stays on the caller's branch.  The next
step hint was updated accordingly (=compass task start= handles the switch when
the user picks up the task).
