* Goal

Move dashboard data sourcing from ad-hoc queries to a TimescaleDB-backed telemetry pipeline.

* Acceptance

- Two new TimescaleDB hypertables: ores_compute_grid_samples_tbl (server-side) + ores_compute_node_samples_tbl (per-node), 30-day retention.
- compute_grid_poller in ores.compute.service: async Boost.ASIO coroutine sampling grid-wide metrics every 30s.
- node_stats_reporter in ores.compute.wrapper: per-task timing + byte transfer accumulator, publishes node_sample_message to NATS every 30s.
- get_grid_stats NATS request/reply replaces multiple ad-hoc queries with a single efficient call.
- ComputeDashboardMdiWindow consumes the unified endpoint.

* Notes
