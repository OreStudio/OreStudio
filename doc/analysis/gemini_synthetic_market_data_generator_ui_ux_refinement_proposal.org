* Core UX Pitfalls
Right now, the configuration dialog hits a classic roadblock: attempting to serve two drastically different audiences (novices vs. advanced quants) with a single, highly mathematical interface.

The interface suffers from a conceptual mismatch between the stochastic process presets (like Brownian Motion or GBM) and the underlying Gaussian Mixture Model (GMM) implementation ($\mu$, $\sigma$, $w$).

 1. Mathematical Cognitive Load
Novices will be instantly intimidated by Greek letters ($\mu$, $\sigma$) and mechanics like =weights sum = 1.0000=. Conversely, advanced quants understand them but will find the UI tedious and manual for engineering complex, multi-regime setups.

 2. "Blind" Configuration
Users are forcing microscopic parameter modifications (e.g., $\mu = 0.000000$, $\sigma = 0.000001$) without any immediate visual feedback on what that actually does to a resulting price chart or mathematical distribution.

 3. GMM vs. Stochastic Process Mismatch
Presets like *GBM drift* imply log-normal price paths, while *Brownian* implies normal increments. However, the interface treats them purely as static GMM stacks for "each update". It is functionally ambiguous whether the GMM represents the distribution of *absolute price changes* ($\Delta S$) or *log-returns* ($\Delta \ln S$).

* Recommended UI Overhaul
To successfully bridge the gap between educational usage and advanced quantitative engineering, consider implementing a *Progressive Disclosure* design pattern.

 Step A: Introduce Dual Modes (Simple vs. Advanced)
Do not show the raw GMM stack to novices by default. Split the tab into two clear workflows:

* *Simple Mode (Preset-Driven):* The user selects a macro behavior (e.g., "High Volatility Bull Market" or "Calm Mean-Reverting"). The UI exposes simple, human-readable sliders like *Trend Direction* (Up/Down), *Volatility* (Low/High), and *Jump Risk* (Rare/Frequent).
* *Advanced Mode (The GMM Stack):* The user clicks "Switch to Expert Mode" to unlock the raw GMM component grid ($\mu$, $\sigma$, weight) for precise distribution engineering.

 Step B: Add Live Visual Feedback
The best way to make this intuitive is to show, rather than tell. Add a split-pane or side panel containing two live-updating charts:

1. *The Distribution Chart:* A PDF (Probability Density Function) curve showing the shape of the compiled GMM distribution. This makes the concept of "stacking bell curves" instantly clear.
2. *The Simulated Path:* A mock 100-step time-series chart showing 3–5 sample price paths generated by the current configuration. Novices will instantly see what "Volatile" looks like relative to "Flat".

 Step C: Fix Parameter Terminology
If a component represents a normal distribution, label the dropdown or row clearly:

* Instead of an ambiguous name like "Constant" alongside a variance input, use *"Normal Distribution"* or *"Regime Component"*.
* Provide tooltips or secondary labels for the Greek letters: $\mu$ /(Mean/Drift)/, $\sigma$ /(Volatility)/, and $w$ /(Probability/Weight)/.
