* Goal

Make 'compass shell -f <name>.ores' work as documented across all provisioning recipes (barclays/acme/tenant/party/bootstrap). Defects, all in compass/the REPL (the .org recipes are uniform and correct for the babel runner): (1) -f resolves against cwd, not scripts/library/, so the documented bare name is 'Script file not found' -- only a full repo-relative path works. (2) compass shell auto-connects, but scripts start with 'connect $ORES_NATS_URL'; the REPL does not expand env vars and ORES_NATS_URL is not exported into the REPL, so it reconnects to a literal string, fails 'No server available', and drops the connection -- all later commands fail 'Not connected'. (3) the tangler injects leading '# ...' lines the REPL rejects as 'Wrong command'. Net: a fresh operator following any provisioning recipe cannot provision; the working path was a hand-edited script (connect line removed) with the full filesystem path.
