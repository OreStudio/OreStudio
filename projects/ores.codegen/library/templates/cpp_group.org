This page documents the =cpp= group of the codegen template
library — the namespace that gathers related facets. Each facet is a
literate org document in this directory whose =mustache= blocks tangle
to the =.mustache= artefacts consumed by =generator.py=; this page
explains what the group's facets have in common and indexes them.

* Summary

The C++ projection across five facets: the entity core
([[id:8C52C48E-9661-452F-A951-136EFC7B6B22][cpp_domain_type]] — class, entity/mapper/repository, table, JSON,
generator), the Qt UI ensemble ([[id:0D776415-7381-4E64-A368-1FE4DC55153C][cpp_qt]]), service scaffolding and the
per-entity service ([[id:CEAC637D-988B-4172-AEC6-DD5469E31C34][cpp_service]]), the new-component skeleton
([[id:99585443-E343-4B88-8A3B-6091482861E3][cpp_component]]), and cross-cutting singles ([[id:E683DAE7-28FD-414C-BD55-DE03AA10E6CB][cpp_support]] — enum,
export, field group, NATS event/handler, protocol). Entity facets are
driven by the entity org-models; component facets by the component
manifest.

* Shared conventions

- ={{{cpp_license}}}= triple-stache injects the licence header.
- Headers guard with =full_name_upper=-derived macros and export via
  the component's export macro header.
- Column loops dispatch on per-field type flags (=is_string=,
  =is_boolean=, =is_timestamp=, …); ={{^last}}= places commas.
- Temporal/non-temporal template twins persist until the Phase-3
  ={{#is_temporal}}= unification.
