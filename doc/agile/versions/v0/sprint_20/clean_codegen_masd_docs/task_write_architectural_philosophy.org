* Result

Added [[id:8D02FC2D-3832-4792-8191-25FC9101F206][Architectural philosophy]]
(=projects/modeling/architectural_philosophy.org=), the narrative "why"
companion to the System Model. It is organised around three reinforcing
decisions — dependencies point down (the four-layer invariant and what
the forbidden Domain→Application edge buys), the domain does not know
its surfaces (NATS as the runtime expression of that rule:
request/reply, pub/sub event propagation, location transparency), and
one domain behind many surfaces (the service layer as the single
contract every surface is a thin client over) — closing with why the
discipline is mechanised by codegen and how the choices compound into
coherence and independent-axis extensibility.

Cross-linked reciprocally with System Model and out to Applied MASD,
Entity lifecycle, the Messaging Reference and Component architecture;
all ids verified to resolve.
