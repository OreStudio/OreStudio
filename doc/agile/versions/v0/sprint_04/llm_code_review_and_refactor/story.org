* Goal

The headline story of sprint 04. Run an LLM-driven review across
the codebase, then act on the findings: restructure components to a
consistent =src/modeling/tests/= layout, generate per-component
PlantUML diagrams, and pick up the smaller refactors (=operator<<=
splits, table representations, =base32_encode= move, main-window
split, Catch2 tags).

* Acceptance

- LLM review pass run across every component delivered to date.
- Findings captured per component; actionable items filed as tasks.
- Components reorganised under the Dogen-style
  =component/{src,modeling,tests}= layout.
- Per-component PlantUML diagrams generated and wired into the build.

* Tasks

In execution order:

- [[id:B08CB742-8E1C-4A0C-87AE-154BA9DEEFCD][Review all code created so far with an LLM]]
- [[id:80B50B7C-F56A-4AF3-A7AB-A469D8715891][Restructure components to follow the Dogen approach]]
- [[id:95C39216-F5D9-466D-A051-FBF65466A8AD][Generate PlantUML diagrams for all components]]

Additional v0 entries belonging here, not migrated as full tasks:

* Decisions

- /Dogen layout as the canonical component shape/ :: =src/=,
  =modeling/=, =tests/= per component. Public/private include split
  deferred — left under =src=.
- /Per-component PlantUML diagrams/ :: every component model points
  at its own diagram; aggregated by a project-level diagram.
- /Format-neutral =operator<<=/ :: a domain type's stream operator
  should not bind to a specific dump format. Dump formats become
  separate functions/visitors.

* Out of scope

- Public/private include split (deferred).
- LLM review of every test file (review focused on production code
  this pass).
