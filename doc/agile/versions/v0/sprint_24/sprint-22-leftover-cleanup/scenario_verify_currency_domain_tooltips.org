* Steps

Each step is its own heading — the title should be five to seven
words so it fits on one line in the QA Validation Runner's step list
without wrapping or truncating (e.g. "Edit and save the record", not
a full sentence describing the whole operation). The body below the
title is a bullet-point checklist, not a prose paragraph: give the
tester every piece of context needed to execute that one step without
looking anything up elsewhere — what UI state must already exist,
exactly what to click or type, and exactly what confirms the step
passed. The panel writes each step's PASS/FAIL/PENDING outcome and

** Connect to tenant Barclays Plc

- Log in against the =jolly_knuth= environment as
  =tenant_admin@barclays_plc= / =Secure-Password-123= and select
  *BARCLAYS PLC*.

** Open Currencies list window

- Menu: Reference Data > Currencies.
- Confirms: the list window opens and shows currency rows (e.g. USD, EUR).

** Hover every Currencies column header

- Hover each column header in the Currencies list: Code, Currency
  Name, Symbol, Fraction, Per Unit, Rounding Type, Precision, Format,
  Monetary Nature, Market Tier, Spot Days, Day Basis, Base
  Precedence.
- Confirms: each header shows a tooltip explaining the domain concept
  (not just the header text itself), except Version/Modified
  By/Recorded At, which have none by design.

** Open a currency's detail dialog

- Double-click any currency row (e.g. USD) to open
  =CurrencyDetailDialog=.
- Confirms: the dialog opens on the General tab.

** Hover every Currency detail field

- Hover the label and the input widget for each field across the
  General, Formatting, and Rounding tabs: ISO Code, Name, Numeric
  Code, Asset Class, Market Tier, Symbol, Fraction Symbol, Fractions
  Per Unit, Format, Rounding Type, Rounding Precision.
- Confirms: every field shows a tooltip explaining the domain concept
  (e.g. Fractions Per Unit explains it drives sub-unit precision, not
  just "Fractions Per Unit").

** Open Currency Pair Conventions list

- Menu: Reference Data > Conventions > Curve Building > Currency
  Pair Conventions.
- Confirms: the list window opens and shows convention rows (e.g.
  EUR/USD).

** Hover every convention column header

- Hover each column header: Pair, Pip Factor, Tick Size, Decimal
  Places, Business Day Convention, Spot Relative, End Of Month.
- Confirms: each header shows a domain-explanation tooltip, except
  Version/Modified By/Recorded At.

** Open a convention's detail dialog

- Double-click any row (e.g. EUR/USD) to open
  =CurrencyPairConventionDetailDialog=.
- Confirms: the dialog opens with the convention's fields populated.

** Hover every convention detail field

- Hover the label and the input widget for each field: Pair Code,
  Pip Factor, Tick Size, Decimal Places, Business Day Convention,
  Spot Relative, End Of Month.
- Confirms: every field shows a tooltip explaining the domain concept
  (e.g. Spot Relative explains forward dates are relative to the
  spot date, End Of Month explains the month-end forward-date rule).
