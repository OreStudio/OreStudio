* Goal

Once every entity across every ores.qt component has migrated onto
the generic =HistoryDialog=, delete =HistoryDialogBase=, every
remaining per-entity =*HistoryDialog= class, and
=cpp_qt_history_dialog.{hpp,cpp}.mustache=/=qt_history_dialog_ui.mustache=;
confirm via zero-diff regeneration guardrails that no facet still
emits a per-entity dialog class.

=ores.qt.refdata= has fully migrated ([[id:DBDFF0B1-E358-4300-B361-644A89FBD169][Migrate ores.qt.refdata onto the
generic HistoryDialog]], DONE). The other =ores.qt= components
(admin, analytics, compute, marketdata, synthetic, trading,
workspace) have not — each is tracked as its own product-backlog
capture ("Commission ores.qt.<component> into codegen", inbox) which
includes migrating that component's history dialogs. This task is
genuinely blocked until those land; it is the final cleanup step
after every component has migrated, not something to start on its
own.

* Notes

The real current blocker is story
[[id:DBDFF0B1-E358-4300-B361-644A89FBD169][Migrate all entities onto the generic HistoryDialog]] (created
2026-07-14): only currency has migrated onto the generic
=HistoryDialog= so far (task 7D8659AB); every other entity across
=admin=, =analytics=, =compute=, =marketdata=, =party=, =refdata=,
=synthetic=, =trading= and =workspace= still uses its hand-rolled
per-entity =*HistoryDialog=. This task cannot start until that
story's per-component tasks are all DONE.

Flagged during PR #1553 review (task 7D8659AB, currency's first
migration onto the generic =HistoryDialog=): currency's own org model
(=projects/ores.refdata/modeling/ores.refdata.currency.org=) still
fully declares the per-entity history facet (=history_request_class=,
=history_response_class=, etc.), and the codegen templates
(=cpp_qt_history_dialog.hpp.mustache=,
=cpp_qt_controller.cpp.mustache=) still generate a per-entity
=CurrencyHistoryDialog=-shaped class and
=VersionNavigationHelper.hpp= include for any entity whose model has
=has_version_navigation= set — both deleted by PR #1553 by hand, not
via codegen. If codegen is ever re-run for currency before this task
retires the template, it would regenerate the just-deleted files
(a build break, or a duplicate dialog alongside the new generic one).
When migrating each entity as part of this task's rollout, turn off
that entity's history-dialog facet config in its own model as part of
the same change that deletes its hand-rolled =*HistoryDialog=, so
regeneration can't resurrect what was just removed.
