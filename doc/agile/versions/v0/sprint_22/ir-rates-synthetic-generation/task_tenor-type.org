* Goal

No tenor-string parser or date-window arithmetic exists anywhere in
the codebase outside QuantLib (confirmed by investigation —
=doc/knowledge/domain/time_structures_and_tenors.org= is purely
conceptual). Add a small, QuantLib-free tenor type: parse labels like
"O/N", "1W", "3M", "2Y" into a comparable duration, and support simple
date-window overlap checks (no business-day calendar/holiday
machinery — that level of precision is only needed for actual
bootstrapping, out of scope here). This is the prerequisite for the
tenor-collision validator task.

* Acceptance

- Parses standard tenor labels (O/N, T/N, S/N, S/W, 1W-1Y per
  =time_structures_and_tenors.org='s existing convention table) into a
  comparable duration type.
- Given two (start, tenor) pairs, can answer whether their date
  windows overlap, using =std::chrono= only — no QuantLib dependency.
- Lives in a location reusable outside =ores.synthetic= (e.g.
  =ores.utility=), since vol-surface pillar work will need it too.
