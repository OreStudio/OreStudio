* Goal

Add the =session= domain type and the open / close / expire
lifecycle on top of the cobalt-based server. A user must be able to
log in and log out.

* Acceptance

- =session= domain type defined.
- Server-side session lifecycle: open, close, expire.
- Client can authenticate with an account and receive a session.
- Client can log out and end the session.

* Notes

- From the v0 entry: "Users must be able to login and logout."

* Result

Login and logout work end-to-end across client and server.
