* Goal

Surface the parsed currency data in the Qt UI through a basic tree and
list view. This is the first UI binding of project data and sets the
pattern for every reference-data widget that follows.

* Acceptance

- A widget exists that lists currencies loaded from the database.
- A tree view variant supports hierarchical grouping (e.g. by region or
  family).
- Selecting a row shows the currency's details elsewhere in the UI.

* Context

Closes the loop on /currencies end-to-end/: imported via XML, stored in
the DB, now displayed in the UI. Sibling to /Add database support for
currency/ and /Add XML parsing support for currency/.

* Notes

- Total time spent in v0: ~1:50.
- Followed Qt's standard model/view pattern from
  https://doc.qt.io/qt-6/modelview.html. This choice propagated to
  every later widget in the project.

* Result

- A working tree/list view of currencies inside =ores.qt=, the first
  data-bound UI in the project.
