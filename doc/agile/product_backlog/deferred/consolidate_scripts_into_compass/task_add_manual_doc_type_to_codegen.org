* Acceptance

- =compass add manual --slug chapter_N_<topic>= scaffolds a well-formed
  =#+type: manual= org file in [[proj:doc/manual/user_guide][=doc/manual/user_guide/=]] by default.
- =--parent-dir= is optional; default is [[proj:doc/manual/user_guide][=doc/manual/user_guide=]].
- The scaffolded file uses =**= top-level headings (correct for
  =:minlevel 2= inclusion in =user_manual.org=).
- =manual= is registered in =doc_generate.py= (=TYPE_TO_TEMPLATE=,
  =DEFAULT_INITIAL_STATE=, =PARENTLESS_TYPES=, output-path branch).
- =compass add --help= lists =manual= in the types line.
- [[proj:doc/meta/document_types.org][=doc/meta/document_types.org=]] documents the =manual= type.
- [[proj:doc/recipes/codegen/how_do_i_create_a_new_doc.org][=doc/recipes/codegen/how_do_i_create_a_new_doc.org=]] has a
  =** New manual chapter= section with a working example.

* Notes

Discovered as a prerequisite while writing the Chapter 4 Reference Data
manual chapter on branch [[id:B3788A76-208E-4422-B277-17DF8FF3A78C][Write currency documentation]]. The =knowledge=
type produced =#+type: knowledge= output which is incorrect for manual
chapters; a dedicated type was needed.

Also fixed a pre-existing =NameError= in =doc_show.py=: =_C_YELLOW= was
referenced in =show_link_row= and =show_anchor= but never defined at
module level, causing every =compass show= invocation to exit with
code 1 after printing the outgoing-links section.
