* Goal

Write the populate script that seeds the synthetic FX spot config
artefact table with a usable set of configs, and register a bundle
(+ membership row) so the set is selectable and applyable from the
Librarian. This is neither the FPML-coding-scheme case (no
=fpml_parser.py= support applies — starting spots aren't an external
enumeration) nor a real external-source import: reuse the
=sql_dataset_refdata=/=sql_populate_refdata= populate-script skeleton
(idempotent dataset registration + artefact insert) for the SQL
scaffolding, but the GMM starting-spot values themselves must be
computed/hand-authored. Reuse the reference-date/internally-consistent-
spot analysis from the deferred sprint 21 task =Synthetic FX spot
library: 30 pairs with historically-consistent starting spots=
(=7B130100-59F1-477D-910A-AF6D2C8D732C=) where useful, rather than
re-deriving it — that task was blocked on exactly the Librarian/bundle
infrastructure this story now builds.
