* Goal

Produce a written design note that resolves the key open questions for the
Codegen Model Dashboard before implementation begins: which colour to assign
each MASD type (following Object Modeling in Color conventions), which graph
layout algorithm to use, how the Mustache-template HTML preview will work
(data source, rendering approach), and how to derive the codegen-specific
JSON slice from =graphdata.json=.

The output is a decisions section in the story, plus any supporting notes
in this task's =* Notes= — enough for an implementer to start without
further design discussion.

* Acceptance

- A colour is assigned to each MASD type in the inventory (entity, lookup_entity, field_group, table, junction, service_registry, module, facet_catalogue, component_catalogue, modeline_catalogue), justified against Object Modeling in Color conventions.
- A graph layout algorithm is chosen and justified (force-directed, hierarchical, or other); trade-offs against the agile dashboard's approach are noted.
- The Mustache-template HTML preview approach is decided: how template org files are identified in the data, what is rendered, and how the expand/collapse interaction works.
- The data pipeline is decided: whether to filter =graphdata.json= client-side or produce a separate =codegendata.json= at site-build time, with rationale.
- All decisions are recorded in the parent story's =* Decisions= section before this task closes.
