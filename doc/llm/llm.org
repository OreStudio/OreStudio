Everything an LLM session needs to operate on ORE Studio lives under
=doc/llm/=: the session entry point, Claude Code permissions, skills,
and project-scope memories. Return to [[id:b69989f6-19e1-430d-ac6d-b2416e4c433f][Compass]] for the full =doc/= map.

* Detail

** Entry point

[[id:f9ad7932-8e11-433c-a812-b57dbc9bf5d8][LLM instructions]] (=llm_instructions.org=) is the mandatory first read
for every session. It lists the non-negotiable rules, links to the
build, code style, architecture, database isolation, testing, git/PR
conventions, and project memory documents.

** Claude Code permissions

[[id:2d83251f-0e24-4b97-b736-08f710316634][Claude Code Settings]] (=claude_code_settings.org=) is the literate
source for =.claude/settings.json=. It contains the minimal set of
Bash globs and sandbox overrides needed to operate in this repository.
Never edit =.claude/settings.json= directly; regenerate it with the
[[id:3ddb2afa-9caa-44a0-a91a-645a95320a5f][deploy_settings]] CMake target.

** Skills

The =skills/= subtree holds all Claude Code [[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skills]]. Each skill lives
at =doc/llm/skills/<slug>/SKILL.org= and is task-shaped: it describes
/when/ to invoke it and /how/ to execute it end-to-end. The full
catalogue is at [[id:2003934B-ED05-D114-404B-08C0E0D844BB][Claude Code Skills]].

Skills are deployed to =.claude/skills/= with the
=deploy_skills= CMake target (see [[id:b5f231a9-5403-482c-8f2a-12d6917c58cd][How do I deploy the skills?]]).
To create a new skill, see [[id:3153aa4b-23d4-4111-9710-5a4971102edc][How do I create a new v2 doc?]] §"New
Claude Code skill".

** Project memories

The =memory/= subtree holds persistent project-scope memories for LLM
sessions — feedback the user gave, facts about the work in flight, and
pointers to external systems. Each memory is a single short file
(=<slug>.org=, =#+type: memory=). The catalogue and inclusion rules
are at [[id:dc0bba22-9d4f-4fdd-b011-e30b02b4b82f][Project memory]].

To write a new memory, use the [[id:7357c672-a49f-4974-9399-a166f761d222][ores-memory skill]] or follow
[[id:391b9231-dba8-4e8c-97ab-a75e2b9fea07][How do I create a memory?]].
