* Acceptance

- Service accepts =--database-host=, =--database-port=, =--database-user=, =--database-password=, =--database-name= (and the equivalent env-var prefix).
- Prodigy invocations pass DB params through these flags.
- Defaults are wired through Boost.program_options rather than handled by ad-hoc code.

* Notes

- Total time spent in v0: ~3:34.
- From the v0 notes: "we are using the defaults incorrectly. Set them as part of the options descriptions setup and remove manual handling code."

* Result

DB connection params come from the command line / environment, not from
hard-coded constants; Prodigy and the service share one configuration
surface.
