* Goal

LLM agents bypass =compass search= because the result format does not
look directly useful: a title, tags, a path, and an ellipsised
snippet, with no obvious next action. Refactor the output to be
focused and actionable — one tight block per hit with a ready-to-run
yellow hint (=compass show <UUID>=, matching the house hint UX used
by audit/fleet), the doc type, and ideally a direct answer extract
for question-shaped queries (recipes' Question/Answer sections).
Fewer, better-ranked results beat ten loose ones.

* Result

Delivered in PR [[https://github.com/OreStudio/OreStudio/pull/1129][#1129]] (merged). Default =compass search= output is now
one tight block per hit in the house style — type icon, doc type,
bold title, description, ready-to-run yellow =compass show <UUID>=
hint — deduplicated to one result per doc (kills the rel/abs
double-indexing repeats). Question-shaped queries run a title-scoped
first pass with recipe paths floated to the top and surface a =💬=
answer extract from the recipe's =* Answer= section: "How do I create
a PR?" now returns the recipe at rank 1 with its answer inline.
=--limit= defaults to 5 (validated positive after review round 1);
=-v= restores the verbose path/snippet form. Two latent bugs fixed en
route: punctuation broke FTS5 syntax, and operator detection used
substring matching, so any query containing "ORE" was parsed as
verbatim FTS5. Search recipe updated.
