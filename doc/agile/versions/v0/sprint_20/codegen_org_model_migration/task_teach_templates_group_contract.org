* Result

Taught =generator.py= and both entity templates the field-group contract.
=domain_identity_group= / =domain_audit_group= / =:group: identity= annotations
in the 9 rates instrument org models now drive nested =identity.= / =audit.=
sub-struct output from the domain-class and mapper templates.  Key deliverables:

- =generator.py=: detects group annotations, derives =has_identity_group=,
  =identity_group_type=, =has_audit_group=, =audit_group_qualified=, and marks
  per-column =is_identity_group_column=; auto-injects instrument_identity and
  audit_record includes into =cpp.includes.domain=.
- =org_loader.py=: fixed =default_value= falsy bug — numeric 0 / 0.0 were
  silently skipped by pystache; value is now kept as the raw string.
- =cpp_domain_type_class.hpp.mustache=: identity/audit conditional blocks
  replace the flat version/tenant/workspace/pk header and flat audit tail.
- =cpp_domain_type_mapper.cpp.mustache=: both map directions gain
  identity/audit conditional blocks with prefixed accesses.
- All 9 instrument org models: trimmed =<optional>= / UUID includes from domain
  includes; removed do-not-regenerate warnings; confirmed codegen is idempotent.
- =generate_trading_instruments.sh= deleted.
- Guard names corrected from =ORES_TRADING_DOMAIN_= to =ORES_TRADING_API_DOMAIN_=
  in the 9 hand-written domain headers (template generates the canonical form).
