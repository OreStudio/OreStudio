How to use =compass=, the [[id:6F7A122F-BC0F-4F3E-BA46-8B793FF15D10][developer toolkit]] for ORE Studio.
One CLI (=projects/ores.compass/compass.sh=) covers five pillars:
Orient, Search, Scaffold, Capture, and Journal.

* Orient

- [[id:41E5F9A8-1C2B-4ECA-8EA0-1ABBAB0B6A31][How do I see where we are?]] — current version, sprint, and in-flight
  stories/tasks (=where= / =status=).
- [[id:0D378978-7CF1-45F5-896F-63E281CC02CB][How do I see what every worktree is doing?]] — branch / story / task /
  PR per git worktree (=fleet=).
- [[id:A7D3C1E5-9F2B-4A8D-B6E4-3C7F1D9A2E5B][How do I get a sprint or story status?]] — all stories or tasks grouped
  by state (=sprint status= / =story status=).

* Search and indexing

- [[id:DEAC11AE-3753-49FA-994A-98594162A605][How do I index notes for compass?]] — build the FTS cache (=index=).
- [[id:BDA8F7FB-5CEC-42D8-BC78-766BBC8D4285][How do I search docs with compass?]] — full-text search (=search=).

* Authoring

- [[id:5B670808-D28B-4818-A2FA-EA5220D35E8D][How do I add a doc with compass?]] — create a doc via codegen
  (=add=).
- [[id:84C9177E-C737-428A-BDF9-E55FA6C9680C][How do I start a unit of work with compass?]] — fetch main, branch,
  scaffold a linked story+task (=story new= / =task new=).

* Captures

- [[id:7072D39C-9607-48F9-8EE6-50E8647DB8ED][How do I capture ideas from freeform text?]] — paste unstructured
  notes; LLM decomposes into well-structured captures in =inbox/=
  (=compass capture --note=).

* Navigating the doc graph

These commands moved into compass from =ores.codegen= (the tool now owns
doc navigation); the recipes live under [[id:95FF6AF4-2320-4CB6-A007-B016B7DA16ED][Documentation recipes]]:

- [[id:D1AE0786-5F2F-4437-8449-0E480C278A1B][How do I find docs matching a pattern?]] — filter the graph
  (=compass list=).
- [[id:E8231C22-CB31-4D85-A9C0-7602953764FF][How do I show a doc by UUID?]] — one doc + inbound/outbound links
  (=compass show=).
