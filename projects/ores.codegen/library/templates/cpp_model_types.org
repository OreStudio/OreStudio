This page is the literate source for the =cpp_model_types= facet of the
codegen template library. Each section documents one template and
holds its source in a =mustache= block tangled to the sibling
=.mustache= file consumed by =generator.py=. The =.mustache= files are
generated artefacts — edit this document, then run the tangle
(=cmake --build --target tangle_codegen_templates= or
=projects/ores.lisp/src/ores-build-codegen-templates.el= directly).
Output paths and profile membership come from
[[proj:projects/ores.codegen/library/profiles.json][profiles.json]].

* Summary

Two templates for model formats of their own: the enum class with
string conversion (=enum= profile) and the field-group struct
(=field-group= profile — shared column blocks for the nested
instrument decomposition). Each renders from its own model file
rather than from an entity.

* The cpp_model_types facet

Standalone model formats: an enum org-model or field-group org-model
is a first-class codegen input that produces exactly one header.
Unlike the entity facets there is no layered projection — one model,
one artefact.
