For the build script that does the work (=.build-site.el=) and the

* Question

How do I build the ORE Studio website locally?

* Answer

The =deploy_site= CMake target invokes Emacs in batch mode against
=.build-site.el=, which publishes every =.org= file in the repository
(excluding =.packages=, =vcpkg=, =build=) to HTML:

#+begin_src sh :results verbatim
cmake --build --preset linux-clang-debug-ninja --target deploy_site
#+end_src

Output lands at =build/output/site/=.

** Previewing locally

The site uses absolute =/OreStudio/= paths (matching GitHub Pages).
Use the serve script to preview with the correct URL structure:

#+begin_src sh :results verbatim
./build/scripts/serve-site.sh
#+end_src

Then open =http://localhost:8000/OreStudio/= in a browser.

To rebuild and serve in one step:

#+begin_src sh :results verbatim
./build/scripts/serve-site.sh --compile
#+end_src

Use =--kill= to kill any process already on the port before starting.
Use =--port= to override the default port 8000, or set the =PORT= env var.

** Prerequisites

- Emacs on PATH. The script uses =org-publish=, =org-id=, and =oc-bibtex=
  from a default Emacs install with =org-mode= shipped.
- The project must be configured (=cmake --preset ...=); the target
  lives in the preset's build directory.

** When to re-run

Any time you add a new =.org= file, change a =:ID:= or =#+title:=, or
edit anything that affects link resolution. The target wipes
=build/output/site/= first so links don't stale.

* Script

=.build-site.el= is the Emacs script the target invokes. The CMake
target definition lives in =CMakeLists.txt= near
=add_custom_target(deploy_site ...)=.

* Tested by

The =build-site.yml= GitHub Action publishes the site on every push to
=main=. The action's success is the integration test for this recipe.
