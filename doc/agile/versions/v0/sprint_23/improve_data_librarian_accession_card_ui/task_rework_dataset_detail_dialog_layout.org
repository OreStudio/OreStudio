* Acceptance

- Persistent header banner shows =Name=/=Code=/=Version=/=ID= once
  (full UUID, selectable), with a one-click Copy ID action; removed
  from the Overview tab's General section.
- Origin/Nature/Treatment render as real badges — the same
  =DelegatePaintUtils::draw_centered_badge= routine and =BadgeCache=
  every other badge in the app uses, not a one-off widget.
- *Provenance* and *Methodology* merge into one tab; lineage
  primitives are visually prominent.
- Embedded as an MDI subwindow (=DetachableMdiSubWindow=), same
  pattern as every other window in the app — not a floating,
  always-on-top =QDialog=.
- No changes to =DetailDialogBase= or other entity dialogs.
- Manually verified in the running Qt client against a real dataset
  via the QA Validation Runner scenario below — PASSED.

The full asymmetric two-column layout (primary content + metadata
sidebar) from the original review is deferred to a follow-up task —
see =* Plan=.

* Plan

Corrected the target from the story's original =DatasetDetailDialog=
(the create/edit form, only 2 tabs) to the actual accession-card
viewer, =DatasetViewDialog= (=projects/ores.qt/refdata/src/DatasetViewDialog.cpp=),
which has the 4 tabs (Overview, Provenance, Methodology, Dependencies)
the review targets. Also found *Dependencies* already has a working
=QGraphicsScene=-based interactive lineage graph — not flat rows as
the review assumed — so that tab needs no rework.

Phase 1 (this task):
1. Add a persistent header banner above the tab widget: Name, Code,
   Version, ID with a one-click copy button; remove the now-redundant
   Name/Code/Version/ID rows from the Overview tab's General section.
2. Merge *Provenance* and *Methodology* into a single
   "Provenance & Methodology" tab (stacked via QSplitter, reusing the
   existing per-tab update methods).
3. Style Origin/Nature/Treatment values as visual pills (background
   colour + bold) instead of plain tree rows.

Deferred to a follow-up task if this phase lands well: the full
asymmetric two-column layout (splitting each tab's tree into a
primary-content region and a metadata sidebar) is a bigger structural
change than fits this task's scope.

* Result

Shipped Phase 1 of the accession-card redesign on =DatasetViewDialog=:
persistent header banner (Name/Code/Version/full selectable UUID +
Copy ID button), Origin/Nature/Treatment rendered as real badges via
a new =OverviewBadgeDelegate= (=DelegatePaintUtils::draw_centered_badge=
+ =BadgeCache=, matching every other badge in the app), *Provenance*
and *Methodology* merged into one tab, and left-aligned section
headers instead of the centered-grey-bar anti-pattern. Also fixed a
real bug found only through manual testing: the dialog was a floating,
always-on-top =QDialog=; converted to a =QWidget= embedded in a
=DetachableMdiSubWindow=, same pattern as every other window in the
app.

Three review rounds with the running client surfaced real defects a
compile-only check couldn't catch (Qt mnemonic bug swallowing the
=&= in the tab title, stretched/oversized pills, a hand-rolled badge
that diverged from the app's actual badge-rendering code, low-contrast
banner text) — see the Review table. The QA Validation Runner scenario
now PASSES all 7 steps with no outstanding notes.

The full asymmetric two-column layout from the original review is
deferred to a follow-up task — the "Compass improvements"-style
catch-all pattern doesn't apply here; file it as a new task on this
story when picked up.
