* What

The currencies list window (=CurrencyMdiWindow=) wires a
=PaginationWidget= (=projects/ores.qt/refdata/src/CurrencyMdiWindow.cpp=,
~line 83, ~line 272-307) the same way other entity list windows do,
but observed while manually testing (task
[[id:7454EE7E-1763-495B-94F4-0AFC9BADB2C5][GitHub-style diff view in history changes tab]], unrelated) paging
doesn't appear enabled/working for currencies specifically. Needs a
repro against a currency list with more entries than one page to
confirm scope (client-side wiring vs. server-side offset/limit
handling) and find the root cause.

* Why

Inconsistent behaviour between entity list windows; if currencies
can't page, large currency lists are unusable/incomplete in the UI.
