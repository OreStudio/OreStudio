This is the entry point for every LLM session on ORE Studio. Read it in
full before acting. Each section below names the rule and links to the
canonical document; follow the link for depth. Non-negotiable rules are
inlined rather than linked.

* Non-negotiable rules

These apply in every session, with no exceptions:

- /UNIX line endings/ — all new files use LF, never CRLF.
- /New org docs via codegen only/ — never write =.org= frontmatter by hand.
  Always use =projects/ores.codegen/generate_v2_doc.sh=.
  See [[id:3153aa4b-23d4-4111-9710-5a4971102edc][How do I create a new v2 doc?]].
- /Check recipes first/ — before searching for how to do anything, open
  [[id:46F5ED07-9E63-77B4-2783-E0147EFF45D0][Recipes]] and find the matching topic.
- /Every new =.org= file must carry =#+version: 2=/ — the CI gate rejects
  files without it. See [[id:23bcac49-325f-4083-b854-87ea747c2abe][How do I audit and fix document version tags?]].

  invariant lives under "Service Table Isolation" in this doc.
- Rule: each service user holds DML only on its own =ores_<service>_*=
  tables. Do not add cross-service grants without documented justification.
  Trigger functions that SELECT across services must be
  =SECURITY DEFINER SET search_path = public, pg_temp=.
