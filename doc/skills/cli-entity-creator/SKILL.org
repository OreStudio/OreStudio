* When to use this skill

When you need to add CLI commands for a new entity in =ores.cli=.
Prerequisites: the domain type must exist ([[id:b1450696-8c51-4cc5-8910-84912a411ab6][domain-type-creator]]),
the repository and table I/O must be implemented. See [[id:608675e1-4284-4902-99ac-4da565390afb][Entity lifecycle]]
for the full layer ordering.

* How to use this skill

1. /Gather requirements/: entity name, command name (plural, hyphenated),
   operations needed (list, export, import, add, delete), key field,
   required and optional add parameters.

2. /Phase 1 — list + export/ (PR: =[cli] Add {entity} list and export commands=):
   add entity enum, create parser class with list/export, register in
   =main_parser.cpp=.

3. /Phase 2 — add + delete/ (PR: =[cli] Add {entity} add and delete commands=):
   create =add_{entity}_options.hpp=, extend parser with add/delete.

4. /Phase 3 — recipe docs/ (PR: =[doc] Add {entity} CLI command recipes=):
   add =how_do_i_*.org= recipes under =doc/recipes/cli/=.

Use [[id:6DAFFC2F-7F94-403B-87D0-2A56C4669B7F][feature-branch-manager]] between phases and [[id:545D6B43-17FC-0234-FF2B-AB6565695616][pr-manager]] to open each PR.
Build after every step: =cmake --build --preset linux-clang-debug-ninja --target ores.cli=.
