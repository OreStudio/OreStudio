* Goal

Produce a running tick loop in =ores.synthetic.service= that continuously generates
EUR/USD mid-price ticks using a seeded GMM stochastic process and publishes each tick
as an =fx_spot_tick= JSON payload to the NATS fan-out subject
=marketdata.v1.tick.fx.rate.eur.usd=. No DB write in this step — that is step 3. The
service starts the tick loop automatically at startup (hardcoded EUR/USD, K=3 GMM with
seeded parameters, fixed-mode clock) so the generation pipeline and NATS publishing can
be validated in isolation before start/stop NATS control is added (step 4).

This is implementation step 2 from the [[id:29573D59-1BA2-4DBB-A12B-8EAC7F7CC4AB][architecture doc]].

* Plan

1. Add =GmmProcess= (=src/processes/gmm_process.hpp= + =gmm_process.cpp=):
   - Constructor takes =k=, =means=, =stdevs=, =weights=, =initial_price=.
   - =next()=: draw a component index from =weights= via discrete_distribution, draw a log-return from the corresponding =normal_distribution=, apply =price *= exp(log_return)=, return price.
   - =current()=: return the current price without advancing.
   - Uses =std::mt19937= seeded with a fixed seed for reproducibility in tests; expose seed as a constructor param defaulted to 42.

2. Add =process_factory= (=src/process_factory.hpp= + =process_factory.cpp=):
   - =make_gmm_process(k, means, stdevs, weights, initial_price)= → =unique_ptr<IStochasticProcess>=.
   - Kept minimal for now; will grow as more process types are added.

3. Add =FxSpotFeed= (=src/fx_spot_feed.hpp= + =fx_spot_feed.cpp=):
   - Implements =IFxSpotFeed=.
   - Constructor takes =nats::service::client&=, =ore_key=, =unique_ptr<IStochasticProcess>=, =ticks_per_hour= (double).
   - =start(handler)=: creates a =std::thread= that loops: sleep for period, call =process_->next()=, build =fx_spot_tick=, call =handler(tick)=, publish to NATS via =rfl::json::write= + =nats_.publish=. Runs until =stop_flag_= is set.
   - =stop()=: sets =stop_flag_= (=std::atomic<bool>=); the thread exits its loop on the next wakeup.

4. Wire into =application.cpp=: construct a =FxSpotFeed= for EUR/USD with seeded GMM parameters; call =feed_.start([](const auto&){})= on a detached =std::thread= launched in the =run()= coroutine body (before the service runner starts). Stop the feed in a shutdown hook or via the thread's natural lifecycle.

5. Check the NATS subject naming convention and confirm =marketdata.v1.tick.fx.rate.eur.usd= is the right derived subject for ORE key =FX/RATE/EUR/USD= (lowercase, =/= → =.=, prefix =marketdata.v1.tick.=).

6. Verify by running the service and subscribing to the tick subject.
