* Goal

Wire the =FxSpotFeed= tick loop into the existing =ores.marketdata.service= so that each
synthetic EUR/USD tick is persisted as a =market_observation= via NATS
=marketdata.v1.observations.save=. At service startup, look up or create the EUR/USD
=market_series= (=FX/RATE/EUR/USD=) via =marketdata.v1.series.list= + =marketdata.v1.series.save=
to obtain a stable =series_id=. Observations should be visible in the existing
=MarketSeriesMdiWindow= observation view without any UI changes.

This is implementation step 3 from the [[id:29573D59-1BA2-4DBB-A12B-8EAC7F7CC4AB][architecture doc]].

* Acceptance

- [x] At service startup =ores.synthetic.service= performs a =marketdata.v1.series.list= request and reuses the existing EUR/USD series if present; otherwise creates it via =marketdata.v1.series.save= and logs the new series UUID.
- [x] Each generated tick results in a =marketdata.v1.observations.save= request containing one =market_observation= (tenant, series_id, datetime, 6-decimal-place value, source "SYNTHETIC").
- [x] Failures to save an observation are logged at WARN level and do not stop the tick loop.
- [x] =FxSpotFeed= constructor accepts =series_id= and =tenant_id= (step 3 wiring); =application.cpp= duplicate include removed.
- [ ] EUR/USD observations appear in =MarketSeriesMdiWindow= observation view (manual verification post-merge).

* Plan

1. Fix duplicate =#include "../registrar.hpp"= introduced by nightly formatter in =application.cpp=.

2. Add =series_id= and =tenant_id= fields to =FxSpotFeed= constructor and private members; add =boost::uuids::random_generator= for per-tick observation UUID generation.

3. Implement per-tick observation save in =FxSpotFeed::start()=: build =market_observation= (id = fresh UUID, tenant_id, series_id, observation_datetime = tick.datetime, recorded_at = tick.datetime, value = formatted to 6 d.p., source = "SYNTHETIC"), serialise with =rfl::json::write=, send via =nats_.request_sync=, log WARN on failure.

4. In =application.cpp=: create =db_ctx= before the feed; at startup call =marketdata.v1.series.list?series_type=FX=, search the response for metric="RATE"/qualifier="EUR/USD", use existing =id= if found; otherwise generate UUID, populate =market_series= fields, save via =marketdata.v1.series.save=, throw on failure. Pass =series_id= and =db_ctx.tenant_id()= to the =FxSpotFeed= constructor; pass =std::move(db_ctx)= to =service::run()=.
