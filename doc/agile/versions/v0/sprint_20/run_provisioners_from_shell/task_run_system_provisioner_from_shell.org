* Goal

Expose the system provisioner flow as ores.shell commands so the entire SystemProvisionerWizard can be reproduced from the command line: bootstrap admin account creation (username, email, password), setup mode selection (single vs multi tenant), tenant details (code, name, type, hostname, description) and tenant admin account creation. Close any gaps identified by the wizard-to-shell mapping analysis; the existing bootstrap command is the starting point.
