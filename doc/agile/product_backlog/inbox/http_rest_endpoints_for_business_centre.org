* What

Add HTTP REST endpoints for =business_centre= (list, get, create,
update, delete, history) once the HTTP-entity codegen profile
([[id:F556E446-EB91-4656-BEDC-15AE0F7B5ECD][Entity commissioning process reference]]'s gap protocol, "❌ GAP" row
for the HTTP layer) exists. Do not hand-code the layer ahead of the
profile — raise/complete the codegen-profile story first, then
regenerate.

* Why

=business_centre= was fully commissioned end-to-end for SQL, domain,
repository, service, protocol, NATS, and Qt during
[[id:2406AA9C-6B67-4B74-954E-2C04AE833FB9][Commission: business_centre]], but HTTP REST support remains a
codegen gap shared by every entity, not specific to =business_centre=
— tracked here so the entity isn't forgotten once the HTTP-entity
profile lands.
