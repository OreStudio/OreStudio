* When to use this skill

When you need to confirm one or more existing skills still match the
v2 contract — periodic audit, after a codebase change that may have
invalidated their instructions, or before adding a new skill that
might overlap.

* How to use this skill

1. /Get the current list of skills/ from the canonical source — never
   maintain it in this file. The [[id:2003934B-ED05-D114-404B-08C0E0D844BB][Skills catalogue]] groups skills by
   area. For a flat, scriptable view:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh list --type skill
   #+end_src

2. /Inspect each skill's metadata/ via the doc tools:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh show <uuid-prefix>
   #+end_src

   This prints the skill's blurb, outgoing links (the recipes /
   reference docs it depends on), and incoming links (other skills
   that link to it).

3. /Check each skill against the v2 contract/. The contract is the
   authoritative source — read =skill= in [[id:8B29A8D8-E004-4E87-B81D-ABEAF68F2B98][document types]]; the items
   below are the ones to focus on during review:

   - *Frontmatter*: =:ID:=, =#+title:=, =#+description:=,
     =#+type: skill=, =#+version: 2=, =#+level: cross=,
     =#+filetags:=, =#+created:= / =#+updated:=.
   - *Markdown export block* with =name:= / =description:= /
     =license:= (Claude Code metadata, tangled into the shipped
     bundle).
   - *Required sections*, in order: =* When to use this skill=,
     =* How to use this skill=, =* Recipes=, =* Reference=,
     =* Artefacts :noexport:= containing the licence tangle.

4. /Check that the skill is task-shaped/, not role-shaped. A skill is
   /how to do a unit of work/. If the doc describes /what someone
   does/ across many tasks, it is a [[id:66F2A9E8-0385-4D56-B4B1-0957208990DC][function]], not a skill — flag
   for re-homing.

5. /Check that procedural detail lives in recipes, not in the skill/.
   A skill is a thin index. Detailed commands, examples, and
   step-by-step procedures belong in =doc/recipes/<topic>/=. If you
   see embedded shell sequences, =cmake --build= invocations, or
   templates inside the skill, replace them with id-links to the
   matching recipe.

6. /Check that the skill does not embed an inventory of other skills/.
   Use id-links to the [[id:2003934B-ED05-D114-404B-08C0E0D844BB][Skills catalogue]] instead — embedded lists
   go stale.

7. /Spot-check the linked recipes and references/ still exist. The
   =compass show= output will flag broken =[[id:...]]= as =BROKEN=.

8. /File findings/. Either:
   - Open a [[id:31C868B9-306E-4282-BA8C-07E647021B34][task]] in the current sprint for each fix-shaped finding,
   - Open a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in =near/= for larger restructures, or
   - Apply the fix directly when it's a one-line correction.
