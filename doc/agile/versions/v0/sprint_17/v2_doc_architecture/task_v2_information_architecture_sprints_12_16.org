## Summary

Completes the v0 → v2 information architecture migration begun in PR #771 + #775. Five sprints (12-16) bring the final 86 stories and 252 tasks across, and mark v0 as RELEASED.

### Sprint 12 — Party + counterparty foundation

20 stories / 62 tasks. The largest mid-v0 sprint.

- Party + counterparty + identifier + contact-information end-to-end (database, domain, Qt).
- GLEIF data integration (datasets, central-bank LEIs, LEI-to-BIC).
- Second RLS layer: party-level isolation scoped to the party subtree within a tenant.
- Audit trail + system account: split =modified_by= (DB user) from =performed_by= (app actor); generation-context KVP for ~36 generators.
- Tenant provisioning wizards (GLEIF + synthetic); books / portfolios / business units modelling.
- FSM framework + trade-envelope schema; =ores.trade= → =ores.trading= rename.

### Sprint 13 — Trade + scheduling foundations

16 stories / 22 tasks. Sprint mission: trade + scheduling support.

- Multi-party login picker + AccountPartiesWidget (closes sprint-12 party-RLS future-story).
- =ores.scheduler= library wrapping =pg_cron= with Quartz.NET-style fluent API.
- Portfolio explorer combining books + portfolios + filtered trade table.
- Party isolation extended to books / portfolios / trades.
- FSM moves from stand-alone home into =ores.dq=.
- Atomic batch save + delete across 23 entities.

### Sprint 14 — Messaging infrastructure pivot

13 stories / 21 tasks. Sprint mission: work on messaging infrastructure.

- ORE Import Wizard (seven-page directory-scan).
- =ores.reporting= subsystem from scratch (design + schema + domain + UI).
- Messaging pivot: pgmq + pg_cron → custom MQ → NNG broker → NATS. The NATS landing removed ~107k lines of binary-protocol code; 10 domain services migrated.
- Party codename for per-party queue isolation (with TOCTOU + sequence-based race fixes).
- JWT moves into =ores.security= with RS256 asymmetric signing.

### Sprint 15 — Compute grid

15 stories / 24 tasks. Sprint mission: implement compute grid.

- BOINC-inspired distributed compute grid (=ores.compute= + =ores.compute.service= + =ores.compute.wrapper= + NATS JetStream lifecycle).
- Compute grid observability (TimescaleDB telemetry + service heartbeat RAG dashboard).
- =*.api= + =*.core= architecture split across all 10 service libraries.
- JWT refresh (configurable lifetimes + auth telemetry hypertable + shell + Qt clients).
- Unified =system_settings= (bool / int / string / JSON) replacing boolean-only =feature_flags=.
- Four CDM instrument phases: rates / FX / bond / credit.

### Sprint 16 — Final v0 sprint

22 stories / 89 tasks. The largest sprint of v0; closes v0.

- ORE import end-to-end: dispatch + mappers for every instrument family + supporting files (market.txt / fixings.txt / calendar / conventions) + server-side =ores.ore.service= + portfolio export + golden roundtrip tests.
- Workflow engine generalised as event-driven saga orchestrator; party provisioning + ORE import as concrete sagas; report execution workflow pipeline.
- Market data subsystem from scratch (parser + schema + repository + service + Qt UI).
- =ores.controller= service lifecycle component; unified NATS heartbeats across all services.
- Qt plugin architecture: =ores.qt.api= + 7 domain plugins (admin / compute / refdata / party / mktdata / trading / workflow).
- Trade detail dialog unification: six instrument families merged behind =IInstrumentForm= registry.
- Per-type instrument tables (rates / FX / equity) replace the generic instruments table from sprint 15.
- Reporting + analytics: DQ-sourced report definitions, report execution workflow, analytics pricing engine.
- Asset class unification: =asset_class= (risk taxonomy) and =product_type= (structural routing) treated as orthogonal.
- UTC-everywhere timestamp policy enforced from DB to wire format.
- ~20 Windows portability fixes (MSYS2, NATS certs, strptime, MOC dllimport, _wfopen_s, SIGQUIT, /FS, DLL export limit, MSVC PDB, xsd::vector, GLib dl-init, constexpr depth).
