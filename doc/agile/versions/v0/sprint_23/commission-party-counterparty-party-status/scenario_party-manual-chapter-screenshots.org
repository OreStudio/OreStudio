* Steps

Each step captures one screenshot referenced by
=doc/manual/user_guide/chapter_8_parties.org=. Save every capture into
=doc/manual/user_guide/assets/images/= under the exact filename given,
overwriting any placeholder that may already exist. Use the Barclays
Plc (GLEIF) environment so the parties list has real, varied data —
several rows with distinct Party Type and Status badges, and at least
one party several levels deep in the hierarchy for the Hierarchy tab.

** Connect and open the Parties window

Start the client, log in against the =prime_origin= environment, and
open the Parties window from the /Reference Data/ menu. Let it load a
full page of parties with at least two different Party Type values
and two different Status values visible, so both badge colours show.
