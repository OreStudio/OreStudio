* When to use this skill

When you need to act as the product owner of ORE Studio:

- File a new idea that does not belong in the current sprint.
- Promote a [[id:671f18e4-e09c-4b3b-bd24-d33df8ae38a6][capture]] from the product backlog into a [[id:699a8d45-b67e-4d3e-9206-24fa17c51ada][story]] in a
  sprint.
- Open a new [[id:0820b7fd-147c-4832-ac25-c043d38d5b61][sprint]] for the current [[id:ca4d4bde-2eab-4f0a-bc8b-7c0bcb17bc82][version]].
- Re-prioritise the [[id:9a17e8df-7e94-4f55-8a14-bcb7d9a16d6b][near]] / [[id:c1bf2c4f-2db3-49c1-94e9-7d2b3e4f1ab8][far]] buckets.
- Update the version identity when scope shifts.

* How to use this skill

1. /Anchor the context/. Identify the current version and sprint
   before doing anything else — never assume.

   #+begin_src sh :results verbatim
   ./projects/ores.codegen/doc_list.sh --type version
   ./projects/ores.codegen/doc_list.sh --type sprint --sort path | tail -5
   #+end_src

   The lexicographically-highest sprint folder is the current
   sprint. Read its =* Status= / =* Mission= before acting.

2. /Pick the right shape/ for what the user is asking. The
   [[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document types]] contract is authoritative; key choices:

   - /A new idea, no immediate plan to start it/ → a [[id:671f18e4-e09c-4b3b-bd24-d33df8ae38a6][capture]] in
     =near/= or =far/=.
   - /A coherent user-visible outcome ready to start/ → a [[id:699a8d45-b67e-4d3e-9206-24fa17c51ada][story]]
     in the current sprint.
   - /A single PR-sized unit of work that fits an existing story/
     → a [[id:31c868b9-306e-4282-ba8c-07e647021b34][task]] inside that story's folder.
   - /A new sprint or version/ → scaffold a [[id:0820b7fd-147c-4832-ac25-c043d38d5b61][sprint]] or [[id:ca4d4bde-2eab-4f0a-bc8b-7c0bcb17bc82][version]].

3. /Scaffold the doc/. Always via the codegen — never hand-written.
   See [[id:3153aa4b-23d4-4111-9710-5a4971102edc][How do I create a new v2 doc?]] for the per-type invocations.

4. /Promote a capture to a story/. The =:ID:= is preserved so links
   keep working. Move the file into the appropriate sprint folder,
   flip =#+type:= from =capture= to =story=, and follow the v2
   story contract from [[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document types]] (Goal / Status / Acceptance
   / Tasks / Decisions / Out of scope).

5. /Open and close sprints/. The lifecycle is described in
   [[id:46dfe47f-967b-4c24-95bf-3e21f97cfd20][Agile process]]: planning ([[id:f5426198-acaf-41da-a043-ff15785e4a1d][planning phase]]), execution ([[id:227958a5-0f78-4cc6-a116-b83988592b68][execution
   phase]]), closure ([[id:7d72a9da-984f-4130-bff6-97b66b233907][closure]] — release notes, release cut,
   post-mortem). Owned by [[id:8f13ba75-edaa-46cc-9be2-e356d60bf4f4][System 3]].

6. /Prioritise/. The near/far split /is/ the prioritisation
   mechanism: items in =near/= are candidate work for the next
   version, items in =far/= are longer-term. Re-bucket by moving
   files between =near/= and =far/= (the =:ID:= is preserved). For
   ordering inside a sprint, edit the =* Stories= table of the
   sprint document.

7. /Update the version identity/ when scope shifts. The version's
   =* Identity= section is the mini-mission that filters candidate
   stories — see [[id:83bc06e5-77dd-4aba-87cb-48a10ec0208a][version identity]] in the glossary.
