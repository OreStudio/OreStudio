* Goal

Implement read and write paths for currency data in ORE's XML format,
seeded with the currency input data from the upstream ORE examples.

* Acceptance

- Currency XML from the ORE examples parses into the project's
  in-memory model without loss.
- The model can be serialised back to XML in ORE's expected shape.
- The XML round-trip is covered by at least one test case.

* Context

Sibling to /Add database support for currency/ — together they form
the import-to-storage half of the currencies-end-to-end story.

* Notes

- Total time spent in v0: ~6:32 (the largest single entry in sprint 1;
  established the XML toolchain that later types reused).
- The choice to parse ORE XML directly, rather than via ORE's own C++
  types, paid off later when codegen took over schema generation.

* Result

- Working XML import + export for currencies, used by every later
  reference-data type as the pattern template.
