* Result

Renamed =book.ledger_ccy= to =book.functional_currency= throughout:
domain model, generated SQL/C++/Qt layers, hand-written references in
=ores.ore=/=ores.synthetic=, and living documentation. All acceptance
items met; unit tests and manual QA both passed.

Manual QA surfaced a real environment bug unrelated to the rename's
correctness: =ccache= served stale compiled object code for several
=book_*.cpp.o= files in =ores.refdata.core.lib= (still emitting
=ledger_ccy= in SQL despite clean, correctly-regenerated source),
causing the running service to query a column the schema no longer
had. Root-caused by comparing per-object-file =strings= output
against source, then fixed by clearing the affected build
directories and rebuilding with =CCACHE_DISABLE=1=. Not a code
defect from this task, but worth knowing this class of failure mode
exists in this environment when a rename changes generated code.

Verified end to end: DB recreate → Barclays Plc provisioning →
services rebuilt clean (ccache disabled) → manual scenario
=scenario_verify_book_functional_currency_rename.org= PASSED (5/5
