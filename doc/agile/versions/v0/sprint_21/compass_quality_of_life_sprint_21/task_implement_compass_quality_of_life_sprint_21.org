* Goal

Make =compass search= genuinely useful as a one-stop research tool for
LLM agents: results should arrive pre-bucketed by relevance dimension,
each hit should carry enough context (description + inline see-also
links) that the agent can decide whether to dig deeper without a
follow-up =compass show=, and historic agile noise should never drown
out current actionable content.

* Acceptance

- Results are split into three labelled buckets rendered in order:
  1. *Recipes* — how-to docs, runbooks, skills
  2. *Now* — current sprint stories, STARTED/BACKLOG tasks, open captures
  3. *Historic* — closed sprints, DONE/ABANDONED tasks, merged PRs
- Each bucket is capped at N hits (default 5); a trailing count shows
  how many were omitted (e.g. "(+3 more)").
- Each hit includes an inline *See also* block listing the top outgoing
  links from that document (title + =compass show <UUID>= hint), capped
  at 3 entries, shown only when outgoing links exist.
- Historic bucket is collapsed by default (shown only with =-v= or when
  the other buckets are empty) to reduce token cost.
- A =-v= / =--verbose= flag restores the current flat unsorted output
  for debugging and completeness checks.

* Result

Delivered in commit =d70fc6bfa=. =compass search= now renders results
in four labelled buckets — Recipes & how-to, Knowledge, Now (current
sprint + captures), Past sprints — plus a dynamic tail for any
doctype not covered by the fixed buckets. Each bucket independently
selects its top-N hits from a large fetch pool (20× the limit),
so historic agile noise can no longer crowd out current actionable
content.

Two-pass title ranking was added to improve relevance for natural-
language queries: a /core-words pass/ (question verbs stripped) ranks
docs whose title/description contains the essential noun phrase; a
/full-words pass/ ranks the remaining query words; body-only FTS hits
fall last. AND semantics (={title description} : w*=) ensure every
query word must appear in title or description — the earlier OR approach
inflated core:N ranks with irrelevant single-word hits.

Each hit shows a =[core:N]=, =[full:N]=, or =[body]= score label inline
so ranking behaviour is debuggable without reading source.
