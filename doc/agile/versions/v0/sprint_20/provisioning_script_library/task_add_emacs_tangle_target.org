* Goal

Add a CMake target to ores.shell that runs emacs in batch mode to org-babel-tangle every org source in scripts/ into its .ores output, so tangled scripts are reproducible build artefacts that never drift from their literate sources. Emacs is assumed present in a correctly provisioned developer environment (per project conventions, no defensive degradation); record it as a required tool in the environment bootstrap documentation.

* Result

=tangle_shell_scripts= added to the root CMakeLists alongside the
sibling emacs targets (the design said "a CMake target of ores.shell";
root-level placement follows the established convention for all emacs
batch targets — =tangle_codegen_templates=, =tangle_clang_format=,
=deploy_*=). =ores-build-shell-scripts.el= clones the codegen-templates
tangler: every org under =projects/ores.shell/scripts/= except
README.org, failing the build on any tangle error. Verified: the
target tangles both library scripts and the output is byte-identical
to the committed artefacts across two runs. Emacs recorded as a
required tool in the bootstrap-developer-environment runbook; the
README's Tangling section now names the target.
