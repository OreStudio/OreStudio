* Result

=provision_all.org= → =provision_all.ores= verified end-to-end: from
a fresh database, =compass shell= + =load= completes all 9 commands
error-free — system provisioned (=super_admin=), tenant provisioned
(bundle =base=, 31 datasets, seed-42 synthetic organisation, 5
parties associated), root party =Lloyds Wealth Management Ltd=
provisioned (counterparties, organisation bundle, all 22 report
definitions, activation).

The first live run of this path surfaced and fixed four real bugs:

1. /Shell:/ =wait_for_instance= declared success when all /visible/
   steps completed — but steps are created incrementally, so a
   31-dataset publish "completed" at 7. Callers now pass the expected
   step count (=datasets_dispatched=), like the GUI's pre-seed;
   =workflow wait= gains =--expect-steps=.
2. /Shell:/ a steps query immediately after dispatch can return "not
   found" — now tolerated within the transient-failure budget instead
   of failing the wait.
3. /Schema:/ the synthetic service had only SELECT on refdata tables;
   it writes the organisation it generates, so it now has DML
   (=iam_service_db_grants_create.sql=).
4. /Synthetic service:/ generated entities were stamped
   =modified_by: system=, rejected by username validation — audit
   fields now flow from the authenticated request into the generation
   options; and the publisher set the party RLS context /after/
   inserting party-counterparty links, so the requester's JWT-scoped
   visible-party set rejected them — context now switches before any
   party-scoped insert, keyed on the generated root party.

Also fixed en route: the script uses =--tenant-hostname default= —
the wizard's single-tenant default (=localhost=) collides with the
seeded Root Tenant's hostname (captured:
wizard_single_tenant_hostname_collision; gemini's PR #1117 review
had suggested the =tenant_admin@default= principal — declined then on
wizard fidelity, vindicated by the seed change). A second capture
records that synthetic parties are born Active, bypassing the party
setup flow.

Unit tests green (shell 23/71, synthetic core 59/1494).
