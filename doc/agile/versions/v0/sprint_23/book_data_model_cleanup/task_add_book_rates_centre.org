* Plan

Modeled =rates_centre_code= on =ores.refdata.book.org= as a nullable
soft-FK text column (mirroring =counterparty=/=party=/=business_unit='s
existing =business_center(_code)= fields and their shared
=ores_refdata_validate_business_centre_fn= trigger), then generated
via =compass codegen entity generate book= (not =codegen regenerate
--component=, which regenerates every entity in the component and
produces large unrelated diffs — see below).

Wired the Qt side as a =flagged_combo= detail field
(=flag_source: business_centre=), which uses codegen's already-built
generic inline combo path (only =currency= has a bespoke shared
helper). This surfaced two library functions codegen expects but
didn't yet exist anywhere in the codebase for the =business_centre=
flag source -- =fetch_business_centre_codes()=
(=ores.qt/LookupFetcher.hpp=/=.cpp=) and =business_centre_flag_icon()=
(=ores.qt/FlagIconHelper.hpp=/=.cpp=) -- added both, mirroring the
existing =currency= equivalents.

Along the way, used =codegen regenerate --component refdata-cpp
--profile all-cpp/qt= once by mistake (following a stale recipe) before
discovering =compass codegen entity generate <entity>= is the correct,
already-existing entity-scoped tool; that mistake produced a 300+-file
diff across unrelated entities that had to be discarded (and once
accidentally reverted this task's own hand-written =LookupFetcher=/
=FlagIconHelper= changes, since the discard filter matched on filename
containing "Book"). Documented the whole flagged-combo-field process,
and filed follow-up captures for the two gaps this surfaced (see
Result).
