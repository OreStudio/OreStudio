This document provides a reference for the ORE Studio messaging protocol.
All inter-service and client-server communication uses [[https://nats.io][NATS]] with JSON
payloads. The binary protocol stack (=ores.comms=) has been replaced entirely
by NATS request/reply and pub/sub.

* Messaging Transport

ORE Studio uses NATS as the sole messaging transport:

- *Request/reply*: synchronous operations (queries, mutations) use
  =nats::service::client::request()=.
- *Pub/sub*: asynchronous notifications use JetStream durable subjects.
- *Subject prefix*: in multi-instance deployments, subjects are prefixed with
  =ores.{tier}.{instance}= (e.g. =ores.dev.local1=). Leave empty for single
  instance or when subjects are managed externally.

Payloads are JSON, serialised with =rfl::json=.
