* Context

The component is called a *broker*: it routes messages by type range to the
appropriate service (routing), and for sensitive workflows optionally persists to
pgmq before dispatching (durable delivery). Both responsibilities are first-class.
Project name: =ores.mq.broker=. Namespace: =broker/=.

The current =ores.comms.service= is a monolith hosting all domain handlers in one
process. The broker architecture introduces =ores.mq.broker= as a central hub;
sensitive workflows optionally write to pgmq for durable delivery before forwarding.

Sessions are no longer keyed by TCP remote address. The JWT from Plan 1 is
included in every message frame; services validate it independently and load
=visible_party_ids= from a cached DB session lookup.

* Phase 0: JWT in Frames — Incremental Validation (no broker required)

This phase ships immediately after Plan 1. It tests JWT-in-frame under real traffic
against the existing =ores.comms.service= before the broker exists. No broker code
is written yet.

** =ores.comms/include/ores.comms/messaging/frame.hpp=
- Rename =reserved2= (bytes 28–31) to =jwt_size= (uint32)
- Reading: after 32-byte header, if =jwt_size > 0=, read those bytes as the JWT string
  before reading the payload
- Writing: populate =jwt_size= and write JWT bytes between header and payload
- Unauthenticated messages (ping, handshake, login_request) set =jwt_size = 0=

** =ores.comms/src/net/connection.cpp=
- Update =read_frame()= / =write_frame()= to handle jwt_size bytes
- Update CRC calculation to cover header + jwt_bytes + payload

** =ores.comms/include/ores.comms/service/auth_session_service.hpp=
- Add =validate_jwt(jwt_string) -> std::expected<session_data, jwt_error>= using
  RS256 verifier (public key from config)
- On JWT present: validate signature, extract session_id claim, load
  =visible_party_ids= from DB (LRU-cached by session_id)
- Keep existing remote_address map as fallback during migration

** =ores.comms/src/messaging/message_dispatcher.cpp=
- Extract JWT from frame if =jwt_size > 0=; validate and cache session
- Fall back to remote_address session map if no JWT present

Protocol version bump to *49.0* — frame now carries optional JWT.

** Verification of Phase 0
- =ores.qt= logs in, receives JWT in =login_response=, includes it in subsequent frames
- =ores.comms.service= validates JWT on every authenticated request — logs confirm
- Tampered JWT rejected with auth error returned to client
- Unauthenticated messages (ping, handshake) unaffected

* Phase 4: Broker Implementation (ores.mq/broker/)

** =ores.mq/src/CMakeLists.txt=
- Add =find_package(nng CONFIG REQUIRED)= and link =nng::nng= (private)

** New files

=include/ores.mq/broker/broker_config.hpp= — endpoints, RS256 public key path, TLS settings,
durable message type ranges (optional pgmq write before dispatch)

=include/ores.mq/broker/routing_table.hpp= / =src/broker/routing_table.cpp=:
- Thread-safe map: =message_type_range → nng_pipe_id=
- =register_route(range, pipe_id)=, =deregister_pipe(pipe_id)=, =find_pipe(message_type)=

=include/ores.mq/broker/service_registry.hpp= / =src/broker/service_registry.cpp=:
- Tracks: service_name, assigned_id, handled_ranges, connected_at, pipe_id

=include/ores.mq/broker/nng_broker.hpp= / =src/broker/nng_broker.cpp=:
- Frontend: raw-mode REP NNG socket (concurrent clients, pipe IDs preserved)
- Backend: raw-mode REQ NNG socket per registered service
- Dispatch loop:
  1. Receive frame from frontend
  2. Read 2-byte message type (after 32-byte header + jwt_size bytes)
  3. Look up service pipe in routing_table
  4. If =broker_config.durable_ranges= matches: write to pgmq via =mq_service::send()=
  5. Forward raw frame to service backend socket
  6. Forward raw response back to client
- NNG pipe callbacks: on pipe close → =routing_table.deregister_pipe(pipe_id)=
- Handle 0xC000 (register_service_request) internally — never forwarded
- Handle 0xC002 (token_refresh_request) internally — forwarded to IAM service

* Phase 6: Service Self-Registration

=ores.comms.service/src/app/application.cpp=:
- After server init, before =server.run()=: connect NNG client to broker backend endpoint
- Send =register_service_request= with all registered subsystem ranges
- On =register_service_response= success: proceed to server.run()
- On failure: log fatal and exit

* Verification

1. Build: =cmake --build --preset linux-clang-debug= — full build passes
2. Start broker: =ores.mq.broker --frontend tcp://0.0.0.0:7001 --backend tcp://0.0.0.0:7002=
3. Start =ores.comms.service --broker tcp://localhost:7002= — registers, log confirms
4. =ores.qt= connects to =tcp://localhost:7001= — login succeeds, JWT in subsequent frames
5. Execute a queue operation — routes to MQ service, response returns correctly
6. Durable path: send a trade booking message — pgmq entry created, archived after ack
7. Kill =ores.comms.service= — broker logs route deregistration for its ranges
8. Restart service — re-registration succeeds, routing resumes
