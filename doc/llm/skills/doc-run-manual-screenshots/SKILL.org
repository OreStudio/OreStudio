* When to use this skill

When a manual chapter is content-complete (per [[id:E8E892B8-184D-474A-839F-FFCA1A5148EF][doc-add-entity-chapter]])
but still carries =[SCREENSHOT NEEDED]= placeholders that need turning
into real, correctly-cropped images before the chapter is ready for
human review.

* How to use this skill

Run [[id:43A1EE0A-374D-4EA1-BC72-BADDDC525D20][How do I capture screenshots for a manual chapter?]] end to end. The
first four steps are prerequisites, non-negotiable and in order —
never skip straight to writing the scenario, and never capture against
a build or database that predates the entity's current state:

1. *Ask the user whether a new build is necessary.* If yes, build
   before continuing.
2. *Check the database is recent* (=compass db status=) — recreate it
   only if drift/staleness says so, not unconditionally.
3. *Provision the Barclays Plc tenant yourself*, via =compass shell -f
   barclays_system_provision.ores= — never hand this to the user. Any
   failure here is usually a real bug; diagnose and fix it, redeploy,
   and re-run from a fresh =db recreate= until it completes cleanly.
4. *Confirm the chapter itself already builds* (=compass build
   --direct manual=) before capturing anything, so a broken link isn't
   mistaken for a capture problem later.

Then capture and wire in the screenshots:

5. Scaffold a =test_scenario=, one step per =[SCREENSHOT NEEDED]=
   placeholder, in the chapter's top-to-bottom order — the capture-step
   count must equal the placeholder count exactly. Every step title is
   five to seven words; every step body is a bullet-point checklist
   (copying the placeholder's =# Capture:= instruction, adjusted only
   for concrete data that differs) giving the tester all the context
   to execute it, not prose. Add the two closing review steps (read
   the chapter in the Qt client's own help viewer; check the built
   PDF) — mandatory, not counted against the placeholder count.
6. Start the Qt client with =--open-scenario= pointing at the new
   scenario (absolute path, via =realpath= — a relative path silently
   fails to load) and ask the user to execute it — do not run the UI
   steps yourself.
7. Locate each capture, crop it tightly to the dialog or window (never
   the full desktop, never with QA Validation Runner chrome in frame),
   and copy it to =assets/images/<name>.png= following the plural/
   singular naming rule.
8. Wire the real image and caption into the chapter, replacing the
   placeholder, then rebuild both the PDF (=compass build --direct
   manual=) and the Qt-embedded help (=compass build --direct help=
   then =compass build deploy_help_qch=) — two independent pipelines,
   both must be rebuilt.
9. Close out: update the task and scenario docs, commit the chapter,
   PDF, and images together, and hand off for human review — this
   skill's job ends at a chapter that reads correctly with real images
   in place, not at marking the task DONE.
