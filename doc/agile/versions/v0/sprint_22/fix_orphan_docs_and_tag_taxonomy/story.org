* Goal

Clean up the org-roam link graph so every document with an =:ID:= is
reachable by at least one inbound link, and introduce a canonical tag
taxonomy so every filetag used in the repo is documented, described,
and validated. The work has two phases: (1) fix four orphan clusters
— the memory catalogue index, the shell-provisioning manual chapter,
the codegen sample-composite test fixtures, and the trading
payment-frequency entity; (2) create a tag inventory document and
wire compass to reject unknown tags at CI time.

* Acceptance

- =memory.org= catalogue lists every file under =doc/llm/memory/=;
  no orphans remain in that subtree.
- =chapter_shell_provisioning.org= is linked from =user_manual.org=
  via an =[[id:...]]= link and appears in the built HTML manual.
- =ores.sample_composite.api=, =ores.sample_composite=,
  =ores.sample_composite.core=, and =ores.trading.payment_frequency_type=
  each have at least one inbound =[[id:...]]= link; orphan detector reports
  zero findings for these documents.
- A tag inventory document exists under =doc/meta/=, lists every
  filetag in use across the repo in logical groups with descriptions,
  and is linked from =document_types.org=.
- =compass lint= (or equivalent) rejects a document whose =#+filetags:=
  contains a tag not present in the inventory.
