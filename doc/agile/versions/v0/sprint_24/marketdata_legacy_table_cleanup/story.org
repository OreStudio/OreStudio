* Goal

Retire the dead pre-codegen marketdata tables
(=ores_marketdata_series_tbl=, =ores_marketdata_observations_tbl=,
=ores_marketdata_fixings_tbl=) so the schema, RLS policies, ER
diagram, and schema-validator ignore list only describe tables that
are actually in use — and stop the codegen-managed
=market_observation= repository/service from silently losing its
hand-written =series_id= filter and pagination logic every time
=compass codegen regenerate= runs.

Surfaced while investigating [[id:82CB5F03-AFD9-4AE6-95EF-57EB683FB1B5][Hotfix: market_fixing entity carries audit
columns absent from lean SQL schema]] (Sprint 21): the legacy tables
have zero C++ references and are superseded by the codegen
=market_series=/=market_observations=/=market_fixings= entities, but
were never decommissioned.

* Decisions

- Preferred fully migrating =market_observation='s hand-written
  =series_id= filter to codegen's existing =list_by= foreign-key facet
  over merely protecting the override in place: every real caller
  always filters by series (no genuine "list all" use case exists), so
  this is the entity's actual list mode, not an optional filter bolted
  onto the generic =.list= endpoint.
- Added a new, reusable =:list_by_default_limit:= codegen property
  (per foreign key) so a =list_by= facet's default page size need not
  always be the generic 100 — set to 1000 for =series_id= given market
  observations are high-volume tick data. Available to any future
  entity/foreign-key with the same high-volume-list shape.
- While migrating, also moved =market_observation= off the legacy
  hand-wired =registrar.cpp= subscription pattern onto the generated
  per-entity sub-registrar pattern already used elsewhere (e.g.
  =ores.synthetic=) — fixing a latent bug where its =history= endpoint
  was never actually subscribed. Not itself in this story's acceptance
  criteria, but a natural inclusion since all five facets were already
  being regenerated.
- A chart-usability gap surfaced by this story's QA pass (FX Spot
  chart plots its whole retained history with no default time window)
  was deliberately kept out of this story's scope — filed as its own
  capture, [[id:7F5BE078-A845-4E90-B470-268F23B9DB46][Shared chart-data
  presentation layer for FX Spot and CRM plots]], since it's
  chart-rendering UX unrelated to the codegen/override problem this
  story solves, and applies equally to CRM plots.
