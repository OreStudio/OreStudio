Wire party onto the =nats-event-cache= facet for real: set
=cached_by: iam=, =read_for_cache: true= on party's model, generate
its cache under =ores.iam=, retire the hand-written
=ores.iam.core/service/party_cache.hpp= prototype, and repoint every
consumer (registrar warm-up/subscribe wiring, =auth_handler=,
=account_handler=, =bootstrap_handler=) at the generated version.
Confirms the facet design end-to-end on a real entity, including the
parent/child aux index =compute_visible_party_ids= depends on.

* Acceptance

- party's model sets =cached_by: iam=, =read_for_cache: true=,
  =cache_aux_type: children_map=, and enables the facet.
- The generated =party_cache.hpp=/=party_cache_registrar.hpp= replace
  the hand-written =ores.iam.core/service/party_cache.hpp= (deleted).
- =compute_visible_party_ids= keeps working via a new, generic
  aux-index mechanism on the =nats-event-cache= facet (see below) —
  not a party-only special case.
- =registrar.cpp=, =auth_handler.hpp=, =account_handler.hpp=,
  =bootstrap_handler.hpp= all build against the generated cache's API
  (=lookup=, unchanged =load=/=compute_visible_party_ids=).
- No regression: unrelated entities regenerate byte-identical; party's
  other (pre-existing, drifted) facets are unaffected.

* Plan

1. Investigate real consumers of the hand-written =party_cache=
   (=auth_handler=, =account_handler=, =bootstrap_handler=,
   =registrar.cpp=) to find the full API surface to preserve —
   discovered =compute_visible_party_ids= depends on a parent/child
   aux index the facet didn't yet support (scope gap, confirmed with
   the user: extend the facet now rather than defer or partially
   migrate).
2. Extend the =nats-event-cache= facet with a generic, entity-agnostic
   aux-index mechanism: a =cache_aux_type= flag plus three paste
   points (type declaration, build-from-entries, extra methods),
   mirroring the existing =custom_subscriptions= paste-point
   convention. Add =partitioned_cache::aux_type= alias (Layer A,
   additive) to support it.
3. Author party's three paste-point blocks (children_map type,
   parent-id-keyed build, iterative =compute_visible_party_ids=)
   reproducing the hand-written prototype's behaviour exactly.
4. Set party's model flags for real (=cached_by=, =read_for_cache=,
   =cache_aux_type=, facet-enabled); generate the two files under
   =ores.iam=.
5. Rewire =registrar.cpp= to call the generated
   =warm_and_subscribe_party_cache=; repoint the three handler headers
   at the new include path and =.lookup(...)= method name; delete the
   hand-written prototype.
6. Verify: regen unaffected entities byte-identical; party's other
   facets' pre-existing drift unchanged; codegen test suite green; no
   stray references to the old path/method name.
