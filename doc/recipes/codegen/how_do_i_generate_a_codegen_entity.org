#+title: How do I generate a codegen entity?
#+description: Use compass codegen entity generate to regenerate all applicable profiles for an entity. Supports --address to restrict to one facet (per-layer commands table included) and --dry-run to preview paths.
#+type: recipe
#+level: cross
#+filetags: :codegen:compass:entity:recipe:
#+created: 2026-06-26
#+updated: 2026-06-26

For background on profiles (facets) and metatypes, see
[[id:08FBD248-257A-A474-DD43-DB08949978F1][ores.codegen]].

* Question

How do I regenerate all codegen output for an entity?

* Answer

Run =compass codegen entity generate <name-or-id>= from the project root:

#+begin_src sh
./compass.sh codegen entity generate country
#+end_src

The entity resolves by name or org-roam UUID prefix. All applicable leaf
facets are run automatically (e.g. =domain=, =repository=, =service=,
=sql=, =qt=, =protocol=, …):

#+begin_example
Generating country (domain_entity, refdata) — 12 profile(s): domain, generator,
  nats-eventing, nats-handler, non-temporal-domain, non-temporal-repository,
  non-temporal-sql, protocol, qt, repository, service, sql
#+end_example

*Restrict to one facet* with =--address= (the current, non-deprecated flag;
=--profile= still accepts the same address strings but prints a deprecation
warning and will be removed — see =projects/ores.compass/src/compass_codegen_entity.py=):

#+begin_src sh
./compass.sh codegen entity generate country --address ores.sql.schema
./compass.sh codegen entity generate country --address ores.cpp.domain
./compass.sh codegen entity generate country --address ores.cpp.qt
#+end_src

*One command per layer* — the facet addresses you'll reach for most often
when regenerating a single layer (e.g. during a commissioning task):

| Layer                    | Facet address                 | What it writes                             |
|--------------------------+--------------------------------+---------------------------------------------|
| SQL schema               | =ores.sql.schema=              | table DDL, insert/update triggers, indexes |
| SQL populate             | =ores.sql.populate=            | seed-data DML                              |
| C++ domain struct        | =ores.cpp.domain=              | domain struct + JSON I/O                   |
| C++ repository           | =ores.cpp.repository=          | repository class (sqlgen queries)          |
| C++ service              | =ores.cpp.service=             | service class wrapping the repository      |
| C++ NATS protocol        | =ores.cpp.protocol=            | request/response message structs           |
| C++ NATS handler         | =ores.cpp.nats-handler=        | message handler class                      |
| C++ NATS sub-registrar   | =ores.cpp.nats-sub-registrar=  | per-entity subject registration            |
| C++ NATS eventing        | =ores.cpp.nats-eventing=       | domain-changed event struct                |
| C++ generator (synthetic)| =ores.cpp.generator=           | synthetic-data generator function          |
| C++ Qt UI                | =ores.cpp.qt=                  | Qt list/detail/history windows and models  |
| C++ enum                 | =ores.cpp.enum=                | enum type + string conversion              |
| CMake component wiring   | =ores.cmake.component=         | component CMakeLists.txt entries           |

=--address= (like =--profile=) accepts only one facet per invocation — there is
no accumulating flag. To touch several layers without regenerating the whole
entity, run the command once per address:

#+begin_src sh
./compass.sh codegen entity generate counterparty_identifier --address ores.cpp.repository
./compass.sh codegen entity generate counterparty_identifier --address ores.cpp.service
./compass.sh codegen entity generate counterparty_identifier --address ores.cpp.protocol
./compass.sh codegen entity generate counterparty_identifier --address ores.cpp.nats-handler
./compass.sh codegen entity generate counterparty_identifier --address ores.cpp.nats-sub-registrar
#+end_src

*Preview paths without writing* with =--dry-run=:

#+begin_src sh
./compass.sh codegen entity generate country --dry-run
#+end_src

*Preview as a unified diff* (generate to temp, compare to disk) with =--diff=:

#+begin_src sh
./compass.sh codegen entity generate country --diff
./compass.sh codegen entity generate country --address ores.sql.schema --diff
#+end_src

=--dry-run= and =--diff= are mutually exclusive.

After generating, review changes with:

#+begin_src sh
./compass.sh codegen entity diff country
#+end_src

* Script

=compass codegen entity generate= (alias =gen=) is the =_cmd_generate=
handler in [[proj:projects/ores.compass/src/compass_codegen_entity.py][=projects/ores.compass/src/compass_codegen_entity.py=]]. It calls
=_generate_single= from =codegen.generate= directly (no subprocess to
=codegen.sh=).

* Tested by

Run =./compass.sh codegen entity generate country --dry-run= and verify
36 output paths are printed across 12 profiles. Run without =--dry-run=
and verify the stale files are updated (=entity show= should then show ✅).
