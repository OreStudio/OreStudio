* Acceptance

- pr-gate is the only required check on main.
- A doc-only PR merges without --force or --admin.
- Recipes and skills name the new checks.

* Plan

Audited all docs referencing CI check names (monitor-until-green,
fix-failing-check, pr-show-checks) — all three were already generic,
using =compass pr checks <N>= without hardcoded check names; no edits
needed. Audited =compass pr merge= guard: it uses =gh pr checks= exit
code, not check names; docstrings mentioned the old "required check
that never reported on a doc-only PR" explanation — updated to drop
the now-stale example. Updated the how-do-I-merge-a-PR recipe's
Script section similarly. Applied the GitHub branch protection change
via the API: removed =roundtrip=, added =pr-gate= as the sole
required check.

* Notes

- /claude-review must not be a required check/: the action's
  anti-tampering guard fails the run on any PR that modifies the
  review workflow itself (the branch copy must byte-match the
  default branch's). Seen live on PR 1152 after the yamllint
  normalisation — red claude-review until merge, self-healed after.
  pr-gate must exclude it, and future workflow edits should expect
  one red review run on their own PR.
