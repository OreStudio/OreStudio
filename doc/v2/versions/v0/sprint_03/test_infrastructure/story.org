* Goal

Bring the test infrastructure to a state where (a) tests don't share
state with the dev database, (b) test data is built from generators
rather than ad-hoc fixtures, and (c) each Catch2 suite shows up as
its own ctest entry on the dashboard.

* Acceptance

- A dedicated test database, separate from the application database.
- A library of generators that build representative test inputs.
- Each Catch2 test suite registered as a ctest entry.
