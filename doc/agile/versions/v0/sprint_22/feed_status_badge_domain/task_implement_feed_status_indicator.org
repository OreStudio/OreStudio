* Plan

Abandoned badges (see story Decisions). Implemented icon+text inline
in the Status column instead:

- Added three new =Icon= enum entries to
  =projects/ores.qt/api/include/ores.qt/IconUtils.hpp= /
  =IconUtils.cpp=: =FeedLive= (ic_fluent_live_20 / soundwave.svg),
  =FeedStale= (ic_fluent_hourglass_20 / hourglass.svg), =FeedPending=
  (ic_fluent_radio_button_20 / record.svg). Reused the existing
  =Icon::PlugDisconnected= for Disconnected — it already pairs
  =ic_fluent_plug_disconnected_20= with =link-broken.svg=, exactly
  the picks made independently for this story.
- Copied the FluentUI (20px, regular+filled) and Solarized
  (Bold+Linear) SVG assets into
  =projects/ores.qt/application/resources/icons/= and registered
  them in =resources.qrc=.
- Rewrote =FxSpotGridWindow.cpp=: replaced =make_badge=/=apply_badge=
  (QLabel with a hardcoded-CSS pill stylesheet) with
  =make_status_indicator=/=apply_status_indicator= — an icon
  =QLabel= (recolored via =IconUtils::createRecoloredIcon=) plus a
  text =QLabel=, laid out in an =QHBoxLayout=, no pill background.
  =RowState::badge= became =statusIcon=/=statusText=.

Caught and fixed one mistake during implementation: copying Solar
assets by filename silently overwrote =link-broken.svg=, which was
already vendored and in use by =Icon::PlugDisconnected= elsewhere in
the app, with a different (stroke-based) rendering from a newer
local checkout of the solar-icons source repo. Reverted that file to
its committed version before the final build; the vendored copy
(fill-based paths) is what ships.

Verified: =ores.qt.mktdata.lib= builds clean.
