** File groups Connectivity, Current User, and System submenus

Open *File*. Confirm, in order: *&Connectivity* (submenu), *Current
&User* (submenu), *S&ystem* (submenu), separator, Exit.

Open File > Connectivity. Confirm it contains, in order:
Connections..., separator, Connect, Disconnect.

Open File > Current User. Confirm it contains My Account... and My
Sessions.... Open both and confirm they still work.

Open File > System. Confirm it contains, in order: System Settings,
separator, Test Scenario Runner, separator, Reset System.... Open
System Settings and confirm it opens. Open Test Scenario Runner and
confirm it opens (this is the window you're using to run this very
scenario).

** Operations gains Telemetry, Message Queue, User Accounts

Open *Operations*. Confirm it now contains, alongside the existing
*Scheduler* (Scheduled Job Definitions, Scheduled Job Instances,
Scheduled Job Monitor) and *Workflows* sections: *Message Queue*
(Queue Monitor), *Telemetry* (Log Viewer, Event Viewer, Shell,
Protocol, Settings — plus Service Dashboard as the first item), and
*User &Accounts* (Accounts, Roles, Tenants, Tenant Types, Onboard
Tenant...). Open one entry from each and confirm the correct
window/dialog opens. Confirm *User Accounts* is clearly
distinguishable from the personal *Account* actions in File
(different labels: "User Accounts" vs "My Account"). Confirm there
is no separate *Configuration* submenu — System Settings lives in
File > System instead (previous step).

** Data Management is retired; Coding Schemes/Audit Trail/Data Catalogue moved to Data Quality

Confirm there is no more top-level *Data Management* menu. Open
*Data &Quality* and confirm it contains: *Badges* (Badge
Definitions, Badge Severities), a *Coding &Schemes* submenu (Code
Domains, Coding Schemes, Coding Scheme Authority Types — named
"Coding Schemes" rather than "Classifications" specifically to not
collide with Reference Data > Codes' unrelated Classifications
submenu), an *Audit Trail* submenu (Change Reason Categories, Change
Reasons), and a *Data Catalogue* submenu (Data Domains, Subject
Areas, Catalogues, Datasets, Dataset Bundles, Methodologies, Origin/
Nature/Treatment Dimensions). Open one entry from each and confirm
the correct window opens.

*** Result

| Field  | Value |
|--------+-------|
| Status | PASS |

** Trading Codes is retired; Purpose Types moved to its own Portfolio Codes submenu

There is no more *Trading &Codes* submenu anywhere (it used to sit
nested inside the *Trading* menu with a single entry, Purpose
Types). Open *Trading* and confirm it now ends at *Trades* — no
Trading Codes submenu after it.

Instead, open *Reference Data > &Codes > &Portfolio Codes* and
confirm *Purpose Types* is there — its own submenu, not inside Book
Codes, since =purpose_type= classifies Portfolios
(=portfolio_entity.purpose_type=), not Books, despite the similarly
named =book_purpose_type= being a distinct entity. Open it and
confirm the correct window opens.

| Status | PASS |

** Synthetic groups its data-generation entries under Configuration

Open *Market Data > &Synthetic*. Confirm *Market Simulator* sits
directly under Synthetic, and a *&Configuration* submenu holds
Generation Configs, FX Spot Configs, GMM Components, IR Curve
Generation Configs, IR Curve Template Entries, and Yield Curve
Process Types. Open one entry from Configuration and confirm the
correct window opens.
