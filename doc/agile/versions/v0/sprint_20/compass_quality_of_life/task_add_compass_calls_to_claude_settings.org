invocations. Review the Claude Code settings source

* Result

Delivered in PR [[https://github.com/OreStudio/OreStudio/pull/1125][#1125]] (merged). The compass allow patterns were
already correct (space-star ≡ =:*= per the Claude Code docs); the
prompts had three other causes, all fixed in
=doc/llm/claude_code_settings.org= and regenerated via =compass build
settings=: the deployed =settings.json= was stale; compound commands
evaluate per subcommand, so pipelines prompted on uncovered
=head=/=tail=/=cat=/=ls=/=cut=/=tr=/=diff= (now allowed); and git
fails in-sandbox against the linked-worktree gitdir, prompting on
every retry — a new =sandbox.excludedCommands= block routes the four
compass forms and =git *= straight to the auto-approving permission
flow. Review round 1 removed =sed=/=awk= from the proposed grants as
arbitrary-execution vectors; the =python3 *=/=find *= precedent is
flagged for a future least-privilege pass. The original capture's
"no full path" ask is served by the root =./compass.sh= wrapper.
