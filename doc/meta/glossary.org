* Task
:PROPERTIES:
:ID: 31c868b9-306e-4282-ba8c-07e647021b34
:END:

The smallest unit of executable work. One task = one PR. Bounded by a
concrete size limit (see [[id:8b29a8d8-e004-4e87-b81d-abeaf68f2b98][document types]]). Owned by [[id:dbfd530b-4b56-48d1-9f03-3a0d0da16552][System 1]]. Create
via [[id:b7fa729c-2d42-408a-9424-b6aa90eb16b4][New task]].

* Story
:PROPERTIES:
:ID: 699a8d45-b67e-4d3e-9206-24fa17c51ada
:END:

A coherent user-visible outcome, composed of one or more tasks. The unit of
sprint planning. Owned by [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]]. Create via [[id:6d6ddb4b-a0aa-42e6-a294-0f9b35cc0752][New story]].

* Sprint
:PROPERTIES:
:ID: 0820b7fd-147c-4832-ac25-c043d38d5b61
:END:

A bounded period of work delivering a set of stories aligned to a sprint
mission. Owned by [[id:8f13ba75-edaa-46cc-9be2-e356d60bf4f4][System 3]]. Create via [[id:f56fba9b-a3c4-4c7c-8a15-bc9c3cc98a12][New sprint]].

* Sprint mission
:PROPERTIES:
:ID: 198da730-d9fb-4bde-b1c0-b930cd019917
:END:

A one-sentence statement of what the sprint exists to achieve. Used to
filter candidate stories. Owned by [[id:8f13ba75-edaa-46cc-9be2-e356d60bf4f4][System 3]].

* Version
:PROPERTIES:
:ID: ca4d4bde-2eab-4f0a-bc8b-7c0bcb17bc82
:END:

A major release target (e.g. 1.0). A coherent set of sprints. Owned by
[[id:6ca8b181-bd81-4ef4-af27-64f5491d62f0][System 4]]. Every version of the project has an entry in
[[id:37c10313-e6e9-4d42-b82d-b6758d0a3ad9][Versions]]. Create via [[id:8fc15ea1-ab30-47be-993f-ed624a57f761][New version]].

* Version identity
:PROPERTIES:
:ID: 83bc06e5-77dd-4aba-87cb-48a10ec0208a
:END:

The mini-mission of a version: what it exists to deliver, what makes it
distinct. Owned by [[id:6ca8b181-bd81-4ef4-af27-64f5491d62f0][System 4]].

* Product identity
:PROPERTIES:
:ID: 436fb40c-422a-465b-b949-e9f0760820c1
:END:

The durable statement of what ORE Studio is and is not. Used to filter
candidate versions, far stories, and feature requests. Owned by [[id:6df00a64-7cc3-40a7-950b-55d2b05a31c7][System 5]].
Create via [[id:f1311596-b8ed-43ad-8c46-76c58c0194c7][New product identity]].

* Function
:PROPERTIES:
:ID: 66f2a9e8-0385-4d56-b4b1-0957208990dc
:END:

A named role that operates the system — /what someone (or some LLM
session) does when they are wearing a given hat/. Each function is one
document describing the role's purpose, the level it operates at, its
inputs, its outputs, the recipes and skills it draws on, and what it
explicitly does /not/ do. Functions are role-shaped (descriptive); a
[[id:4f6384fe-cde9-40f6-b13d-8a84b6cd77f7][skill]] is task-shaped (operational). When an LLM session takes on a
function, the function doc is the priming context; the skills it lists
are the operational tools it then uses. Examples: the six per-system
roles ([[id:b544713a-ba23-4d5a-9cee-084212d7c8ba][S1 Agent]] / [[id:eeca6981-3240-44d9-ada7-4b8c64a76b44][S2 Orchestrator]] / [[id:f1d99662-c6b1-4839-92e1-d061ccf6b83f][S3 Sprint Planner]] / [[id:049b086e-f131-4d7a-8244-c1b2ff297ab6][S3* Auditor]] /
[[id:66eb6824-52b8-4603-b302-f805da9ec443][S4 Version Planner]] / [[id:5cad21da-21fb-460b-9286-8998b4c6ef52][S5 Identity Steward]]) under =doc/functions/=.

* Skill
:PROPERTIES:
:ID: 4f6384fe-cde9-40f6-b13d-8a84b6cd77f7
:END:

A concrete, executable capability — /how to do a kind of work/ — in
Claude Code's skill format (=doc/skills/<slug>/SKILL.org=). Each skill
is self-contained: it describes when to use it, the procedure, and any
artefacts it expects. A skill is /task-shaped/: invokable in one
session to perform a unit of work (e.g. =semi-autonomous-developer=,
=new-sprint=, =pr-shepherd=). Contrast with [[id:66f2a9e8-0385-4d56-b4b1-0957208990dc][function]], which is
/role-shaped/: a description of what someone does, listing the skills
that role draws on. Functions compose skills; skills do not own
functions. Create via [[id:6054cc20-5f41-482f-a587-759512577b1d][New Claude Code skill]].

* Recipe
:PROPERTIES:
:ID: 117a95f9-b03d-490b-bb74-da9fe74537e2
:END:

A how-to document answering one operational question (e.g. "how do I run
the tests"). Has executable scripts where applicable. Tested by CI.
Create via [[id:0c57932d-d13a-480f-b426-49525aa8ba3d][New recipe]].

* Knowledge document
:PROPERTIES:
:ID: bd1f3458-f991-46be-abeb-5f06c2633375
:END:

A durable factual document about the domain, architecture, or components.
Not state, not process — just facts that outlive any sprint. Create via
[[id:f497aa0a-f604-436a-8555-c17b14302c99][New knowledge document]].

* Plan
:PROPERTIES:
:ID: 58d491a6-e4bd-4b0d-86bd-bb8465dfd4c7
:END:

A transient implementation strategy for a single task or story. Lives next
to the work, dies with it. Key decisions distil into a =decisions.org= on
the story before the plan is archived.

* Discovered task
:PROPERTIES:
:ID: 4ea506f5-c3e9-4f92-8024-a5eed0b3df43
:END:

Work spotted in passing while executing another task. Filed without triage
into =tasks/discovered/=; processed later by [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]] or
[[id:8f13ba75-edaa-46cc-9be2-e356d60bf4f4][System 3]].

* Backlog
:PROPERTIES:
:ID: e7acb613-01ed-4351-b704-e6c279e31157
:END:

A TODO state, not a folder. Tasks and stories in =BACKLOG= have been
triaged but not started. The /product backlog/ (work not yet pulled
into a sprint) lives separately under =agile/product_backlog/{near,far}/=
as a tree of [[id:671f18e4-e09c-4b3b-bd24-d33df8ae38a6][captures]] — see below.

* Capture
:PROPERTIES:
:ID: 671f18e4-e09c-4b3b-bd24-d33df8ae38a6
:END:

A pre-sprint idea: something we may want to do but have not yet pulled
into a sprint as a story. Captures live at
=doc/agile/product_backlog/{near,far}/<slug>.org=. The bucket is the
source of truth: =near/= = candidate work for the next version,
=far/= = longer-term wishlist.

Captures are /not stateful/ — no TODO keyword, no =* Status= table.
They have only two effective states: /pending/ (the file exists in a
bucket) and /promoted/ (the file has been converted into a =story=).
On promotion the file is moved into the appropriate sprint folder and
its =#+type:= flips from =capture= to =story=; *the =:ID:= is
preserved* so existing links keep working. Create via [[id:978e0a8f-197c-48da-a405-457e0aee91d9][New capture]].

* TODO state
:PROPERTIES:
:ID: 8a246d60-6a87-4b54-a023-564ec1965c61
:END:

The TODO keyword on a stateful document's =* Status= headline. The full
vocabulary is =DISCOVERED BACKLOG STARTED BLOCKED | DONE ABANDONED=
(active states left of the bar, terminal states right). Documents that
are not stateful (knowledge, recipes, functions, meta) have no Status
headline.

* Status headline
:PROPERTIES:
:ID: 26af1bc4-8b83-4aa4-a324-6fbebc2de960
:END:

The =* Status= headline inside a stateful document. Carries the TODO
keyword (state) and four subheadings (=** Now=, =** Waiting on=,
=** Next=, =** Last touched=) that describe the live operational
detail. Updated every time an orchestrator or agent acts on the
document.

* Predecessor story
:PROPERTIES:
:ID: cf9b011e-7ff1-4fdb-8b5f-148adf43a75f
:END:

A story in a prior sprint that the current story continues. Captured
as =#+predecessor: <id>= frontmatter plus a =Continued from: [[id:...][...]]=
line in the body. See cross-sprint stories in [[id:8810e0f9-c5b9-4cfa-a710-774ad6c771dd][lifecycle]].

* Successor story
:PROPERTIES:
:ID: 65976df1-57a0-4ba3-b6a5-a947886c20bd
:END:

A story in a later sprint that picks up where this one left off.
Captured as =#+successor: <id>= frontmatter plus a =Continued in:
[[id:...][...]]= note in =* Decisions=. Audit verifies bidirectional
linkage.

* Agent
:PROPERTIES:
:ID: 366d6521-79e0-42ce-97f4-5fd0f443a923
:END:

A subprocess forked by a [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]] orchestrator LLM to execute one
task. Operates at [[id:dbfd530b-4b56-48d1-9f03-3a0d0da16552][System 1]]. See [[id:f58ca9e7-c65f-4d17-a9a7-95fde25b9ebd][execution model]].

* Orchestrator
:PROPERTIES:
:ID: 32fbc685-b81d-4af8-9b10-294f9055bc75
:END:

The [[id:bf4f183f-bb33-44f2-a53f-d9bd602e9be8][System 2]] LLM session the user has assigned to a story. Forks
agents, reviews their output, decides what to do with discovered work.
