This page is a [[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]] in the *inbox* bucket of the product backlog — a pre-sprint idea, not yet pulled into a sprint as a story.

* What

Add HTTP REST endpoints for =contact_type= (list, get, create, update,
delete, history) once the HTTP-entity codegen profile
([[id:F556E446-EB91-4656-BEDC-15AE0F7B5ECD][Entity commissioning process reference]]'s gap protocol, "❌ GAP" row
for the HTTP layer) exists. Do not hand-code the layer ahead of the
profile — raise/complete the codegen-profile story first, then
regenerate.

* Why

=contact_type= was fully commissioned end-to-end for SQL, domain,
repository, service, protocol, NATS, and Qt during
[[id:E1F9C83E-645D-4CFC-B1C9-656651C55BCD][Commission: contact_type]], but HTTP REST support remains a codegen
gap shared by every entity, not specific to =contact_type= — tracked
here so the entity isn't forgotten once the HTTP-entity profile lands.
