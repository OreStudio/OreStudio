* Goal

When =compass where=, =compass list=, or =compass status= output is
displayed in the dashboard output pane, any file path (e.g.
[[proj:doc/agile/versions/v0/sprint_18/ores_studio_emacs_dashboard/story.org][=doc/agile/versions/v0/sprint_18/ores_studio_emacs_dashboard/story.org=]])
should be clickable: pressing =RET= or clicking it opens the file in
the same output pane, keeping the dashboard visible.

  dashboard output pane.

* Notes

** Approach options

*** Option A — compilation-error-regexp

Add a custom entry to =compilation-error-regexp-alist= that matches
the file paths in compass output (e.g. lines matching
=^\s*\(doc/[^ ]+\.org\)=).  Compilation mode then renders those as
clickable links automatically.

Pros: zero custom rendering code; integrates with existing compilation
buffer.  Cons: regexp must match compass output format exactly; format
could change.

*** Option B — post-process buffer with overlays

After the compilation buffer is populated, scan for =.org= paths and
add button overlays.  This is more robust to format changes but
requires knowing when the compilation finishes.

*** Option C — custom compass output mode

Write a small =ores/compass-output-mode= minor mode that activates in
compass compilation buffers and uses =font-lock= + =button.el= to
highlight paths.

** Preferred approach

Option A (compilation-error-regexp) if compass output has a stable,
parseable format.  If the format is irregular, fall back to Option C
(custom minor mode with font-lock buttons).

Paths should be resolved relative to the checkout root, which is
available via =ores/checkout-root= at the time the compass command is
launched.  The button action should call =ores/dashboard--display= so
the file opens in the dashboard output pane rather than a new window.
