* Goal

There is no =task move=: relocating a task between stories means
reverting the old story's wiring, re-running =compass add= with
=--id= into the new parent, deleting the old file, and hand-editing
tables. A =compass task move <slug> --story <target>= should rehome
the file, rewire both stories' =* Tasks= tables, and update the
task's parent links — preserving the task's UUID throughout.

* Acceptance

- =compass task move <slug> --story <target>= moves the task file to
  the target story's folder, UUID intact.
- The source story's =* Tasks= row is removed and a row is added to
  the target story's table.
- The task's parent links (=Parent story= row, intro id-link, story
  filetag) point at the new story.

* Plan

New subcommand: =compass task move <slug-or-uuid> --story <target>=.

Three changes to =compass.py=:

1. *=_remove_task_row_from_story(story_path, task_id)=* — find and
   extract the task's row from the source story's =* Tasks= table,
   restoring an empty placeholder row if the last data row is removed.
   Returns the raw row text so the state/start/end cells are preserved.

2. *=_cmd_task_move(task_ident, story_ident)=* — the main implementation:
   a. Resolve task (UUID/prefix/slug, same logic as =task done=).
   b. Resolve target story via =resolve_story=.
   c. Guard: same story → error; destination file exists → error.
   d. Extract the row from source story with the new helper.
   e. Rewrite the task file: replace =:old_story_slug:= in =#+filetags:=
      (and sprint/version slugs if the path hierarchy differs); replace
      all =[[id:OLD-STORY-UUID][...]]]= occurrences with
      =[[id:NEW-UUID][New Title]]=.
   f. Insert the extracted row into the target story's =* Tasks= table
      (replacing an empty placeholder if present).
   g. =shutil.move= the file to the target folder.

3. *Wire into =cmd_task=*: add a =move= subparser with =task= positional
   and =--story= required argument.
