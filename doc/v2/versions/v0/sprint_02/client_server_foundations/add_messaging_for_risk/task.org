* Goal

Use the risk module as the first real consumer of the comms layer:
define a small request/response surface, implement it on both
client and server, and prove it works end-to-end.

* Acceptance

- Risk message types defined in =ores.comms=.
- Client can send a risk request and receive a response.
- Server-side handler invokes the risk module and returns results.
- An end-to-end test exercises one request/response pair.

* Notes

- This is the first time the architecture's value is visible to a
  user-facing module; the pattern propagates to later modules.

* Result

- Risk request/response operational over cobalt.
