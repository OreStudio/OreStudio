** Summary
Add platform-agnostic null device detection to support database setup on both Unix-like systems and Windows/MINGW environments. This change enables psql output redirection (=\o=) to work correctly across different operating systems by using =/dev/null= on Unix and =NUL= on Windows.

** Changes
- *setup_database.sh*: Added platform detection logic to identify the appropriate null device (=/dev/null= vs =NUL=) and pass it as a psql variable
- *All populate SQL scripts*: Replaced hardcoded =/dev/null= with the =:null_dev= psql variable in =\o= (output redirection) commands across 80+ SQL files in the following modules:
  - crypto, dq, flags, fpml, iam, ip2country, iso, lei, ore, refdata, reporting, solvaris, testdata, variability

** Implementation Details
The setup script now:
1. Detects if =/dev/null= exists to determine the platform
2. Sets =_null_dev= to either =/dev/null= (Unix) or =NUL= (Windows)
3. Passes this value to psql via =-v null_dev="${_null_dev}"=

All SQL populate scripts use the parameterized variable instead of hardcoded paths, ensuring compatibility across platforms without requiring separate script versions.
