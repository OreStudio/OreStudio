The PR review runbook ends at: fix → commit → push → reply to comments →
resolve review threads. *Do not merge the PR.*

*Why:* The user merges PRs themselves. Attempting to merge after completing
review steps is out of scope for the runbook and was explicitly corrected
(2026-05-30, PR #940).

*How to apply:* After resolving all review threads, stop. Do not call
=gh pr merge=. The final step is resolving threads; report completion and
wait for the user to merge.
