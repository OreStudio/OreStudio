* Goal

Add codegen support for resolving a lookup entity's value as-of a
given timepoint (not just "latest"), turn it on for =book_status= and
=regulatory_book_type=, and wire the Book history dialog to use it so
a historical Book version renders its status/type badge from the
data that was actually in force at that version's =recorded_at=,
instead of always showing the current lookup value.

* Plan

1. Design the as-of query shape at the repository layer (SQL
   =WHERE recorded_at <= :timepoint ORDER BY ... LIMIT 1= per
   code, optionally unfiltered) and thread it up through
   service/protocol/NATS handler.
2. Extend the codegen templates (=cpp_domain_type_repository.*=,
   =cpp_service.*=, =cpp_protocol.hpp=, =cpp_nats_handler.hpp=, and
   their literate =.org= counterparts) to emit the as-of method set
   whenever an entity opts in.
3. Flip =:has_as_of_lookup: true= on =book_status.org= and
   =regulatory_book_type.org=, regenerate, and hand-verify the diff
   is confined to the two entities.
4. Add repository-level Catch2 tests for both entities:
   resolves-prior-version, before-creation-is-empty, and
   without-code-filter.
5. Wire a real caller: extend =ores.qt.api::LookupFetcher= with an
   as-of fetch, and use it from =BookController=/
   =BookDetailDialog= so the Book history view resolves each
   version's status/type badge from the as-of data instead of the
   latest lookup value.
6. Build the full affected surface and run
   =ores.refdata.core.tests= to confirm no regressions.

* Result

Implemented the as-of lookup facet end-to-end and piloted it on the
two entities that surfaced the gap:

- Codegen templates (=cpp_domain_type_repository.{cpp,hpp}=,
  =cpp_service.{cpp,hpp}=, =cpp_protocol.hpp=,
  =cpp_nats_handler.hpp=, and their literate =.org= counterparts)
  gained an as-of (timepoint) query path alongside the existing
  latest/paginated ones, gated behind a =:has_as_of_lookup:= flag so
  entities opt in explicitly.
- =book_status.org= and =regulatory_book_type.org= flipped the flag
  and regenerated; the resulting repository/service/protocol/handler
  diffs are confined to those two entities plus the shared templates.
- Added repository-level Catch2 coverage for both entities:
  resolving the version in force at a timepoint, an empty result
  before the entity existed, and the unfiltered-by-code path.
- Wired to a real caller: =ores.qt.api::LookupFetcher= gained an
  as-of fetch, used from =BookController=/=BookDetailDialog= so the
  Book history dialog now resolves each historical version's
  status/type badge from the as-of data rather than the current
  lookup value — the facet is proven live, not dead code.
- Verified: full build of =ores.refdata.core=, =ores.refdata.api=,
  =ores.qt.api=, =ores.qt.refdata= and =ores.refdata.core.tests=
  green — 287 assertions across 163 test cases, no regressions. An
  unrelated mid-task OOM kill (exit 137) on a build was transient and
  resolved on retry with no code change.
