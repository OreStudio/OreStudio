* When to use this skill

When the user asks to create a new ORE Studio component — a new
project under =projects/=.  Most domain components are split into
=.api=, =.core=, and =.service= parts; decide which parts are needed
before starting.  See [[id:F27E5DF7-6223-4B6F-80A5-CEFBEB1BC756][Component architecture]] for the split rules,
facet placement, and CMake dependency chain.

* How to use this skill

Codegen is *mandatory* for all boilerplate.  Do not hand-write
CMakeLists or stub files.

1. /Gather requirements/: component short name, parts needed
   (=.api=, =.core=, =.service=, or a subset), one-line brief, longer
   description.

2. /Create a JSON model per part/ under
   =projects/ores.codegen/models/COMPONENT/= using the schema in
   [[id:08FBD248-257A-A474-DD43-DB08949978F1][ORE Studio Codegen]].  Name the files
   =ores_COMPONENT_{api,core,service}_component.json=.

3. /Run codegen/ for each part:
   =cd projects/ores.codegen && ./run_generator.sh models/COMPONENT/ores_COMPONENT_api_component.json --profile component-api=
   Repeat with =--profile component-core= and =--profile component-service= as needed.

4. /Register/ each part in =projects/CMakeLists.txt= in dependency
   order (=.api= → =.core= → =.service=).

5. /Add a PlantUML stub/ (=modeling/ores.COMPONENT.PART.puml=) for each
   part — codegen does not yet produce this.  See
   [[id:B0B99088-D88E-4980-A4EE-7D6A9A8E96A5][component-model-creator]] for the diagram conventions.

6. /Build and verify/ the stub tests with [[id:654D4A70-E63D-4C18-B5A5-886066E36314][cmake-runner]]:
   =cmake --build --preset linux-clang-debug-ninja --target ores.COMPONENT.api.tests=

7. /Document the architecture/ using [[id:B0B99088-D88E-4980-A4EE-7D6A9A8E96A5][component-model-creator]].

* Templates

Scaffold profiles (each generates 9 files for =.api=/=.core=; 18 for =.service=):

| Profile | Key templates |
|---|---|
| =--profile component-api= | =cmake_component_root.mustache= + =cmake_component_api_src.mustache= + 7 others |
| =--profile component-core= | =cmake_component_root.mustache= + =cmake_component_core_src.mustache= + 7 others |
| =--profile component-service= | =cmake_component_root.mustache= + =cmake_component_service_src.mustache= + 16 others |

Full template list is in [[id:08FBD248-257A-A474-DD43-DB08949978F1][ORE Studio Codegen]]§"Profile catalogue".
