* Question

How do I generate the sprint health chart CSVs (and then the PNGs) for
a sprint using compass?

* Answer

1. /Generate CSVs for the current sprint/ (auto-detected):

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh sprint charts
   #+end_src

2. /Generate CSVs for a specific sprint/:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh sprint charts --sprint 19
   #+end_src

3. /Then render the four PNGs with gnuplot/:

   #+begin_src sh :results verbatim
   gnuplot -e "sprint=19" scripts/sprint_prs_commits.gnuplot
   gnuplot -e "sprint=19" scripts/sprint_line_churn.gnuplot
   gnuplot -e "sprint=19" scripts/sprint_pr_cycle.gnuplot
   gnuplot -e "sprint=19" scripts/sprint_stories_done.gnuplot
   #+end_src

   =compass sprint charts= writes three TSV files under
   =build/output/sprint_NN/= (=sprint_activity.csv=,
   =sprint_progress.csv=, =pr_cycle_times.csv=); gnuplot reads those
   and writes the PNGs into =doc/agile/versions/v0/sprint_NN/=.

4. /Optional overrides/:

   #+begin_src sh :results verbatim
   ./projects/ores.compass/compass.sh sprint charts \
     --sprint 19 \
     --start-date 2026-05-29 \
     --end-date 2026-06-05 \
     --output-dir /tmp/sprint_out
   #+end_src

* Script

=compass sprint charts= lives in
=projects/ores.compass/src/compass.py= (=cmd_sprint_charts= +
=_sc_*= helpers). The CMake =sprint_charts= target calls it
automatically when =cmake --build … --target sprint_charts= is run.

* Tested by

Manual: run =compass sprint charts --sprint 19= and confirm three TSV
files appear under =build/output/sprint_19/=.
