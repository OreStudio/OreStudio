* Goal

Stories can be blocked at multiple levels: story C blocked by B, B blocked
by A. =compass where= currently shows STARTED items but does not surface
blocked chains or their root causes. This task extends =compass where= to
display blocked story dependency trees in the Orient pillar output, so the
user can see the full chain at a glance without opening individual files.

* Acceptance

- =compass where= output includes a "Blocked chains" section listing every
  story that is =BLOCKED=, with its =Waiting on= link, followed by that
  story's blocker recursively — presented as an indented tree.
- Cycles (A blocked-by B, B blocked-by A) are detected and reported as
  "⚠ cycle detected" rather than looping infinitely.
- Stories with no blocked entries produce no "Blocked chains" section.
- =--format json= output includes a =blocked_chains= key with the tree
  as nested objects.
- CI passes.

* Plan

- Read story state from each =story.org= in the current sprint (same glob
  as =compass sprint status=).
- For each story with =State: BLOCKED=, extract the =Waiting on= field
  (org-roam link or plain text) to identify the blocker.
- Recurse until the chain reaches a non-blocked story or a cycle.
- Render as indented lines under a "Blocked:" header in =compass where= output.
