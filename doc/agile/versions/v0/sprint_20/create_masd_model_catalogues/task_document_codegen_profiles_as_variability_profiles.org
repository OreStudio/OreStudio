* Goal

Make the variability configuration of the code generator discoverable in the org-roam knowledge graph by creating one org-mode document per logical profile group in =profiles.json=. Each document names the MASD facet (or facet group, or component archetype) the entry corresponds to, lists the model types it binds to, describes the artefacts it activates, and links to the relevant literate template org files.

Concrete deliverables:
- 16 facet documents (=sql=, =domain=, =repository=, =service=, =qt=, =protocol=, =nats-eventing=, =nats-handler=, =generator=, =non-temporal-sql=, =non-temporal-domain=, =non-temporal-repository=, =plantuml=, =enum=, =field-group=, =service-registry=)
- 3 facet-group documents (=all-cpp=, =all=, =non-temporal=)
- 4 component-archetype documents (=component=, =component-api=, =component-core=, =component-service=)
- All wired into the [[id:43FF3591-AD18-4CE3-8239-A220862C9B5A][Applied MASD]] and [[id:0BD6F9EC-34DD-44D9-8AA1-917EE5966396][ORE Studio Variability Model]] knowledge docs via =[[id:...]]= links.

* Result

Closed *DONE — superseded*, not by building the 23 planned per-profile
documents but by a deeper correction that made them the wrong artefact.

A variability deep-dive (the [[id:0BD6F9EC-34DD-44D9-8AA1-917EE5966396][ORE Studio Variability Model]] rewrite)
established that =profiles.json='s entries are *facets* (physical-space
activation), not VMM /variability profiles/ — and that the genuine
non-structural variability lives in the authored configuration knobs
(=sql.*=, =qt.*=, licence), not in the activation catalogue. The task's
premise — "document codegen profiles /as variability profile/ org
files" — therefore embodied the same facet/variability conflation the
deep-dive removed. Producing one "variability profile" doc per profile
group would have re-enshrined that error and duplicated artefacts that
already exist.

The task's real goal — make the generator's variability discoverable in
the knowledge graph — is met by what already exists plus this sprint's
work:

- *Facet discoverability*: the literate template facet/group docs
  (=sql_schema.org=, =cpp_qt.org=, the =*_group.org= indexes, …) created
  by the cross-link story already document, per facet, the templates and
  model types each activates, and are catalogued from [[id:43FF3591-AD18-4CE3-8239-A220862C9B5A][Applied MASD]].
- *Variability discoverability*: the rewritten [[id:0BD6F9EC-34DD-44D9-8AA1-917EE5966396][Variability Model]] now
  documents both VMM halves — activation (Part 1) and the feature
  catalogue of configuration knobs (Part 2) — and the relevant facet
  docs link to it.

No new per-profile docs were created by design. The sibling task to make
=profiles.json= a tangled artefact ([[id:BFBFB825-18EF-4274-A72E-5924E69FFBAC][Migrate profiles.json to a literate
facet_catalogue.org source]]) moved to the [[id:D85E2B30-8C64-46DF-819D-B50D2E70B73C][codegen org-model migration]]
story for local1 to pick up.
