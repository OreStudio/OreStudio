## Summary
- `lld-link: undefined symbol: ores::qt::CronExpressionWidget::staticMetaObject` when linking `ores.qt.scheduler.dll` on Windows.
- The widget lives in `ores.qt.compute` but is consumed by both `ores.qt.compute` (ReportDefinitionDetailDialog) and `ores.qt.scheduler` (JobDefinitionDetailDialog). On Windows the Qt-MOC `staticMetaObject` was not exported across the compute DLL boundary for scheduler to pick up.
- Move `CronExpressionWidget`, `CronEditorDialog`, and `CronFieldWidget` to `ores.qt.api` — the existing home for shared cross-plugin widgets (`PaginationWidget`, `ProvenanceWidget`). Both compute and scheduler already link `ores.qt.api.lib` publicly, so no consumer changes are required.
- Add `ores.scheduler.api.lib` to `ores.qt.api` link libraries (needed by `CronExpressionWidget::isValid` which calls `scheduler::domain::cron_expression::from_string`).
