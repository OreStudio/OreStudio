* Steps

Each step is its own heading — the title should be short (it's shown
as a single list entry in the QA Validation Runner); put any longer
instructions in the body below the title. The panel writes each
step's PASS/FAIL/PENDING outcome and notes back as a =*** Result=
child heading directly under it.

** Log in and select Barclays Plc

Log in as =tenant_admin@barclays_plc= / =Secure-Password-123=. In the
party picker, select =BARCLAYS PLC=. Confirm the main window enables
its menus (Reference Data, Trading, etc.) once logged in.

** Book Statuses list window opens and shows seeded rows

Open Trading > Trading Codes > Book Statuses. Confirm exactly three
rows appear — Active, Closed, Frozen — each with Description and
Display Order populated, no error banner.

** Detail dialog opens and displays fields correctly

Double-click the "Active" row. Confirm the detail dialog opens showing
Code (read-only), Name, Description, Display Order, Version, and
Modified By populated correctly, with no crash or blank fields.

** Edit and save a book status

Change the "Active" row's Description to something new and Save.
Confirm the save succeeds with no error, the grid refreshes to show
the new description, and Version increments by 1.

** History dialog opens and shows the new version

Right-click "Active" > View History (or the equivalent toolbar
action). Confirm both versions (before and after the edit above)
appear in the history grid, each with distinct valid_from/valid_to
timestamps, and that closing the history dialog does *not* close the
whole application (regression class from PR #1449's =window()->close()=
bug — confirm it only closes the history sub-window).

** No error toast or log error during the whole flow

Check the client's log tail
(=build/output/<preset>/publish/log/ores.qt.<colour>.log=) for any
ERROR-level line logged during the steps above — in particular,
confirm no "Server reported failure" message appears (the new
success-check added to ClientBookStatusModel.cpp), which would
indicate the backend rejected a request that the UI didn't surface as
a visible error.
