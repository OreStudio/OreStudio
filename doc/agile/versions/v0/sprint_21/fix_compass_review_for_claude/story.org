* Goal

Restore the compass PR review round workflow after the switch from Gemini to Claude as the
automated code reviewer. Claude posts its reviews as issue-level conversation comments, not as
GitHub review threads with inline line comments. This structural difference breaks two compass
commands:

- =compass review reply <PR> <comment-id> "<message>"= fails with HTTP 404 because it calls the
  review-comment reply API, which does not accept issue comment IDs.
- =compass review resolve <PR>= reports "no unresolved threads" because Claude's comments are not
  GitHub review threads and therefore have no resolvable threads to close.

Update compass so that both commands work correctly whether the reviewer is Claude (issue comments)
or a human reviewer (review threads with inline comments).

* Acceptance

- =compass review list <PR> --detail= clearly distinguishes Claude bot comments (issue comments)
  from human review threads (inline review comments).
- =compass review reply <PR> <comment-id> "<message>"= works for both issue comment IDs (Claude)
  and review comment IDs (human inline comments), routing to the correct GitHub API endpoint.
- =compass review resolve <PR>= resolves human review threads when present; when the only review
  comments are Claude bot issue comments (which have no resolvable thread), it reports a clear
  status rather than a misleading "no unresolved threads".
- The =pr-address-review= skill and runbook are updated (or remain consistent) with the new
  compass behaviour.
