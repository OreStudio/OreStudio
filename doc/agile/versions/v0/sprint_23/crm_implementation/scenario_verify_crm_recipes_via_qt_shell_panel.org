* Notes

Previous run (2026-07-16T21:12:02Z, commit 846904dd3) failed on two
points, both fixed since: the single =crm_rates.ores= recipe has been
split into per-CRM recipes (=crm_rates_majors=/=exotics=/=scandies=)
plus a dedicated =crm_rates_matrix= recipe; the flat table's
misalignment was Unicode arrow glyphs (▲/▼/—) breaking libfort's
byte-based column-width calculation in =crm_rate_formatter= --
replaced with plain ASCII +/- signs. Also dropped the =--party=
requirement entirely: =nats_client::login_info= now carries
=default_party_id= from the login response, populated at all three
places a shell session authenticates -- =accounts_commands::process_login=,
=application.cpp::auto_login=, and (found by this scenario's first
re-run) =ShellMdiWindow='s own internal re-login for the Qt-embedded
shell, which was still missing it -- so =crm rates= (and any future
party-scoped command) uses the logged-in account's own party by
default everywhere, matching Qt's =ClientManager::currentPartyId()=.
=--party= is now purely an override for an account managing more
than one party.
