* Goal

Establish the first project database table — for currency reference
data — and the code path that writes parsed currencies into it. This
task defines the pattern every later reference-data type follows.

* Acceptance

- A table for currency data exists in the project database.
- Currency rows can be inserted and retrieved.
- The DB write path is wired into the currency import code path.

* Context

Pathfinder task for the wider /currencies end-to-end/ story. The shape
chosen here propagates to every subsequent SQL component in the
project.

* Notes

- Total time spent in v0: ~1:19.
- This task pre-dated the codegen-driven SQL schema; the table was
  hand-written. Later cleanup folded it into the generated schema.

* Result

- Initial currency table committed to the project database under
  =ores.refdata= (then later moved as the schema evolved).
