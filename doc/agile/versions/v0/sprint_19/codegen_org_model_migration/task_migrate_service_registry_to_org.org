* Goal

Convert =projects/ores.codegen/models/services/ores_services_service_registry.json=
to =projects/ores.services/modeling/ores.services.service_registry.org=. The
service registry is a single global catalogue of ORE Studio
backend services with their psql vars, IAM roles, DDL grant
prefixes and read-only select prefixes. Codegen renders it
through the =service-registry= profile into 5 outputs (shell
vars, IAM users, grants, account/role populates).

Extend the codegen pipeline with a new
=ores.codegen.service_registry= kind: dedicated
=load_org_service_registry_model= producing the JSON-equivalent
dict shape, filename-suffix dispatch on =service_registry.org=,
and exclusion from the generic entity match so routing is
unambiguous.

* Plan

Org file shape:

- One top-level =* <service name>= heading per service.
- Property drawer carries the scalars =:psql_var:=,
  =:env_key:=, =:iam_role:=, =:description:=, =:email:=, plus
  optional =:role:= (only =synthetic= has this today).
- =** DML prefixes= / =** Select tables= / =** Select prefixes=
  sub-headings carry the per-service lists as org bullets
  (=- <prefix>= per line).
- Suffix-routing uses bare =service_registry.org= (no
  underscore prefix) because the filename leads with the
  =ores.services.= namespace (=ores.services.service_registry.org=);
  the underscore convention used by =_table.org= /
  =_lookup_entity.org= would not match the dot-separated form
  here.
