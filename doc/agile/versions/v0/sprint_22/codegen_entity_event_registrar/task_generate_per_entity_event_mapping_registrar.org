* Goal

Add a register_<entity>_event_mapping(event_source, event_bus, nats) function to codegen's per-entity registrar output (same generated file pair as register_<entity>_handlers), add a small service-layer aggregator that calls it for every entity, and replace application.cpp's hand-wired register_mapping/event_bus.subscribe blocks with calls to the aggregator — starting with purpose_type, then migrating the rest of ores.refdata's entities.

* Acceptance

- Codegen emits register_<entity>_event_mapping into the existing generated <entity>_registrar.hpp/cpp pair for domain_entity models with eventing enabled.
- A service-layer aggregator (colocated with application.cpp) calls each generated per-entity function; application.cpp's entity-change-event pipeline section collapses to a single aggregator call.
- purpose_type migrated first and cross-client NATS eventing re-verified working; remaining ores.refdata entities in application.cpp migrated with zero loss of functionality.
- Pattern documented (recipe or knowledge doc) so other components can adopt it.

* Plan

Promoted the duplicated =publish_entity_event= helper into
=ores.eventing.core= first, so the generated registrar would have a
shared function to call. Added =component_service=/=component_service_dir=
template variables (mirroring the existing =component_core= pair) and a
new =ores.cpp.nats-event-registrar= facet with two archetypes, output
into the =.service= layer since the eventing pipeline types aren't
available in =.core=. Verified against a UUID-keyed entity (=book=,
=entity_singular_short=-prefixed id field) and a text-keyed one
(=purpose_type=, =primary_key.column=-suffixed) to get the mustache
branching right. Regenerated all 16 currently hand-wired refdata
entities, found 10 had stale =nats-eventing= output (a breaking
id-field rename), backed those out and deferred to a capture rather
than risk the rename. Wrote the =event_registrar= aggregator and
migrated the remaining 6 entities' =application.cpp= wiring to it.
Along the way, discovered =purpose_type='s Qt facet had never been
regenerated since a shared history-dialog Close-button fix landed on
main — resynced it.
