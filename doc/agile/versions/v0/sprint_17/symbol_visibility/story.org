* Goal

Apply =-fvisibility=hidden= globally so that only explicitly exported symbols
cross the shared-library boundary. Fix every missing =EXPORT= macro, break
cross-plugin compile-time dependencies, and build CLI and shell as static
libraries so they link cleanly on Windows.

* Acceptance

- All tasks complete; PR-by-PR breakdown in =git log=.

* Tasks

In execution order:

- Symbol visibility migration: -fvisibility=hidden across all shared libraries (#738)
- Fix missing export symbol definitions in trading.api and trading.core (#743)
- Break cross-plugin symbol dependencies — plugin isolation phase 1 (#749)
- Build ores.shell.lib as STATIC to fix Windows plugin DLL link (#764)
- Build ores.cli.lib as a static library (#774)
- Fix missing ORES_SHELL_EXPORT on host class (Windows linker error) (#780)
- Add export macro to parser class for DLL visibility (#791)

* Decisions

- /Static CLI and shell/ :: building =ores.cli.lib= and =ores.shell.lib= as
  STATIC avoids Windows DLL export count limits and simplifies the link graph.
- /Plugin isolation phase 1/ :: cross-plugin compile-time dependencies were
  broken explicitly in a dedicated PR; runtime dependencies via plugin APIs are
  retained.

* Out of scope

- Full plugin runtime isolation (deferred to v1).
