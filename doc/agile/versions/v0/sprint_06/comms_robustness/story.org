* Goal

Make the comms substrate production-ready: structurally separate
handshake from session, split the bloated =protocol.hpp= along
component lines, cover the stack under multi-threaded use, add retry
with give-up notification to the client.

* Acceptance

- Each component's message types live in its own header.
- A dedicated =handshake_service= owns version negotiation.
- Multi-threaded integration tests in place; identified races fixed.
- Client retries transient failures; raises a give-up event on
  exhaustion.

* Decisions

- /Protocol per component/ :: the previous monolithic =protocol.hpp=
  meant every protocol change touched every component. Splitting along
  component lines means changes are local to ownership.

* Out of scope

- Cross-process eventing (the LISTEN/NOTIFY thread starts but the
  full client-fan-out is a later sprint).
