* Goal

Add a =compass entity= pillar that treats entities as first-class objects
across the whole codebase. Today the only handles are model file paths and
component names; this pillar surfaces entity-level operations so a developer
never needs to know where files live or which profiles apply.

Five sub-commands:

1. *list* — enumerate every known entity across all components; return
   name, ID (org-roam UUID), component, and model path.
2. *add <name>* — scaffold a new entity: run codegen to create the model
   org file and wire it into the component. Delegates to existing doc-generate
   machinery.
3. *generate <name-or-id>* — regenerate all applicable codegen profiles for
   the entity (resolves model path from the index; no path required from the
   user). Equivalent to running =compass codegen generate= for every profile
   that applies to that entity's model type.
4. *show <name-or-id>* — list every file that codegen produces for the entity
   across all components, with staleness: whether the file is older or newer
   than the template that generates it.
5. *diff <name-or-id>* — show a unified git diff of all generated files for
   the entity; useful immediately after a =generate= to review what changed
   before committing.

All sub-commands call Python functions directly; no shelling out to =codegen.sh=
or =git= wrappers except where =git diff= is the canonical output.

* Acceptance

- =compass entity list= prints a table of entity name, org-roam ID, component, and model path for every entity in the index.
- =compass entity add <name> --component <comp>= scaffolds the entity model file via codegen and wires it into the component.
- =compass entity generate <name-or-id>= regenerates all applicable profiles for that entity; accepts =--profile= to restrict to one layer; accepts =--dry-run=.
- =compass entity show <name-or-id>= lists all generated output files with a staleness indicator (file mtime vs template mtime).
- =compass entity diff <name-or-id>= prints a unified git diff of all generated files belonging to that entity.
- All sub-commands resolve the entity by name or org-roam UUID prefix; no model path required from the user.
- Implementation calls Python functions directly; no subprocess to =codegen.sh= or bash wrappers.
