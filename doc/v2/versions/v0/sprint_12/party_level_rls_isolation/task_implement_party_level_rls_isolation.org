* Goal

Add a second RLS layer scoped to the party subtree within a tenant. System parties see all; operational parties see only their subtree.

* Acceptance

- SQL: =ores_refdata_visible_party_ids_fn= (recursive CTE traversing party hierarchy); =ores_iam_current_party_id_fn= + =ores_iam_visible_party_ids_fn= session helpers.
- C++: =tenant_aware_pool= + =context= grow optional =party_id= + =visible_party_ids=; =SET app.current_party_id= + =SET app.visible_party_ids= on acquire.
- =context::with_party()= creates a party-aware context.
- =session_data= + =session_info= carry party context.
- Login flow resolves party via =account_parties=; auto-select for single-party users; compute visible set via the SQL function.
- =ores_refdata_party_counterparties_tbl= as the first party-scoped junction with dual (tenant + party) RLS.
- Junction codegen fixes for tenant_id / UUID / schema / field naming / service-protocol exclusion.
- Integration tests: system party sees all; leaf party sees only its subtree; cross-party data invisible.
