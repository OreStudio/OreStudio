#+description: Codegen terminology is inconsistent and misaligned with MASD. Problems: (1) domain_entity/junction are MASD stereotypes, not 'types' — a type is currency or country; calling them types conflates stereotype with model instance. (2) profile/layer/tier/facet used interchangeably: profile = named set of facets (correct, keep); facet = single codegen concern — this IS the MASD term, already in facet_catalogue.org, must be used consistently everywhere instead of 'layer'; layer used loosely for SQL/C++/Qt groups = in MASD these are technical spaces; tier (used in task 2EA448B4) = wrong word, should be stereotype group. (3) Technical spaces map to components: SQL space = ores.sql, C++ refdata space = ores.refdata, Qt space = ores.qt.refdata — this correspondence is implicit and undocumented. (4) 'type' is overloaded: model type (stereotype) vs field type (text, integer, uuid). Action: audit all uses across codegen source, facet_catalogue, templates, compass commands, docs, runbooks; define canonical MASD-aligned glossary; update everything; fix task 2EA448B4 before implementation.
