* Goal

Redesign the Data Librarian's dataset detail dialog
(=DatasetViewDialog=, the "accession card") away from its current
single-column, alternating-row property-sheet table toward a
persistent-header, asymmetric two-column layout, per
[[id:211CB159-F633-4897-A899-56CD91625396][OreStudio Data Librarian UI/UX Review]]. Phased: cosmetic wins
(header banner, UUID copy, status pills) before the structural
column/tab rework. *Dependencies* already has a working interactive
lineage graph (=QGraphicsScene= node/socket/connection rendering,
not flat rows as the review assumed) — out of scope for this story,
only cosmetic tab-count changes touch it, if any.

* Acceptance

- A persistent header banner replaces the per-tab repeated
  =Name=/=Version=/=ID=/=Code= fields, with a one-click UUID copy
  action.
- Status identifiers (=Primary=, =Actual=, =Raw=, etc.) render as
  visual pills instead of plain text.
- *Provenance* and *Methodology* tabs are merged into a single tab;
  lineage primitives (=Lineage Depth=, =Upstream Derivation=) are
  visually prominent rather than buried at the bottom of a property
  grid.
- The dialog uses an asymmetric two-column layout (primary content +
  metadata sidebar) instead of a single-column table, for *Overview*
  and the merged *Provenance & Governance* tab.
- =DetailDialogBase= (shared by 78 other entity dialogs) is
  untouched; the new layout lives in =DatasetViewDialog= only.

* Decisions

- The dialog turned out to be =DatasetViewDialog= (the read-only
  4-tab viewer), not =DatasetDetailDialog= (the 2-tab create/edit
  form) the story originally named — corrected mid-task.
- *Dependencies* already had a working =QGraphicsScene= lineage
  graph, contrary to the source review's assumption of flat rows —
  no rework needed there.
- Governance badges must go through the same rendering path as
  everywhere else in the app (=DelegatePaintUtils::draw_centered_badge=
  + =BadgeCache=, via a small =QStyledItemDelegate=), not a one-off
  =QLabel=/stylesheet pill — caught only by manually testing in the
  running client, not by compiling.
- =DatasetViewDialog= is now a =QWidget= embedded in a
  =DetachableMdiSubWindow= (was a floating, always-on-top =QDialog=)
  — matches every other window in the app.

* Out of scope

- *Dependencies* tab internals — it already has a working
  interactive lineage graph, contrary to the review's assumption of
  flat rows.
- Any change to =DetailDialogBase= or the other 77 entity dialogs
  built on it.
