#+title: ores-shell script library
#+description: Runnable .ores scripts generated from the shell recipes.

This directory holds the ores-shell script library that the shell's
=load= command (and the Qt script panel) consume. The scripts are
*generated*, not hand-written: every =.ores= under [[file:library/][=library/=]] is tangled
from a shell recipe under =doc/recipes/shell/=, which is the single
source of truth.

* Layout

- =library/<category>/= — the generated =.ores= artefacts, grouped into
  category sub-folders (=provisioning/=, =accounts/=, =tenants/=,
  =compression/=, …) taken from each recipe's category filetag, so the
  library is browsable rather than one flat list. Each script leads with
  its recipe title and description as =#= comments, then a
  =# GENERATED … do not edit by hand= banner, and is committed so the
  shell and the Qt panel load it directly. The whole tree is packaged
  (installed to =share/ores/shell-scripts=, structure preserved).
- There are no =.org= sources here: a recipe at
  =doc/recipes/shell/<name>.org= tangles to
  =library/<category>/<name>.ores=.
- The trailing =exit= a recipe uses to terminate org-babel's REPL is
  stripped from the generated script — =load= and the UI must not close
  the shell.

* Authoring

To add or change a script, edit (or create) its *recipe*, never the
=.ores=. See [[id:D4E65EF1-54F4-47E1-9388-58BE9E36FBE3][How do I create a recipe?]]. The recipe's =ores-shell= src
block is the script body; =load='s stop-on-error semantics mean an
error-free completion is meaningful, so order the commands so failures
abort early.

* Generating

=compass build --direct tangle_shell_scripts= regenerates the whole
library (Emacs only, no cmake/vcpkg; the equivalent =tangle_shell_scripts=
CMake target works in a full environment):

#+begin_src sh
./compass.sh build --direct tangle_shell_scripts
#+end_src

It runs =projects/ores.lisp/src/ores-build-recipe-scripts.el=, which
tangles each recipe's =ores-shell= block into =library/<name>.ores=
(via =org-babel-tangle-file='s TARGET-FILE, with LANG-RE keeping the
=sh= runner blocks out) and prepends the generated header.

* Running

#+begin_src sh
ores.shell
ores-shell> load projects/ores.shell/scripts/library/provisioning/barclays_system_provision.ores
#+end_src

or, with the connection inherited from the environment:

#+begin_src sh
compass shell -f barclays_system_provision.ores
#+end_src
