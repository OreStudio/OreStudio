* Plan

The audit (commit-this-page) already located the gaps; this plan executes against them in dependency order.

1. *Patch the v2 doc templates.* For each =v2_doc_*.org.mustache=, replace lowercase concept-link UUIDs with their uppercase canonical form:
   - =v2_doc_story.org.mustache=: =699a8d45-...= → =699A8D45-B67E-4D3E-9206-24FA17C51ADA= ([[id:699A8D45-B67E-4D3E-9206-24FA17C51ADA][story]])
   - =v2_doc_task.org.mustache=: =31c868b9-...= → =31C868B9-306E-4282-BA8C-07E647021B34= ([[id:31C868B9-306E-4282-BA8C-07E647021B34][task]])
   - =v2_doc_sprint.org.mustache=: =0820b7fd-...= → =0820B7FD-147C-4832-AC25-C043D38D5B61= ([[id:0820B7FD-147C-4832-AC25-C043D38D5B61][sprint]])
   - =v2_doc_version.org.mustache=: =ca4d4bde-...= → =CA4D4BDE-2EAB-4F0A-BC8B-7C0BCB17BC82= ([[id:CA4D4BDE-2EAB-4F0A-BC8B-7C0BCB17BC82][version]]); =436fb40c-...= → =436FB40C-422A-465B-B949-E9F0760820C1= ([[id:436FB40C-422A-465B-B949-E9F0760820C1][product identity]])
   - =v2_doc_capture.org.mustache=: =671f18e4-...= → =671F18E4-E09C-4B3B-BD24-D33DF8AE38A6= ([[id:671F18E4-E09C-4B3B-BD24-D33DF8AE38A6][capture]])

   Look up each canonical ID with =grep -n '^:ID:' doc/meta/glossary.org doc/meta/document_types.org= before editing — do not eyeball the casing.

2. *Add defensive =.upper()= calls in =projects/ores.codegen/src/v2_doc_generate.py=.* The line that builds the variables dict (currently around line 332) passes =args.parent_id= and =args.predecessor_id= as-is. Wrap both in =.upper() if x else ""= so a lowercase argument still renders an uppercase link. Mirror the pattern already used for =new_id= at line 318.

3. *Fix the release-notes UUID.* In =build/scripts/generate_release_notes.py=, change =str(uuid.uuid4())= to =str(uuid.uuid4()).upper()= (currently at line 316).

4. *Uppercase the hardcoded constants in =regenerate_backlog_indexes.py=.* Five literals identified by the audit: =GLOSS_CAPTURE=, the two =bucket_ids= values (=near=, =far=), and two link references in the docstring template strings. =sed -i 's/\([0-9a-f-]\{36\}\)/\U\1/g'= on the file is correct here — the file contains no other 36-char hex spans.

5. *Run the regression sweep.* From repo root:
   #+begin_src sh :results verbatim
   # any lowercase :ID:
   grep -rEn '^:ID:\s+[A-Za-z0-9-]*[a-f]' --include='*.org' doc/ projects/

   # any lowercase id-link, skipping template placeholders
   grep -rEn '\[\[id:[0-9a-f-]{20,}[a-f][0-9a-f-]*\]' --include='*.org' doc/ projects/
   #+end_src
   Both must return zero lines.

6. *Smoke-test the codegen end-to-end.* Generate a throwaway story and task under =/tmp/uuid-smoke/=:
