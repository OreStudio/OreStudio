#+title: Don't override --instance-name on compass client start
#+description: Overriding --instance-name for parallel Qt clients strips the environment identity from the client; vary --colour only.
#+type: memory
#+memory_subtype: feedback
#+level: cross
#+filetags: :memory:feedback:
#+created: 2026-07-06
#+updated: 2026-07-06

When starting multiple Qt clients in parallel with =compass client start=,
do not pass =--instance-name=. Leave it unset so it defaults to the
environment's own label (=ORES_CHECKOUT_LABEL=, e.g. =eager_maxwell=).
Distinguish parallel instances with =--colour= only.

*Why:* =compass_services.py= sets =--instance-name= to =ctx.label= by
default precisely so the client's status bar/tray shows which environment
it belongs to. Overriding it to an arbitrary name (e.g. =clientA=,
=clientB=) strips that identity, making the running client
indistinguishable from any other environment's client in the fleet/status
view.

*How to apply:* Whenever launching two or more Qt clients from the same
environment for a manual test (e.g. verifying NATS eventing across two
sessions), start them with distinct =--colour= values and no
=--instance-name= override.
