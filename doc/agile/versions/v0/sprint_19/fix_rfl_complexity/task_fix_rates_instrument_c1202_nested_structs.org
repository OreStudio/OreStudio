* Goal

All 9 types in =rates_instrument_variant= have 15–20 fields, all above the MSVC
Mode 2 C1202 threshold (~12 fields in a clean TU). After task 10 fixed
=swap_leg=, the next struct to fire in =ClientManagerExportPortfolio.cpp= is
=inflation_swap_instrument= (18 fields), followed by each of the remaining 8
types in turn. See [[id:D75F4921-B431-4E78-99AB-62A47F761D1D][Investigation: MSVC C1202 and rfl complexity]] for the full
taxonomy.

Field audit:

| Struct | Fields |
|--------+--------|
| =fra_instrument= | 20 |
| =swaption_instrument= | 19 |
| =knock_out_swap_instrument= | 18 |
| =inflation_swap_instrument= | 18 |
| =vanilla_swap_instrument= | 17 |
| =rpa_instrument= | 17 |
| =callable_swap_instrument= | 17 |
| =balance_guaranteed_swap_instrument= | 16 |
| =cap_floor_instrument= | 15 |

All share a common 12-field block: version, tenant_id, workspace_id,
instrument_id, trade_type_code, party_id, trade_id (identity — 7) +
modified_by, performed_by, change_reason_code, change_commentary,
recorded_at (audit — 5).

Fix: extract =instrument_identity= (7 fields) and =instrument_audit= (5 fields)
as shared nested sub-structs. Each instrument becomes:

#+begin_src cpp
struct <type>_instrument final {
    instrument_identity identity;   // 7 fields
    /* type-specific fields (≤8) */
    instrument_audit    audit;      // 5 fields
};
#+end_src

Parent Literal: 3 fields (or identity + n_specific + audit). No Literal exceeds
9 fields. Wire format changes to nested JSON (same justification as task 10 —
internal-only, server and client in same repo).
