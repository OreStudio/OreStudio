Expose =compass codegen generate= and =compass codegen regenerate= subcommands that import and call =codegen.generate.cmd_generate= / =cmd_regenerate= directly — no bash intermediary. Compass becomes the single documented entry point for code generation. Wire any existing callers (recipes, skills, runbooks) to use the =compass codegen= verbs instead of calling =codegen.sh= directly.

* Result

Added =cmd_codegen= to =projects/ores.compass/src/compass.py= (≈70 lines). The
function parses =generate= / =regenerate= subcommands with identical flags to
=codegen/cli.py=, extends =sys.path= to make =ores.codegen= importable, then
imports and calls =cmd_generate= / =cmd_regenerate= from =codegen.generate=
directly — no subprocess, no =codegen.sh=. =compass codegen= is registered in
the dispatch chain, the =_KNOWN_COMMANDS= list, and the help epilog.

Verified:
- =compass codegen --help= lists both subcommands.
- =compass codegen generate --model ores.refdata.country.org --profile protocol --dry-run= resolves =country_protocol.hpp=.
- =compass codegen generate --model ores.refdata.country.org --profile all-cpp --dry-run= resolves all 18 C++ artefacts correctly.
